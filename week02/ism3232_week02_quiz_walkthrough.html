Pay particular attention to Q8 (rm safety ritual) and Q13 (the SA explanation) — these are the highest-stakes questions because the concepts prevent data loss.
Question-by-question walkthrough
1
Multiple Choice — Auto-graded
What is the correct command to move up one directory level?
A.cd ~○
⚠ cd ~ jumps to the home directory — a different destination, not one level up.
B.cd ..✓
C.cd /○
⚠ cd / jumps to the root of the entire filesystem — completely different from going up one level.
D.cd up○
⚠ 'cd up' is not a valid command in any shell.
💬 Discussion — why B is correct

cd .. moves up one level. The .. always refers to the parent directory. Note the space between cd and ..

2
Multiple Choice — Auto-graded
Which command lists ALL files including hidden ones with permissions?
A.ls○
⚠ ls shows visible files only — hidden files starting with . are not shown.
B.ls -l○
⚠ ls -l adds the long format (permissions, size, date) but -a is needed to show hidden files.
C.ls -la✓
D.ls -h○
⚠ ls -h adds human-readable file sizes but does not show hidden files.
💬 Discussion — why C is correct

ls -la uses two flags: -l (long format) and -a (all files including hidden). Hidden files start with a dot like .venv and .gitignore.

3
Multiple Choice — Auto-graded
What does the .. symbol represent in a file path?
A.The current directory○
⚠ / is the root of the entire filesystem — the very top level, not your home directory.
B.The home directory○
⚠ . (single dot) is the current directory, not the home directory.
C.The parent directory✓
D.The root directory○
⚠ .. (double dot) is the parent directory — one level up from where you are.
💬 Discussion — why C is correct

.. is a shortcut for the parent directory — one level up. A single dot . means the current directory.

4
Multiple Choice — Auto-graded
Which command creates an empty file called notes.txt?
A.mkdir notes.txt○
⚠ mkdir creates directories, not files. mkdir notes.txt would create a folder named notes.txt.
B.create notes.txt○
⚠ 'create' is not a shell command.
C.touch notes.txt✓
D.new notes.txt○
⚠ 'new' is not a shell command.
💬 Discussion — why C is correct

touch creates an empty file if it does not exist. It is the standard way to create new files from the terminal.

5
Multiple Choice — Auto-graded
What is the correct command to copy notes.txt to a file called backup.txt?
A.mv notes.txt backup.txt○
⚠ mv renames or moves the file — the original disappears from the source location.
B.cp notes.txt backup.txt✓
C.copy notes.txt backup.txt○
⚠ 'copy' is not a standard Unix command.
D.dup notes.txt backup.txt○
⚠ 'dup' is not a shell command.
💬 Discussion — why B is correct

cp source destination copies a file leaving the original in place. mv moves or renames it — the original disappears from the source.

6
Multiple Choice — Auto-graded
What must you ALWAYS do before using the rm command?
A.Activate the venv○
⚠ The venv is not relevant to safe file deletion. Activating it changes nothing about rm's behaviour.
B.Run python3○
⚠ Running python3 does not protect against deleting files.
C.Run pwd and ls to confirm your location and what is there✓
D.Save all open files in VS Code○
⚠ VS Code file saves do not affect terminal file deletion.
💬 Discussion — why C is correct

rm is permanent — no Trash, no undo. Always run pwd (confirm location) and ls (confirm contents) before deleting anything.

7
Multiple Choice — Auto-graded
What does "tree -L 2" display?
A.Files modified in the last 2 days○
⚠ tree does not show files modified in the last N days — that is the 'find -mtime' command.
B.The directory tree limited to 2 levels deep✓
C.The 2 largest files○
⚠ tree does not report file sizes.
D.File permissions○
⚠ Permissions are shown by ls -la, not tree.
💬 Discussion — why B is correct

tree shows a visual directory structure. -L 2 limits the depth to 2 levels, showing a project overview without too much detail.

8
Multiple Choice — Auto-graded
What does "head -5 report.txt" display?
A.The last 5 lines○
⚠ head -5 shows the FIRST 5 lines. tail shows the last.
B.The first 5 lines✓
C.The 5 largest files○
⚠ '5 largest files' is shown by du or ls -lS, not head.
D.Lines 5 through the end○
⚠ There is no 'lines 5 through end' variant of head — that would be tail -n +5.
💬 Discussion — why B is correct

head -n displays the first n lines of a file. tail -n shows the last n lines.

14
Multiple Choice — Auto-graded
Which command prints the contents of a file directly to the terminal?
A.less○
B.open○
C.cat✓
D.print○
💬 Discussion — why C is correct

cat prints the entire file to standard output. less opens a scrollable viewer (press q to quit). open opens in the OS default app.

15
Multiple Choice — Auto-graded
What does "code ." do differently from "code myfile.py"?
A.They are identical○
B.code . opens the folder as a workspace; code myfile.py opens a single file✓
C.code . is faster○
D.code myfile.py is an error○
💬 Discussion — why B is correct

code . opens VS Code with the entire current folder as the project workspace. code myfile.py opens just that one file without workspace context.
