Section 1 — The zsh Prompt — Anatomy and the Golden Rule
1

The zsh Prompt — Anatomy and the Golden Rule 0–10 min

Lecture only · No typing yet

🗣 Say — opening
“Last week we installed the tools. This week we use them. Everything from here on happens in the terminal. Before we type a single command, I want you to understand what you are looking at.”
✏ Write on board — leave up for the session
yourname@MacBook-Pro ~ %     ← break down each piece: username · machine · current directory · prompt character
🗣 Say — the golden rule
“You are always somewhere. Every command you run executes relative to your current location in the filesystem. Knowing where you are — and how to move — is the skill that makes everything else work. We will reinforce this every class.”
💡 Key distinction to make
% is the zsh prompt character. $ is bash. If a student sees $ they are in bash, not zsh — stop and fix it before continuing.
Section 2 — Navigation Commands — Live Demo
2

Navigation Commands — Live Demo 10–35 min

Instructor demos each command · Students type immediately after

🗣 Say before starting
“I will run each command first. You watch. Then you run it. Do not type ahead. We do this together so nobody gets lost.”
Run each command — pause and explain the output before moving on
pwd               # Print Working Directory — where am I?
ls                # list visible files
ls -la            # list ALL files including hidden ones
cd Documents      # enter Documents (relative path)
cd ..             # go up one level
cd ~              # jump home from anywhere
cd ~/ism3232      # go to course folder
mkdir test_folder # create a directory
touch test_file.txt # create an empty file
clear             # clear the screen
tree              # show directory tree
tree -L 2         # tree limited to 2 levels deep
⚠
Common mistake to intercept early
Students write cd.. without the space — a bash habit. zsh requires the space. Correct it immediately when you see it; it becomes muscle memory fast.
💡 Pause point after ls -la
Ask: “What do you see that you did not see with plain ls?” Look for: hidden files starting with a dot (.venv, .gitignore, .zshrc will all matter later). Surface the concept now so it is not a surprise in Week 3.
Section 3 — File Operations — Live Demo
3

File Operations — Live Demo 35–55 min

Safety rule first, then demo each command

⚠
Safety rule — demonstrate this explicitly
Before any rm command: run pwd and ls. Show this yourself. Say: “I never delete without confirming where I am and what is here. This is not a beginner habit — it is a professional habit. Senior engineers do this.”
🗣 Say — rm is permanent
“Unlike the Trash or Recycle Bin, rm deletes immediately with no recovery. There is no undo. Demonstrate this — create a test file, show it with ls, rm it, show it is gone.”
File operations — demo each one
cp file.txt copy.txt          # copy
mv old.txt new.txt             # rename
mv file.txt ../other/          # move
cat file.txt                   # print contents
head -5 file.txt               # first 5 lines
tail -5 file.txt               # last 5 lines
less file.txt                  # scrollable (q to quit)
code .                         # open folder in VS Code
code hello.py                  # open specific file
# ALWAYS before rm:
pwd && ls
rm file.txt                    # permanent delete
📸
Screenshot moment — tell students: “Run pwd + ls before your rm command right now. Take a screenshot showing both commands and their output — this is Assignment 2 deliverable #2.”
Section 4 — Student Practice
4

Student Practice 55–70 min

Independent work · Instructor circulates

🗣 Say — instructions
“For the next 15 minutes, work through the practice sequence on the lecture page on your own. I will circulate. If you get stuck, try one more thing before raising your hand.”
💡 What to watch for while circulating
Students who finish early: ask them to explain what ls -la shows that ls does not, and what the permissions column means. Students who are stuck: check they are in the right directory first — 80% of errors are location errors.
⚠
Assignment 2 walkthrough — last 10 minutes
Read the five deliverables aloud. Emphasize: the rm screenshot must show the pwd + ls safety check immediately before the rm command. Phone photos of the screen are not accepted. README must list every command with a one-line description.
