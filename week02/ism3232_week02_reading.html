The zsh prompt — reading it before typing anything

The golden rule

You are always somewhere. Before you run any command — especially before rm — know exactly where you are. Run pwd and read the output.

What every part of the prompt means
yourname@MacBook-Pro ~/ism3232 %
│         │            │         └── zsh prompt character (type here)
│         │            └── current directory (~/ = home)
│         └── machine name
└── username

The % at the end is the zsh prompt character. Dollar sign ($) means bash — if you see this, stop and switch to zsh first.

Navigation commands

Run each command — pause and read the output before moving to the next
pwd                   # where am I right now?
ls                    # what files/folders are here?
ls -la                # all files including hidden ones (starts with .)
cd ~/ism3232          # navigate to course folder
cd module01_setup     # navigate into subfolder (relative path)
cd ..                 # go up one level
cd ~                  # jump home from anywhere
mkdir week2_practice  # create a new folder
cd week2_practice     # enter it
tree ~/ism3232 -L 2   # visualise the whole course structure
code .                # open current folder as VS Code workspace
⚠ Common mistake

cd.. without a space causes an error. It must be cd .. — two characters with a space between them.

File operations

File creation and manipulation
# Create files
touch notes.txt commands.txt hello_week2.py

# Write content to a file
echo 'Week 2 practice notes' > notes.txt

# Read file contents
cat notes.txt          # print whole file
head -5 notes.txt      # first 5 lines
tail -5 notes.txt      # last 5 lines

# Copy, rename, move
cp notes.txt backup.txt   # copy (original stays)
mv notes.txt renamed.txt  # rename (original gone)
mv renamed.txt ../        # move up one folder

The rm safety ritual — the most important section this week

⚠ rm is permanent

There is no Trash. There is no Ctrl+Z. When you rm a file, it is gone. Run this three-step ritual before every single rm command, every time, no exceptions.

The rm safety ritual — three steps, always in this order
# Step 1: confirm where you are
pwd

# Step 2: confirm what is here
ls

# Step 3: now — and only now — delete
rm filename_you_want_to_delete.txt

# Confirm it is gone
ls

Key terms this week

pwdPrint Working Directory — shows your current location.
lsList — shows the contents of the current directory.
ls -laList all, including hidden files, in long format.
cdChange Directory — move to a different folder.
mkdirMake Directory — create a new folder.
touchCreate an empty file (or update its timestamp).
cpCopy a file — the original stays in place.
mvMove or rename a file — the original is gone from the source.
catPrint the full contents of a file to the terminal.
rmRemove — permanently delete a file. No undo.
code .Open the current folder as a VS Code workspace.
