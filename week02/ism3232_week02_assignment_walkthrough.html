Requirement-by-Requirement Walkthrough
1
Navigation sequence — at least 8 commands

Student must show 8+ navigation commands in a single continuous terminal session. All must show output below them.

🔍 What the grader checks
At least 8 of: pwd, ls, ls -la, cd, cd .., cd ~, mkdir, touch, clear, tree -L 2
Commands run in a logical sequence (not random order)
Output visible below each command
zsh prompt (%) throughout — not bash ($)
💡 What counts as one command
Each unique command invocation counts separately. Running ls and then ls -la are two separate commands.
2
File operations sequence

All five operations must be demonstrated: cp, mv, cat, head, tail. Check each explicitly.

🔍 What the grader checks
cp: a file was copied and the copy exists (verify with ls after cp)
mv: a file was renamed or moved (verify with ls before and after)
cat: contents of a file printed to the terminal
head: first lines of a file shown (with -n or default 10 lines)
tail: last lines shown (accept any -n value)
⚠ Common mistakes at this step
Student shows the commands typed but no output — unacceptable without visible results
Student confuses cp and mv — cp leaves the original, mv does not
cat on an empty file shows nothing — check that the file had content first
3
rm safety ritual — screenshot required

This is the most important item in Assignment 2. The screenshot must show pwd THEN ls THEN rm — in that order, in one session.

🔍 What the grader checks
pwd command visible and output shows correct location
ls command visible immediately after pwd, before rm
rm command runs after both confirmation steps
A subsequent ls or the terminal output confirms the file is gone
✅ Full credit
pwd /Users/student/ism3232/module02_zsh/week2_practice ls commands.txt notes.txt scratch.py rm notes.txt ls commands.txt scratch.py
❌ Partial / no credit
Only shows rm notes.txt — no pwd or ls before it. Or shows them in wrong order (rm then ls).
⚠ Common mistakes at this step
Missing pwd before rm — most common failure on this requirement
pwd and ls shown but not immediately before rm (other commands between them)
The rm command line is cut off and not fully visible in the screenshot
4
hello_week2.py — created and run from terminal

File must have been created with touch (not by clicking New File in VS Code). Check the terminal sequence.

🔍 What the grader checks
touch command visible in the terminal (not created via VS Code UI)
File is at module02_zsh/week2_practice/hello_week2.py
python3 hello_week2.py run from the terminal — output visible
Output: 'Week 2 complete'
5
README.md with command reference

Open the uploaded README and verify the command count and description quality.

🔍 What the grader checks
At least 10 commands listed (count them)
Each command has a description — not just the command name alone
Descriptions are the student's own words, not copy-pasted from the lecture
⚠ Common mistakes at this step
README has commands listed but no descriptions — partial credit only
Fewer than 10 commands — award partial (4/5 for 8+, 3/5 for 6+, 0 for under 6)
README describes zsh in general terms without listing individual commands
Known Common Mistakes
⚠ Known common mistakes (from the assignment page)
Showing bash ($) instead of zsh (%) in screenshots
Creating files by clicking in VS Code instead of using touch from the terminal
Missing the pwd + ls before rm — this is the entire point of Requirement 3
README with command names but no descriptions — one sentence per command is required

Grading Flow
Check that all three screenshots are present. Note which requirements each screenshot covers.
Screenshot 1 (navigation): Count the commands. Verify zsh prompt. Check output is visible.
Screenshot 2 (file ops): Identify each of the five operations — cp, mv, cat, head, tail. Mark each found/missing.
Screenshot 3 (rm ritual): Verify exact order: pwd → ls → rm. This is pass/fail for the top point tier.

💡
Additional grading notes
Students who missed the rm safety ritual should get pointed feedback about why it matters — this habit prevents data loss in real work. Encourage them to review the lab walkthrough before Assignment 3.
💬
Rubric discussion
The rm safety ritual (10 pts) is the highest-stakes conceptual requirement. It is binary in one sense: either the student demonstrated the ritual or they did not. However, award 7/10 if all three commands are present but not in the ideal sequence. Award 5/10 if only pwd and rm are shown without ls.
