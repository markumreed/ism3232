SLO-1 To develop the skills needed to develop software.
Unit 1 Developer Foundations Weeks 1–4

Unit Objective:  Build and maintain a complete professional developer environment — shell, editor, virtual environments, linter, test runner, and version control — and execute the nine-step submission ritual without reference.

Week Topic Weekly Objective — By the end of this week you will be able to:
W1 Developer Mindset & Setup Set up VS Code, verify Python/zsh/Git installation, create the course folder structure from the terminal, and run the first Python script — completing the full edit→save→run loop for the first time.
W2 zsh Navigation & File Operations Navigate the file system without a GUI using pwd, ls, cd, mkdir, touch, tree, cp, mv, and rm; execute the three-step rm safety ritual from memory.
W3 Virtual Environments & .zshrc Create, activate, and verify a virtual environment; install packages and freeze requirements; configure ten required shell aliases and the mkcd function in .zshrc.
W4 Search Tools, Ritual & Git Search codebases with ripgrep and find; execute all nine steps of the pre-submission ritual in order; initialise a Git repository, make meaningful commits, and push to GitHub.
SLO-2 To understand necessary techniques for effective software development.
Unit 2 Python Foundations Weeks 5–8

Unit Objective:  Write correct, tested Python programs using all four core data types, control flow, functions with type hints, modules, and pytest — diagnosing errors from tracebacks without assistance.

Week Topic Weekly Objective — By the end of this week you will be able to:
W5 Variables, Data Types & Operators Declare variables of all four types; apply arithmetic, comparison, and logical operators; read user input with input() and wrap it correctly; format business output with f-string specifiers.
W6 Conditionals, Loops & Dictionaries Implement multi-tier business rules with if/elif/else; write for loops with the accumulator pattern initialised outside the loop; model business records as a list of dicts and perform CRUD operations.
W7 Functions, Modules & pytest Write functions with type hints, docstrings, and return values (never print); understand local scope; organise code into separate modules; write all six required pytest test types including boundary tests.
W8 Debugging & AI Literacy Read any Python traceback bottom-up and identify error type, file, and line; debug with strategic print() statements; complete the six-step Debug-First workflow; write the AI literacy reflection.
Unit 3 Object-Oriented Design Weeks 10–12

Unit Objective:  Design and implement a two-class OOP system (entity + manager) using the design-first approach — writing design.md before any code, with all six required pytest test types passing.

Week Topic Weekly Objective — By the end of this week you will be able to:
W10 OOP I — Classes & Objects Explain the class/instance mental model; write a complete entity class with __init__, default status attribute, at least three methods returning values, and __repr__; demonstrate instance independence.
W11 OOP II — Composition & Inheritance Implement a manager class using composition (instance-level list in __init__); apply light inheritance with super().__init__(); demonstrate the OOP→SQL mapping (class→table, instance→row, attribute→column).
W12 OOP III — Applied Practice Write design.md before any class code and commit it first; implement two entity classes and one manager; pass all six required test types including the instance independence test; identify all four common OOP mistakes.

Unit 4 Capstone Build Weeks 13–16

Unit Objective:  Design, build, test, and publicly demonstrate a complete deployable business application — incorporating SQL persistence, a Streamlit UI, and a responsible GenAI feature with all six required controls.

Week Topic Weekly Objective — By the end of this week you will be able to:
W13 Capstone Design & SQL Foundations Complete a ten-field PROPOSAL.md with specific business rules (threshold + consequence); write all five required SQL operations in the sqlite3 shell; design a schema with correct column types; commit schema.sql.
W14 Python + SQL Integration Implement all five database functions using the context manager pattern and ? placeholders exclusively; apply row_factory for dict-based results; write all database tests using tmp_path for isolation.
W15 Streamlit Interface Explain Streamlit's rerun execution model; implement all five required tabs (Submit, View All, Filter, Update Status, Report) wired to the live database; call create_table() before any widget.
W16 GenAI Feature & Final Demo Implement all six GenAI controls (disclosure, review gate, no PII, mocked tests, specific limitations, can explain); deliver the eight required presentation topics with a live working demo.
