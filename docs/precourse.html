Tool 1 — VS Code
💻
Visual Studio Code
Mac + Windows

Your primary code editor for every script, project, and capstone file in this course.

🍎 Mac / Linux
🪟 Windows
1
Download VS Code

Go to code.visualstudio.com and click the big blue Download for Mac button. This downloads a .zip file.

2
Install

Open your Downloads folder. Double-click the .zip to unzip it. Drag Visual Studio Code.app into your Applications folder. Open it from Applications or Spotlight (⌘Space → type “code”).

3
Install the 'code' command

Open VS Code. Press ⌘Shift+P to open the Command Palette. Type Shell Command: Install 'code' command in PATH and press Enter. This lets you open VS Code from the terminal by typing code .

1
Download VS Code

Go to code.visualstudio.com and click Download for Windows. This downloads a .exe installer.

2
Install

Run the installer. Accept the license agreement. On the Select Additional Tasks screen, check both Add to PATH boxes. Click through to Install. This is important — without adding to PATH you cannot use code from the terminal.

Restart and verify

After installation completes, restart your computer. Open the Start menu and search for Visual Studio Code to confirm it installed.

With VS Code open, click the Extensions icon in the left sidebar (four squares) or press ⌘Shift+X (Mac) / Ctrl+Shift+X (Windows).

Tool 2 — WSL + Ubuntu (Windows Only)
🪟
WSL — Windows Subsystem for Linux
Windows Only

Gives Windows a real Linux environment with zsh. Required for all terminal work in this course. Mac users skip this section.

💡
What WSL is
WSL lets your Windows machine run a full Ubuntu Linux terminal alongside Windows. Every terminal command in this course runs inside this Ubuntu environment. It is not a separate computer — it shares your files and runs natively on Windows 10/11.
🍎 Mac / Linux
🪟 Windows
Mac users — skip this step

zsh is the default shell on macOS Catalina and later. You do not need WSL. Proceed to Tool 3.

1
Check Windows version

WSL requires Windows 10 version 2004 or higher, or Windows 11. To check: press Win+R, type winver, press Enter. If your build number is below 19041, run Windows Update first.

2
Open PowerShell as Administrator

Click Start. Search for PowerShell. Right-click it and choose Run as administrator. Click Yes on the UAC prompt.

3
Run the WSL install command

In the Administrator PowerShell window, paste this command and press Enter:

wsl --install

This installs WSL 2 and Ubuntu automatically. It will take 5–15 minutes depending on your internet speed. You will see a progress bar.

4
Restart your computer

When the installation finishes you will be prompted to restart. Do it. WSL will not work until after the restart.

5
Complete the Ubuntu setup

After restarting, Ubuntu will open automatically and ask you to create a UNIX username and password. Choose a simple username (no spaces, all lowercase). This is separate from your Windows login. The password will not show characters as you type — that is normal.

6
Install zsh inside Ubuntu

In the Ubuntu terminal, run these commands one at a time:

sudo apt update
sudo apt install zsh -y
chsh -s $(which zsh)

When asked for a password, enter the Ubuntu password you created in Step 5. After running chsh, close the Ubuntu window and reopen it — you should now be in zsh (the prompt changes).

✓ Verify
$ echo $SHELL
Expected: Should return /usr/bin/zsh or /bin/zsh
⚠
Windows — if wsl --install fails
Some Windows machines require virtualization to be enabled in BIOS/UEFI before WSL can install. If you get an error about virtualization: restart, enter BIOS setup (usually Del or F2 during startup), find “Virtualization Technology” or “Intel VT-x” and enable it. Save and restart. Then try wsl --install again.
Tool 3 — Python 3
🐍
Python 3
Mac + Windows

The programming language for this course. You need Python 3.10 or higher.

🍎 Mac / Linux
🪟 Windows
1
Check if Python 3 is already installed

Open Terminal (or the VS Code integrated terminal). Run:

python3 --version

If you see Python 3.10.x or higher, you are done. Skip to the verification box below.

2
If Python 3 is missing or too old — install Xcode Command Line Tools

If the Xcode method does not work, go to python.org/downloads and download the latest Python 3 installer for macOS. Run the .pkg installer.

$ python3 --version
Expected: Python 3.10.x or Python 3.11.x or higher
1
Python is installed inside WSL Ubuntu

If you completed the WSL setup in Tool 2, Ubuntu comes with Python 3. Open your Ubuntu/WSL terminal and run:

python3 --version

If you see Python 3.10 or higher, you are good. If the version is older:

2
Update Python in Ubuntu if needed
sudo apt update
sudo apt install python3 python3-pip -y
python3 --version

This updates Python to the version available in Ubuntu’s package manager.

✓ Verify
$ python3 --version

Tool 5 — zsh (and setting up the VS Code terminal)
🖥
zsh
Mac + Windows (via WSL)

The shell (command-line environment) used in every class session. Every terminal command in this course runs in zsh.

🍎 Mac / Linux
🪟 Windows
1
Verify zsh is your default shell

Open Terminal and run:

echo $SHELL

Expected output: /bin/zsh. macOS Catalina (2019) and later use zsh by default.

2
If you see /bin/bash instead

Run this to switch:

chsh -s /bin/zsh

Enter your password. Close the terminal and reopen it. Run echo $SHELL again to confirm.

3
Configure the VS Code integrated terminal to use zsh

Open VS Code. Open Settings: ⌘,

Search for: terminal default profile

Click Terminal > Integrated > Default Profile: Osx and set it to zsh.

Close any open terminal panels in VS Code. Open a new one with ⌘`. The prompt should show %.

✓ Verify
$ echo $SHELL
Expected: Should show /bin/zsh
1
zsh was set up in WSL during Tool 2

If you completed the WSL setup, zsh is already your shell inside Ubuntu. Open the Ubuntu terminal and confirm:

echo $SHELL

Expected: /usr/bin/zsh or /bin/zsh

2
Configure the VS Code integrated terminal to use WSL

Open VS Code. Press Ctrl+Shift+P to open the Command Palette.

Type: Terminal: Select Default Profile and press Enter.

From the list, choose Ubuntu (WSL) or wsl.exe.

Open a new terminal in VS Code with Ctrl+`. It should open inside Ubuntu with a zsh prompt.

✓ Verify
$ echo $SHELL
Expected: Should show /usr/bin/zsh or /bin/zsh

Troubleshooting — Common Problems and Fixes
⚠ Problem
I see $ in my terminal prompt instead of %
✓ Fix
You are running bash, not zsh. On Mac: run chsh -s /bin/zsh and restart the terminal. On Windows: make sure VS Code is set to use the WSL (Ubuntu) profile, not PowerShell.
⚠ Problem
python3: command not found
✓ Fix
On Mac: run xcode-select --install to get Python 3. On Windows/WSL: run sudo apt install python3 -y inside the Ubuntu terminal.
⚠ Problem
git: command not found
✓ Fix
On Mac: run xcode-select --install. On Windows/WSL: run sudo apt install git -y. Then configure your identity with git config --global user.name.
⚠ Problem
code: command not found when typing in terminal
✓ Fix
In VS Code, press ⌘Shift+P (Mac) or Ctrl+Shift+P (Windows). Type Shell Command: Install 'code' command in PATH. Press Enter. Restart the terminal.
⚠ Problem
WSL installation fails with a virtualisation error
✓ Fix
Enter BIOS/UEFI at startup (usually Del, F2, or F10). Find Intel VT-x or AMD-V or Virtualization Technology and enable it. Save and restart. Run wsl --install again.
⚠ Problem
VS Code terminal opens PowerShell instead of WSL
✓ Fix
Press Ctrl+Shift+P in VS Code. Type Terminal: Select Default Profile. Choose Ubuntu (WSL). Open a new terminal panel. It should now open in Ubuntu/zsh.
⚠ Problem
brew: command not found after installing Homebrew
✓ Fix
After Homebrew installs, it prints one or two commands to add it to your PATH. Run those commands exactly. Then close and reopen the terminal.
⚠ Problem
chsh: PAM authentication error when switching to zsh on WSL
✓ Fix
This happens when WSL is not set up correctly. Try: sudo chsh -s $(which zsh) $USER instead. If that fails, reinstall WSL by running wsl --unregister Ubuntu and then wsl --install again.
⚠ Problem
Python version is 2.7 or older
✓ Fix
Run python3 --version not python --version. On modern systems, python may still point to Python 2. This course always uses python3.
⚠ Problem
I cannot create a GitHub repository
✓ Fix
Your email must be verified before you can create repositories. Check your inbox for the verification email from GitHub. If you cannot find it, go to github.com/settings/emails and resend.
Getting Help Before Day 1
💬
If something does not work
Post in the course discussion board with: (1) your operating system and version, (2) which step you are on, (3) a screenshot of the error message. The instructor monitors the board before the semester starts. Do not wait until Day 1 to report a setup problem. Setup issues that are not fixed before Week 1 will cause you to fall behind immediately.
What to include in a help post

• Operating system: e.g., macOS 14.2 Sonoma, Windows 11 Home 23H2

• Step: e.g., “Tool 2, Step 3 — installing zsh inside Ubuntu”

• What you ran: the exact command you typed

• What happened: the exact error message (screenshot preferred)

• What you already tried: so we do not repeat suggestions
