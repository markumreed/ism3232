Section 1 — Why Streamlit?
Streamlit turns Python scripts into browser-based business tools

Streamlit lets you build an interactive web interface for your Python application without writing HTML, CSS, or JavaScript. Every widget — text inputs, buttons, dropdowns, tables — is a Python function call. The interface re-runs the entire script top to bottom whenever the user interacts with it.

For a business application in this course, Streamlit is the front-end layer that connects to database.py and models.py. Users add records, view records, update statuses, and see reports — all through Streamlit widgets backed by SQLite.

Install and run Streamlit
# Activate venv first
source .venv/bin/activate
pip install streamlit
pip freeze > requirements.txt

# Run your app
streamlit run app.py
# Opens in browser at http://localhost:8501
# Ctrl+C to stop
Section 2 — Core Streamlit Components
Essential Streamlit widgets and layout — try each one
import streamlit as st

# Page setup (call once, at the top)
st.set_page_config(page_title="Request Tracker", layout="wide")

# Text display
st.title("Business Request Tracker")
st.header("Submit a New Request")
st.write("Any text, markdown, or value — very flexible")

# Input widgets — each returns a Python value
requester = st.text_input("Your name")
category  = st.selectbox("Category", ["Travel", "Equipment", "Software"])
amount    = st.number_input("Amount ($)", min_value=0.0, step=10.0)
notes     = st.text_area("Notes (optional)")

# Button — returns True only on the click that triggered the rerun
if st.button("Submit Request"):
    if requester and amount > 0:
        add_record(requester, category, amount, notes)
        st.success(f"Request submitted for {requester}")
    else:
        st.error("Name and amount are required.")

# Display a table from a list of dicts
records = get_all_records()
if records:
    st.dataframe(records, use_container_width=True)
else:
    st.info("No records yet.")

Putting it together — a minimal working app.py skeleton
import streamlit as st
from database import create_table, add_record, get_all_records, update_status, get_status_report

create_table()   # idempotent — safe to call on every run

st.set_page_config(page_title="Request Tracker", layout="wide")
st.title("📋 Business Request Tracker")

tab1, tab2, tab3 = st.tabs(["Submit", "View & Filter", "Report"])

with tab1:
    st.header("Submit a New Request")
    requester = st.text_input("Your name")
    category  = st.selectbox("Category", ["Travel", "Equipment", "Software"])
    amount    = st.number_input("Amount ($)", min_value=0.0, step=10.0)
    if st.button("Submit"):
        add_record(requester, category, amount)
        st.success("Submitted!")

with tab2:
    records = get_all_records()
    st.dataframe(records)

with tab3:
    report = get_status_report()
    st.dataframe(report)

FastAPI is a concept introduction — no deliverable required
This 60–90 minute lecture covers what an API is, how routes and requests work, and how FastAPI structures a Python web service. Students run one minimal demo route. There is no FastAPI assignment or capstone requirement.
Minimal FastAPI demo — run with: uvicorn main:app --reload
from fastapi import FastAPI

app = FastAPI()

def home():
    return {"message": "Business API is running"}

@app.get("/requests")
def get_requests():
    return {"requests": []}
