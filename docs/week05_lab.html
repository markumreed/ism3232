⚠
The most common Week 5 bug
input() always returns a string. If you forget to convert it with int() or float() and then try arithmetic, you get a TypeError. Show yourself by trying unit_price * input('qty: ') without conversion.

Part 4 — pytest

tests/test_week5.py
def test_tax_at_seven_percent():
    subtotal = 200.00
    tax = subtotal * 0.07
    assert tax == 14.0

def test_over_limit_true():
    assert 1500 > 1000 is True

def test_over_limit_false():
    assert 500 > 1000 is False

def test_type_of_string():
    name = 'ISM3232'
    assert type(name) == str

def test_type_conversion():
    s = '42'
    assert int(s) == 42
Run: pytest -v — all five must pass.
