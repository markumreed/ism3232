SQL fundamentals — five required operations

Run every query in the sqlite3 shell — show output each time
-- Open the shell
sqlite3 data/requests.db

-- CREATE
CREATE TABLE IF NOT EXISTS requests (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    requester TEXT    NOT NULL,
    category  TEXT    NOT NULL,
    amount    REAL,
    status    TEXT    DEFAULT 'Pending'
);

-- INSERT
INSERT INTO requests (requester, category, amount)
    VALUES ('Taylor', 'Equipment', 1200.00);

-- SELECT
SELECT * FROM requests;
SELECT * FROM requests WHERE status = 'Pending';
SELECT * FROM requests ORDER BY amount DESC;

-- UPDATE (WHERE is required)
UPDATE requests SET status = 'Approved' WHERE id = 1;

-- GROUP BY
SELECT status, COUNT(*) AS count, SUM(amount) AS total
FROM requests
GROUP BY status;

.quit
⚠ UPDATE without WHERE is catastrophic

UPDATE requests SET status = 'Approved' — without WHERE — changes every single row. Always test your UPDATE with a SELECT first to confirm which rows will be affected.

sqlite3 shell commands for debugging
.tables          -- list all tables
.schema requests -- show CREATE TABLE statement
.mode column     -- aligned output
.headers on      -- show column names
.quit            -- exit the shell

Schema design — choosing the right types

TEXTNames, categories, statuses. Any string value.
INTEGERIDs, counts, whole numbers.
REALPrices, rates, decimal numbers. Use for all money values.
NOT NULLThis column is required — INSERT fails if omitted.
DEFAULT 'Pending'Automatically used if no value is provided on INSERT.
PRIMARY KEY AUTOINCREMENTDatabase assigns a unique integer ID automatically. Never provide this yourself.

Module Quiz

Test your understanding before moving on. Click each answer to reveal it.

Q1 · Multiple Choice
In the sqlite3 shell, which command lists all tables in the database?
  • a) SHOW TABLES;
  • b) LIST TABLES;
  • c) .tables
  • d) SELECT * FROM tables;
Answer
sqlite3 dot commands start with . and are not SQL. .tables lists tables; .schema tablename shows the CREATE TABLE statement.
Q2 · Multiple Choice
What does PRIMARY KEY AUTOINCREMENT do on a column?
  • a) Requires a value to be manually supplied
  • b) Automatically assigns a unique incrementing integer id to each new row
  • c) Prevents duplicate rows across all columns
  • d) Creates an index on the column
Answer
You never insert the id yourself; SQLite assigns the next available integer automatically.
Q3 · Code Trace
What does this SQL statement return?
SELECT requester, amount
FROM requests
WHERE status = 'Pending'
ORDER BY amount DESC;
Answer
The requester name and amount for all rows where status is 'Pending', sorted with the highest amounts first (descending).
Q4 · Multiple Choice
Which SQL data type stores decimal numbers like prices?
  • a) INTEGER
  • b) TEXT
  • c) REAL
  • d) BLOB
Answer
REAL stores floating-point numbers (e.g., 1250.00). INTEGER stores whole numbers; TEXT stores strings; BLOB stores binary data.
Q5 · Multiple Choice
What does NOT NULL mean in a column definition?
  • a) The column stores only zero values
  • b) The column cannot be left empty — a value must be provided
  • c) The column stores text, not numbers
  • d) The column is a foreign key
Answer
An INSERT without a value for a NOT NULL column raises an error. Omitting it allows NULL (missing) values, which can cause unexpected bugs.
Q6 · Multiple Choice
How many required fields does the ISM3232 project proposal have?
  • a) 5
  • b) 8
  • c) 10
  • d) 12
Answer
All 10 fields are required: project name, business problem, primary user, records stored, key operations, Python entities, SQL tables, success criterion, stretch goal, and tech stack confirmation.
Q7 · True / False
In SQL, an UPDATE statement without a WHERE clause modifies every row in the table.
Answer
True — UPDATE requests SET status = 'Approved'; changes every row. Always add WHERE id = ? or another specific condition to target only the intended row.
Q8 · True / False
The sqlite3 shell .mode column and .headers on settings are saved permanently to the database.
Answer
False — These dot commands configure the current session's display only. They are not stored. Add them to ~/.sqliterc to make them permanent.
Q9 · Short Answer
What is the difference between CREATE TABLE and CREATE TABLE IF NOT EXISTS?
Answer
CREATE TABLE raises an error if the table already exists. CREATE TABLE IF NOT EXISTS checks first and silently skips creation if the table is there. Always use IF NOT EXISTS in create_table() functions so they are safe to call multiple times, including in tests.
