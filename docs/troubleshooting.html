Troubleshooting Guide

The most common errors in ISM3232, organised by type. Read the traceback from the bottom up — the last line is the most specific. Find your error type below, read the cause, apply the fix.

The one rule for reading any error

Read the traceback from the bottom up. The bottom line gives you the error type and the exact message. The line above gives you the file name and line number. Start there. Do not read from the top.

Environment & Setup

ModuleNotFoundError First seen: Week 1
ModuleNotFoundError: No module named 'anthropic'

Cause: You ran a script without the venv active, or a package is not installed.

Fix: Run source .venv/bin/activate — the prompt must show (.venv) — then pip install -r requirements.txt.

ruff: command not found First seen: Week 4
zsh: command not found: ruff

Cause: ruff is not installed in the active environment.

Fix: source .venv/bin/activate then pip install ruff.

pytest not found First seen: Week 7
command not found: pytest OR No module named pytest

Cause: pytest is not installed in the active environment, or the venv is not activated.

Fix: Activate the venv first: source .venv/bin/activate. Then run pip install pytest if it is still missing.

Permission denied (.venv) First seen: Week 3
Permission denied: .venv/bin/activate

Cause: The venv activation script is not executable, or you are on a system with restricted execution policy (Windows).

Fix: On Mac/Linux: chmod +x .venv/bin/activate. On Windows WSL: ensure you are running in WSL, not PowerShell.

FileNotFoundError First seen: Week 1
FileNotFoundError: [Errno 2] No such file or directory: 'data/requests.db'

Cause: Python could not find a file at the path you specified. Either the file does not exist, or your current working directory is wrong.

Fix: Run pwd to confirm your location. Run ls to confirm the file exists. Use absolute paths or ensure you run the script from the correct directory.

Python Syntax & Logic

SyntaxError First seen: Week 5
SyntaxError: invalid syntax

Cause: Python could not parse the file. Missing colon after def or if, unclosed parenthesis or bracket, mismatched quotes.

Fix: Read the line number in the traceback. Check for a missing :, an unclosed ( or [, or a mismatched quote. The error is often on the line before the reported line.

IndentationError First seen: Week 5
IndentationError: unexpected indent

Cause: Python uses indentation to define code blocks. A line is indented inconsistently — mixing tabs and spaces, or wrong number of spaces.

Fix: Use 4 spaces per indent level. Never mix tabs and spaces. VS Code shows these as red marks — enable "Render Whitespace" in the View menu.

TypeError First seen: Week 5
TypeError: can only concatenate str (not 'int') to str

Cause: An operation was applied to a value of the wrong type. Most common: adding input() output to a number.

Fix: Wrap input() immediately: float(input(...)) or int(input(...)).

NameError First seen: Week 7
NameError: name 'result' is not defined

Cause: A variable was used before it was defined, or it was defined inside a function (local scope) and accessed outside.

Fix: Check the variable is defined before the line that uses it. If it is inside a function, use return to pass the value out rather than accessing it directly.

KeyError First seen: Week 6
KeyError: 'amout' ← typo, should be 'amount'

Cause: You accessed a dictionary key that does not exist — usually a typo in the key name.

Fix: Check the spelling of the key. Use print(record.keys()) to see available keys. Consider .get('key', default) for optional keys.

AttributeError First seen: Week 10
AttributeError: 'NoneType' object has no attribute 'status'

Cause: A method or attribute was called on an object that does not have it. Common in OOP: calling a method with the wrong name, or on the wrong type.

Fix: Check the spelling of the method name. Use dir(obj) to see what an object has. In OOP, make sure you are calling the method on an instance, not the class.

Capstone-Specific

altair TypedDict error (Python 3.14) First seen: Week 15
TypeError: _TypedDictMeta.__new__() got an unexpected keyword argument 'closed'

Cause: Streamlit 1.41 ships with altair 5.x which is incompatible with Python 3.14.

Fix: Delete .venv/, update requirements.txt to use streamlit==1.57.0 and altair>=6.0.0, then recreate the venv and reinstall.

API key errors (Anthropic) First seen: Week 16
anthropic.AuthenticationError: Error code: 401

Cause: The ANTHROPIC_API_KEY environment variable is missing, empty, or not loaded from .env.

Fix: Create a .env file in the project root with ANTHROPIC_API_KEY=your-key-here. Confirm python-dotenv is installed and load_dotenv() is called at the top of ai_feature.py.

Still stuck?

After trying the fix above, if the error persists:

  1. Copy the full traceback (all lines from "Traceback" to the last line)
  2. Write one sentence describing what you were trying to do when the error appeared
  3. Post both to the course discussion board

Do not post just the last line of the error. The full traceback is what allows a diagnosis.
