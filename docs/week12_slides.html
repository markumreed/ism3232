Design-First Approach

15–40 min  Build own multi-class system

🧩 Always design before you type

Before writing any code this week, students should write down: What are my two entity classes? What attributes does each one have? What methods does each one need? What does the manager class hold and what can it do? A class design written in comments or a README takes five…
“Five tests minimum. At least one must test a boundary or edge case — where the business rule changes. If your approval threshold is $1,000, test with exactly $1,000.”

from models import Employee, Department

def test_give_raise_increases_salary():
    e = Employee(1, "Alex", "Eng", 70000)
    e.give_raise(10)
    assert e.salary == 77000

def test_deactivate_removes_from_headcount():
    dept = Department("Eng", 300000)
    e = Employee(2, "Sam", "Eng", 60000)
    dept.hire(e)
    e.deactivate()
    assert dept.headcount() == 0

def test_payroll_excludes_inactive():
    dept = Department("Eng", 300000)

    e2.deactivate()

MistakeWhat it looks likeFix
