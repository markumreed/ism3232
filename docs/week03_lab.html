Mental model — the private room
A venv is a private room for each project. When you activate it you walk in — every package installs on that room’s shelves. When you deactivate you walk out. Other projects are completely unaffected.
Part 1 — Virtual Environment Workflow
1
Create and activate the venv
cd ~/ism3232/module02_zsh
python3 -m venv .venv
source .venv/bin/activate
# Prompt should now show (.venv)

which python3   # should point inside .venv/
pip list        # should show minimal packages
📸
Screenshot showing the (.venv) prefix in the prompt and the which python3 output.
2
Install packages and freeze
pip install pytest ruff
pip list                       # confirm both are installed
pip freeze > requirements.txt
cat requirements.txt           # confirm it was written
📸
Screenshot showing pip install output and cat requirements.txt.
3
Verify the environment is isolated

Open a second terminal tab (without activating the venv) and run:

In the non-venv terminal
which python3     # points to system Python
pip list          # does NOT include pytest and ruff

This confirms the venv is isolated. Switch back to the venv terminal.

Part 2 — .gitignore
4
Create .gitignore before anything is committed

The .venv/ folder must never be committed to Git. Create the file now:

cd ~/ism3232/module02_zsh
echo '.venv/' > .gitignore
echo '__pycache__/' >> .gitignore
echo '*.pyc' >> .gitignore
cat .gitignore          # confirm all three lines
ls -la                  # .gitignore and .venv should both appear
📸
Screenshot of ls -la showing both .venv/ and .gitignore.
Part 3 — .zshrc Aliases and Functions
5
Add required aliases to .zshrc
Open the file
code ~/.zshrc

Add the following lines to the file — do not delete anything that is already there:

Paste into ~/.zshrc
# ISM3232 required aliases
alias ll='ls -la'
alias c='clear'
alias py='python3'
alias gs='git status'
alias ga='git add .'
alias gcmsg='git commit -m'
alias gp='git push'
alias gl='git log --oneline'
alias tree2='tree -L 2'

# Required shell function
mkcd() { mkdir -p "$1" && cd "$1" }

Save the file, then apply the changes:

source ~/.zshrc

# Test each alias
ll                   # should work as ls -la
gs                   # should run git status
mkcd testdir         # should create and enter testdir/
pwd                  # confirm you are inside testdir/
📸
Screenshot showing ll, gs, and mkcd all working after source ~/.zshrc.
