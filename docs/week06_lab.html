Part 2 — Two Business Rules
2
Apply two distinct business rules inside the loop

Replace the simple loop with one that applies two rules. Define thresholds that make sense for your domain:

week6_lab.py — business rules
LIMIT = 1000
HIGH  = 2000
total = 0
flagged = []
high_value = []

for rec in records:
    if rec['status'] == 'Pending':
        total += rec['amount']
        if rec['amount'] > LIMIT:
            flagged.append(rec)
        if rec['amount'] > HIGH:
            high_value.append(rec)

Run and confirm the counters are correct.

Part 3 — Formatted Summary and File Output
3
Print a summary and save it to data/week6_summary.txt
week6_lab.py — output
import os
os.makedirs('data', exist_ok=True)

lines = [
    f'Pending total:   ${total:,.2f}',
    f'Needs review:    {len(flagged)}',
    f'High-value:      {len(high_value)}',
]
for line in lines:
    print(line)

with open('data/week6_summary.txt', 'w') as f:
    f.write('\n'.join(lines) + '\n')

print('\nRecords needing review:')
for r in flagged:
    print(f"  ID {r['id']}: {r['name']} - ${r['amount']:,.2f}")

Part 4 — pytest
4
Write five pytest tests
tests/test_week6.py
def test_accumulator():
    amounts = [500, 1500, 200]
    total = sum(a for a in amounts)
    assert total == 2200

def test_flag_filter():
    records = [{'amount': 500}, {'amount': 1500}, {'amount': 800}]
    flagged = [r for r in records if r['amount'] > 1000]
    assert len(flagged) == 1

def test_status_filter():
    records = [{'amount': 500, 'status': 'Pending'}, {'amount': 1000, 'status': 'Approved'}]
    pending = [r for r in records if r['status'] == 'Pending']
    assert len(pending) == 1

def test_dict_key_access():
    rec = {'id': 1, 'amount': 750, 'status': 'Pending'}
    assert rec['amount'] == 750

def test_list_of_dicts_length():
    data = [{'x': 1}, {'x': 2}, {'x': 3}]
    assert len(data) == 3

Run: pytest -v — all five must pass.
