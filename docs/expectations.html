What 'full workflow required' means

The submission ritual is graded

From Week 2 onwards, every assignment requires the complete nine-step ritual before pushing. The instructor can see your Git log. A single commit called "final" pushed at midnight tells a story. A commit history showing ruff and pytest run at each step tells a different one.

The nine steps — in order, every time
  1. pwd — confirm your location
  2. tree -L 3 — confirm the project structure
  3. source .venv/bin/activate — prompt must show (.venv)
  4. ruff format . — auto-format all Python files
  5. ruff check . — must be clean before continuing
  6. pytest — must be green before continuing
  7. git status — see what changed
  8. git add . — stage everything
  9. git commit -m "descriptive message" then git push
⚠ If ruff or pytest fail — stop

Fix errors before continuing the ritual. Never push broken code. The ritual is not complete until steps 5 and 6 both pass cleanly.

AI use policy

Permitted: Ask AI to explain what a traceback error message means. Ask AI to explain what a piece of code does in plain English. Ask AI to suggest test cases for a function you already wrote.

Prohibited: Generating any submitted code. Fixing bugs for you (you must attempt this yourself first). Any AI use during the midterm exam.

✓ The Debug First, Then Ask workflow

For Assignment 8, there is a required six-step AI literacy activity. Read the traceback → write your hypothesis → try print() debugging → if stuck, ask AI to explain the error only → fix it yourself → write the reflection. The reflection is 30 of 70 points and is not optional.

How to use this site

This site has three types of pages for each week:

  • Reading — read this before class. It explains the why behind each topic. Typically 4–5 minutes.
  • Lecture — the in-class page. Same material as the reading, formatted for following along during the live session, with timing guides and annotated code blocks.
  • Lab — the in-class practice activity. Follow along during the lab session.
  • Slides — the Reveal.js slide deck shown during the lecture. Press F for fullscreen, S for speaker notes, Escape to exit.
