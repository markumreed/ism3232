Search Tools

0–15 min  Search tools — rg, find, grep

🧩 Why developers need codebase search

As projects grow, you need to find where a function is defined, where a variable is used, or which file contains a specific error. Clicking through VS Code is slow. Knowing rg means answering these questions in seconds from anywhere. Required search commands — ripgrep, find,…
“Git tracks every change to your code. GitHub is where your repositories live and how you submit every major assignment. I see your commit history. A single commit called 'final' tells me nothing about your process. Meaningful commits at logical checkpoints tell me you understand your own work.”

🖥 Terminal

# ripgrep (rg) — fast, respects .gitignore
rg 'def '              # find every function definition in the project
rg 'class '            # find every class definition
rg 'TODO'              # find all TODO comments
rg -l 'import'         # list only filenames containing a match
rg 'approve' --type py # search only Python files

# find — locate files by name or type
find . -name '*.py'    # all Python files in current tree
find . -name '*.txt'   # all text files
find . -type d         # all directories

# grep — universal, slower than rg
grep -R 'TODO' .       # recursive search (rg is preferred)

# tree — visual structure
tree -L 3              # directory tree 3 levels deep

The Professional Pre-Submission Ritual

30–50 min  Pre-submission ritual

🧩 zoxide — smarter cd

Git tracks every change to your code — what changed, when, and why. GitHub is where your repositories live and how you submit every major assignment in this course. Instructors see your commit history. A single commit called "final" tells us nothing about your process. Git…

🖥 Terminal

# One-time identity setup (if not done in Week 1)
git config --global user.name 'Your Name'
git config --global user.email 'you@email.com'

# Initialize a repository in the current folder
git init
git status                   # see all untracked files

# Create .gitignore before first commit
echo '.venv/' > .gitignore
echo '__pycache__/' >> .gitignore

# Stage and make first commit
git add .
git commit -m 'initial commit: week 4 zsh and workflow module'

# Connect to GitHub repository (create repo on github.com first)
git remote add origin https://github.com/YOURUSERNAME/ism3232-module02.git
git branch -M main
git push -u origin main

# Verify success
git log --oneline    # see commit history
git status           # should say nothing to commit
