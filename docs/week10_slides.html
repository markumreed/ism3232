What Is a Class? What Is an Object?

0–15 min  What is a class? Mental model

🧩 The core mental model

A class is a blueprint. An object is a specific thing built from that blueprint. A BusinessRequest class describes what every request has (attributes) and what every request can do (methods). Each individual request — request #101, request #102 — is a separate instance of that…
“The BusinessRequest class is the purchase request form template. req_101 is a filled-out form for Taylor's laptop request. req_102 is Jordan's equipment request. Same template, different data, completely independent.”

🐍 Code

class BusinessRequest:
    """Represents a single business purchase or travel request."""

    def __init__(self, request_id, requester, category, amount):
        self.request_id = request_id
        self.requester  = requester
        self.category   = category
        self.amount     = amount
        self.status     = "Pending"   # default — not passed in

    def approve(self):
        """Mark this request as approved."""
        self.status = "Approved"

    def reject(self):
        """Mark this request as rejected."""
        self.status = "Rejected"

    def requires_review(self):
        """Return True if the amount exceeds the approval limit."""
        return self.amount > 1000

    def __repr__(self):
        return f"BusinessRequest({self.request_id}, {self.requester}, ${self.amount}, {self.status})"

Writing the BusinessRequest Class

15–40 min  Write BusinessRequest live

🧩 Class = blueprint

Encapsulation means bundling the data (attributes) and the operations on that data (methods) into a single unit. The BusinessRequest class knows how to approve itself, reject itself, and determine whether it needs review. The caller does not need to know the internal logic — it…
“Encapsulation means bundling the data and the operations on that data into a single unit. The BusinessRequest class knows how to approve itself, reject itself, and determine whether it needs review. The caller does not need to know the internal logic — it just calls the method.”

🐍 Code

# Create two separate instances from the same class
req_101 = BusinessRequest(101, "Taylor", "Travel", 1200)
req_102 = BusinessRequest(102, "Jordan", "Equipment", 450)

# Access attributes
print(req_101.requester)          # Taylor
print(req_101.status)             # Pending

# Call methods
print(req_101.requires_review())  # True (1200 > 1000)
print(req_102.requires_review())  # False (450 <= 1000)

req_101.approve()
print(req_101.status)             # Approved
print(req_102.status)             # Pending — unchanged

# Instances are independent — changing one does not affect the other
print(req_101)   # BusinessRequest(101, Taylor, $1200, Approved)
print(req_102)   # BusinessRequest(102, Jordan, $450, Pending)
