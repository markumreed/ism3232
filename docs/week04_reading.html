Search tools — finding things fast

Three tools — run from ~/ism3232/
# ripgrep — search file CONTENTS (fast, respects .gitignore)
rg 'def '          # every function definition in the codebase
rg 'print'         # every print statement
rg 'TODO'          # every TODO comment
rg -l 'import'     # filenames only (not full matches)

# find — search by FILE NAME
find . -name '*.py'      # all Python files
find . -name '*.md'      # all README files

# tree — visualise structure
tree -L 3                # 3 levels deep
Why rg instead of grep

ripgrep automatically skips .venv/, .git/, and __pycache__/ because they are listed in .gitignore. grep searches everything including those thousands of files. For any real project, use rg.

The pre-submission ritual — memorise this

Every major push follows this nine-step sequence. Your instructor can see your Git log. A single commit called 'final' pushed at midnight tells a story. A commit history showing ruff and pytest run at each step tells a different one.

The complete ritual — run before every graded submission
# 1. Confirm location
pwd

# 2. Confirm structure
tree -L 3

# 3. Activate the venv (prompt must show (.venv))
source .venv/bin/activate

# 4. Auto-format
ruff format .

# 5. Check for issues (must be clean before continuing)
ruff check .

# 6. Run tests (must be green before continuing)
pytest

# 7. See what changed
git status

# 8. Stage
git add .

# 9. Commit with a descriptive message
git commit -m 'week 4: add search tools and complete ritual demo'

# Push
git push
⚠ If ruff or pytest fail — stop

Fix the errors before continuing. Never push broken code. The ritual is not complete until steps 5 and 6 both pass cleanly.

Git and GitHub

One-time identity setup — run once per machine
git config --global user.name 'Your Full Name'
git config --global user.email 'you@email.com'
git config --global --list   # verify both are set
First repository workflow — from local to GitHub
# Create .gitignore before anything else
echo '.venv/' > .gitignore
echo '__pycache__/' >> .gitignore

git init                   # initialise Git in this folder
git status                 # see all untracked files
git add .                  # stage everything
git commit -m 'initial commit: week 4 module'

# Connect to GitHub (create the repo on github.com first)
git remote add origin https://github.com/YOURUSERNAME/REPONAME.git
git branch -M main
git push -u origin main

# Verify
git log --oneline          # shows your commit
git status                 # shows 'nothing to commit'

Good commit messages describe what changed: 'add calculate_tax function to business_rules.py'. Bad messages: 'update', 'fix', 'final'. Write messages as if a colleague will read them tomorrow.

Key terms this week

ripgrep (rg)Fast content search tool that respects .gitignore.
git initInitialises a new Git repository in the current folder.
git add .Stages all changes for the next commit.
git commit -mCreates a snapshot of the staged changes with a message.
git pushSends commits to the remote repository on GitHub.
git log --onelineShows a compact history of all commits.
git statusShows which files have changed since the last commit.
ruff format .Auto-formats all Python files in the current folder.
ruff check .Reports style and logic issues without modifying files.
