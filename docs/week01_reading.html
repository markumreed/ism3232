Why this course starts with tools, not code

💻 Windows users — WSL required

All commands on this page use zsh on Mac/Linux. Windows users must run these inside WSL (Windows Subsystem for Linux), not PowerShell and not Command Prompt. Open VS Code, press Ctrl+`, and confirm the terminal shows a Linux prompt (ending in %). If you have not installed WSL yet, complete the pre-course setup tutorial before continuing.

Most programming courses start with print('Hello world'). This one starts with the terminal. That is intentional. The terminal is where professional developers live. Learning it first means every line of code you write from Week 2 onwards is written the right way — in the right folder, in the right environment, committed with a meaningful message.

By the end of this week you will have a working development environment, a course folder structure, and one working Python script. That is enough.

The directory tree — the most important mental model

Key idea

Your computer organises every file inside a tree of folders. You are always somewhere in that tree. Every command you run affects files relative to where you currently are.

The root of the tree is / on Mac/Linux. Your home folder is ~ (which expands to /Users/yourname). Your course folder will live at ~/ism3232/.

How to read a path
/Users/yourname/ism3232/module01_setup/hello_ism3232.py
│               │        │               └── the file
│               │        └── the module folder
│               └── your course folder
└── root of the filesystem

Absolute paths start from / and work from anywhere. Relative paths start from wherever you are right now. cd .. moves up one level. cd ~ jumps home from anywhere.

Your development environment

What you need installed

  • zsh — your shell. The prompt ends with %. If it ends with $ you are in bash — fix this before continuing.
  • VS Code — your code editor. Open the integrated terminal with Ctrl+` (Windows) or Cmd+` (Mac).
  • Python 3.10+ — verify with python3 --version. Note: use python3, not python.
  • Git 2.x — verify with git --version.
Six verification commands — run each one and read the output
echo $SHELL       # must show /bin/zsh
zsh --version     # must be 5.x or higher
python3 --version # must be 3.10 or higher
git --version     # must be 2.x
pwd               # shows your home directory
ls                # shows Desktop, Documents, etc.
⚠ Screenshot required

Take one screenshot showing all six commands and their outputs. This is Assignment 1, deliverable 1. All six outputs must be visible in a single window.

Creating the course folder structure

Run these commands in order — type each one, do not paste
cd ~                        # go home
mkdir ism3232               # create the course folder
cd ism3232                  # enter it
mkdir module01_setup        # create the first module folder
cd module01_setup           # enter it
pwd                         # confirm: ends in /ism3232/module01_setup

Your first Python script

Create and run hello_ism3232.py
touch hello_ism3232.py      # create empty file
code hello_ism3232.py       # open in VS Code
Type this — do not paste
# hello_ism3232.py
# Author: Your Name

print('Hello, ISM3232!')

course_name  = 'Business Application Development'
credit_hours = 3
weekly_hours = credit_hours * 2

print(f'Course: {course_name}')
print(f'Expected weekly hours: {weekly_hours}')
print('Environment verified. Week 1 complete.')
Save and run
python3 hello_ism3232.py

Key terms this week

shellThe program that reads your typed commands and executes them. We use zsh.
terminal / terminal emulatorThe window you type commands into. VS Code has one built in.
directoryA folder. Directories contain files and other directories.
absolute pathA path starting from root (/). Works from anywhere.
relative pathA path starting from your current location. Shorter, but location-dependent.
~ (tilde)Shorthand for your home directory. cd ~ takes you home from anywhere.
.. (dot dot)The parent directory. cd .. moves you up one level.
