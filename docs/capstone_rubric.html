5. GenAI Feature (ai_feature.py)15 points
PointsRequirement
15All six controls implemented and demonstrable. (1) Disclosure label visible in UI. (2) Second button required before saving. (3) No names, IDs, or amounts in the API prompt — only category + description. (4) All AI tests use unittest.mock.patch — pass without API key. (5) At least two specific limitations in README. (6) Student walks through every line during presentation.
✓ Full marks looks likeTwo buttons visible in UI. Prompt contains only category and description. pytest passes with no ANTHROPIC_API_KEY set. README limitations are specific ('may be inaccurate for technical equipment descriptions'), not generic ('AI can make mistakes').
⚠ Common issueAuto-saving AI output without a second button (Control 2). Sending requester name in prompt (Control 3). AI tests calling the real API (Control 4). Generic limitations ('AI is not perfect') in README (Control 5).
10Feature works but one or two controls missing. Most common: no second button (auto-save), or AI tests not mocked.
6AI function exists and calls the API but three or more controls absent.
0No AI feature, or all six controls absent, or tests call the real API.
