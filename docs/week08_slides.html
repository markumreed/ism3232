Debugging with print()

15–35 min  print() debugging

🧩 Strategic print() statements reveal the gap between expectation and reality

Debugging is finding where the program stops doing what you expect. print() lets you inspect values at specific execution points. Start from the error line and work backwards, printing variables until you find where the data became wrong. Debugging workflow — add prints, run,…

🐍 Code

def process_requests(requests):
    total = 0
    for req in requests:
        print(f"DEBUG: req = {req}")                        # see each record
        amount = req["amount"]
        print(f"DEBUG: amount = {amount} ({type(amount).__name__})") # check type
        total += amount
    print(f"DEBUG: total = {total}")                    # verify accumulation
    return total

03

Required AI Literacy Activity: Debug First, Then Ask

35–55 min  AI literacy activity

🖥 Terminal

Step 1: Read the traceback. Write what you think is wrong in your own words.
Step 2: Try to fix it yourself using print() statements.
        Document what you tried and what you observed.
Step 3: If still stuck, ask AI to explain the error message ONLY.
        Prompt: "Explain this Python error without giving me corrected code."
        Do NOT ask AI to fix the code.
Step 4: Fix the code yourself based on the AI explanation.
Step 5: Run pytest — all tests must pass before continuing.
Step 6: Write reflection.md covering:
        — What was the bug?
        — What did AI explain (paste the AI response)?
        — What did you change and why?
        — How did you verify the fix?
        — Can you explain every line of the fixed code without AI help?
