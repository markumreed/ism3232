Section 1 — Variables and Data Types
Variables store business information

A variable is a named container. In business scripts, variables hold prices, quantities, statuses, request amounts, and employee IDs. Name them descriptively: request_amount is better than x.

The four core data types with business examples
# String — text, always in quotes
product_name = "Laptop"
status       = "Pending"
# Integer — whole numbers
quantity     = 3
employee_id  = 10042
# Float — decimal numbers
price        = 899.99
tax_rate     = 0.07
# Boolean — True or False (capital letters required)
is_approved  = False
# Check type of any variable
print(type(quantity))   # <class 'int'>

Section 2 — Operators
Arithmetic, comparison, and logical operators
# Arithmetic
total   = price * quantity
tax     = total * tax_rate
final   = total + tax
average = total / quantity     # always float
whole   = 105 // 10            # floor division → 10
remain  = 105 % 10             # modulo (remainder) → 5
# Comparison — result is always a bool
total > 1000                   # True or False
status == "Pending"            # True if exact match
quantity != 0                 # True if not zero
# Logical
total > 500 and is_approved   # both must be True
total > 1000 or quantity > 10 # either must be True
not is_approved               # reverses the boolean
Section 3 — Input, Output, and f-strings
print(), input(), and f-string formatting
# f-string — embed variables directly in a string
print(f"Product: {product_name}")
print(f"Price: ${price:.2f}")              # .2f = 2 decimal places
print(f"Total: ${price * quantity:.2f}")    # expressions work inside {}
print(f"Over limit: {total > 1000}")        # booleans work too
# input() always returns a string — convert for math
name   = input("Enter your name: ")
amount = float(input("Enter request amount: "))
print(f"Request from {name}: ${amount:.2f}")
⚠
input() always returns a string
This is the most common Week 5 bug. If the user types 1200, Python receives the string "1200", not the number 1200. You must convert it: float(input(...)) or int(input(...)). Show this failure live by trying to add the string to a number.
