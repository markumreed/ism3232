Section 2

Course Description

📋 Official Description (cannot be changed)

Presentation of business application development using an object-oriented programming language. Good program design techniques are emphasized. Business applications are developed.

Study of software development methodologies, including data structures, indexing, file processing and user interfaces. Good program design techniques are emphasized, and illustrative business software is developed using a modern programming language.

Students learn to build complete, deployable business applications using Python — from the command line through OOP design, SQL integration, Streamlit interfaces, and a controlled GenAI feature. Emphasis throughout is on professional developer workflow: virtual environments, automated linting, testing, version control, and iterative design.

Developer Workflow & Submission Ritual

From Week 2 onwards, every assignment requires the complete nine-step pre-submission ritual. The instructor reviews your Git log. A single "final" commit at midnight tells a story.

🔁 The Nine Steps — Required Before Every Major Push
#CommandPurpose
1 pwd Confirm your terminal is in the correct project directory.
2 tree -L 3 Confirm the project folder structure looks correct.
3 source .venv/bin/activate Activate the virtual environment. Prompt must show (.venv).
4 ruff format . Auto-format all Python files.
5 ruff check . Lint check. Must return 0 errors before continuing.
6 pytest Run all tests. Must be green before continuing.
7 git status Review what has changed since the last commit.
8 git add . Stage all changes.
9 git commit -m "descriptive message" + git push Commit with a meaningful message and push.
⚠ Rule: If ruff check or pytest fail — stop. Fix errors before continuing. Never push broken code.
Section 7

AI Use Policy

✓ Permitted
  • Ask AI to explain what a traceback error means
  • Ask AI to explain what a piece of code does in plain English
  • Ask AI to suggest test cases for code you already wrote
  • Use AI to understand a concept after attempting it yourself
  • Week 16 only: use the Anthropic API as a controlled application feature
✗ Prohibited
  • Generating any submitted code
  • Asking AI to fix bugs (you must attempt first)
  • Any AI use during the midterm exam
  • Submitting AI-generated work without disclosure
  • Using AI to write the AI literacy reflection (Assignment 8)
🔍 The Debug-First, Then-Ask Workflow (Assignment 8)
  1. Read the traceback bottom-up. Write the error type and line number.
  2. Write your hypothesis before touching any code — explain the code out loud, line by line, to a rubber duck (rubber duck debugging, from The Pragmatic Programmer, Hunt & Thomas, 1999).
  3. Add strategic print() statements and run the script.
  4. If stuck after 10 minutes — ask AI to explain the error only, not to fix it.
  5. Fix the code yourself, then run pytest — all tests must pass.
  6. Write the required reflection. This is 30 of 70 points and is not optional.

Section 11

Academic Continuity

In the event of unforeseen circumstances — including public health emergencies, extreme weather, natural disasters, or extended facility closures — the following continuity plan takes effect.

📢 Communication

All updates distributed through Canvas announcements. Check Canvas daily. The course website remains accessible throughout any disruption.

🎓 Instruction

In-person sessions transition to synchronous online delivery via the university's designated platform (link in Canvas). Recordings posted within 24 hours.

📝 Assignments

Deadlines extended by a minimum of 72 hours for any week directly affected. All submissions continue via GitHub — the submission method does not change.

📋 Exams

The Midterm transitions to online format if in-person is not possible. Format (open notes, individual, same coverage) remains unchanged. 72-hour notice minimum.

🎯 Capstone

Final demonstrations via video conferencing if in-person is not possible. Requirements (live working application, eight topics) remain unchanged.

⚠ Student Responsibility

Students are responsible for monitoring Canvas announcements. Failure to check Canvas is not grounds for a deadline extension or grade appeal.

Section 12

Course Policies

12.1 Grade Dispute Policy

If you wish to discuss the grading of an assignment, you must contact your professor within one (1) week of receiving your grade on Canvas. After this one-week window has passed, it will be assumed that you accept the grade as recorded. No further discussions regarding the grading of that assignment will be possible.

⚠ Note: Grade disputes submitted after the one-week deadline will not be considered under any circumstances.
12.2 Late Work Policy

There are no opportunities to make up missed assignments, labs, quizzes, or exams. There are also no opportunities to redo any assignments, labs, quizzes, or exams for a revised grade.

⚠ Note: All work is due at the stated deadline. Missed or incomplete work will receive a grade of zero with no exceptions outside of the Medical Excuse Policy below.
12.3 Medical Excuse Policy

Students should not attend class if they are ill, particularly with: fever, gastrointestinal symptoms, or respiratory symptoms (sneezing, runny nose, sore throat, coughing).

Contact campus health immediately:
• Tampa & Sarasota-Manatee: Student Health Services — (813) 974-2331
• St. Petersburg: Wellness Center — (727) 873-4422

Required: A verification of care letter must be presented to the instructor upon the student's return to class.

12.4 Extra Credit Policy

There are no extra credit opportunities in this course. The optional Automation Bonus (+5%) is the only additional credit available and supplements — it does not replace — primary coursework.

12.5 Group Work Policy

There is no group work of any kind in this course. All assignments and assessments are to be completed individually. Sharing code or submitting another student's work constitutes academic dishonesty and will be reported to the Office of Student Conduct.

12.6 First Day Attendance Policy

Students must complete the First Day Attendance Quiz by the stated due date. This quiz serves as confirmation of your enrollment and active participation in the course.

⚠ Note: Students who do not complete the First Day Attendance Quiz by the deadline are subject to being dropped from the course.
12.7 Grades of Incomplete

The current university policy concerning incomplete grades will be followed. An "I" grade may be awarded only when a small portion of the student's work is incomplete and only when the student is otherwise earning a passing grade. For undergraduate students, the time limit may not exceed two academic semesters and/or graduation, whichever comes first. "I" grades not removed by the end of the time limit will be changed to "IF" or "IU."
