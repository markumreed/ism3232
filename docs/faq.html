How to read an error: Start at the bottom of the red output. The last line is the error type and message. The line above is the file name and line number. Read bottom-up, not top-down.
Setup & Installation
wsl --install fails with a virtualisation error

WSL requires hardware virtualisation to be enabled in your machine’s BIOS/UEFI. Many machines ship with it off.

Fix:

  • Restart your computer and enter BIOS setup — usually by pressing Del, F2, or F10 during startup (the key is shown briefly on the boot screen)
  • Find the setting labelled Intel VT-x, AMD-V, or Virtualization Technology and enable it
  • Save and restart
  • Run wsl --install again in an Administrator PowerShell window
echo $SHELL shows /bin/bash — I should be in zsh

zsh is not your default shell yet. On Mac this is unusual (zsh has been the default since macOS Catalina); on WSL it means the chsh step was skipped or did not take effect.

Mac fix:

chsh -s /bin/zsh

Enter your password, then close the terminal and reopen it. Run echo $SHELL again — it should return /bin/zsh.

WSL/Ubuntu fix:

sudo apt install zsh -y
sudo chsh -s $(which zsh) $USER

Close and reopen the Ubuntu terminal. The prompt changes from $ to %.

VS Code terminal opens PowerShell instead of Ubuntu / WSL

VS Code defaults to PowerShell on Windows. You need to switch the default terminal profile to WSL.

Fix:

  • In VS Code, press Ctrl+Shift+P to open the Command Palette
  • Type Terminal: Select Default Profile and press Enter
  • Choose Ubuntu (WSL) or wsl.exe from the list
  • Close any open terminal panels and open a new one with Ctrl+`
  • The terminal should now open inside Ubuntu with a zsh prompt
brew: command not found after installing Homebrew

Homebrew installs correctly but its binary is not yet in your shell’s PATH. The installer prints the exact commands needed to fix this — most people scroll past them.

Fix:

  • Scroll back through the terminal output from the Homebrew install
  • Look for two lines starting with “Next steps:” that contain echo and eval commands
  • Copy and run those commands exactly as printed
  • Close the terminal and reopen it, then run brew --version to confirm

If you no longer have the output, run the installer again — Homebrew detects that it is already installed and only re-prints the PATH setup steps.

chsh: PAM authentication error when switching to zsh in WSL

Plain chsh fails in some WSL configurations due to how PAM (authentication) is set up in the Ubuntu environment.

Fix: Use sudo chsh with the current user specified explicitly:

sudo chsh -s $(which zsh) $USER

If that still fails, edit /etc/passwd directly (requires root) or reinstall WSL:

wsl --unregister Ubuntu
wsl --install

Note: reinstalling WSL erases the Ubuntu filesystem. Only do this if the sudo chsh fix does not work.

code: command not found when typing in the terminal

The code shell command needs to be registered separately. On Mac it is a one-time VS Code setup step; on Windows it should happen automatically when VS Code is installed with the PATH option checked.

Mac fix:

  • Open VS Code
  • Press ⌘Shift+P to open the Command Palette
  • Type Shell Command: Install ‘code’ command in PATH and press Enter
  • Restart the terminal and run code --version to confirm

Windows WSL fix: The code command from inside WSL calls the Windows VS Code binary. Make sure VS Code for Windows (not Linux) is installed, and run code . from inside the WSL terminal — VS Code should open with the WSL remote extension active.

Git push asks for a password but rejects it

GitHub stopped accepting plain passwords for Git operations in 2021. You need to use a Personal Access Token (PAT) instead.

Fix:

  • Go to github.com → Settings → Developer Settings → Personal Access Tokens → Tokens (classic)
  • Click Generate new token (classic)
  • Give it a name (e.g., “ISM3232 laptop”) and check the repo scope
  • Click Generate token and copy the token immediately — it is only shown once
  • When Git asks for your password, paste the token

To avoid entering it every time, run:

git config --global credential.helper store

The next time you enter the token, Git will remember it.

git: command not found on Mac

Git is included with Xcode Command Line Tools, which may not be installed on your machine.

Fix:

xcode-select --install

A dialog appears — click Install. It takes 5–10 minutes. After it completes, run git --version to confirm.

Ubuntu doesn’t open automatically after a Windows restart

WSL does not auto-start after a reboot — you need to open it manually each session.

Ways to open the WSL terminal:

  • Open the Start menu and search for Ubuntu
  • Open PowerShell and type wsl
  • Open VS Code — if the terminal is set to WSL, it opens automatically when you open a terminal panel
Common Python Errors
Before searching here: If you have a full traceback, the Troubleshooting Guide has more detailed error-by-error explanations. This FAQ is for quick “what went wrong” answers.
ModuleNotFoundError: No module named ‘…’

The package is not installed in the currently active Python environment, or the virtual environment is not activated.

Fix:

  • Activate the virtual environment first: source .venv/bin/activate
  • The prompt must show (.venv) before running any script
  • If the package is still not found: pip install -r requirements.txt
  • Confirm with: pip list | grep package-name
source .venv/bin/activate
pip install -r requirements.txt
python3 your_script.py
FileNotFoundError: No such file or directory

Python cannot find the file at the path you gave it. The most common cause is running the script from a different directory than the one containing the file.

Fix:

  • Run pwd in the terminal to confirm your location
  • Run ls to see what files are in the current directory
  • Always run scripts from the project root directory: python3 src/script.py not cd src && python3 script.py
  • Use paths relative to the project root, not to the script file
SyntaxError: invalid syntax

Python cannot parse the file. It reports a line number, but the actual mistake is often one line before the reported line.

Most common causes:

  • Missing colon : after def, if, for, class
  • Unclosed parenthesis (, bracket [, or brace {
  • Backslash inside an f-string expression: use a variable instead
# Wrong — backslash in f-string
names = ["Ana", "Ben"]
print(f"First: {names[0]\n}")   # SyntaxError

# Fix — move the newline outside
print(f"First: {names[0]}")
IndentationError — unexpected indent or mixed tabs and spaces

Python is strict about consistent indentation. Mixing tabs and spaces — even invisibly — causes this error.

Fix:

  • Use 4 spaces per level, never tabs
  • In VS Code: View → Appearance → Render Whitespace to see what is there
  • Bottom-right corner shows Spaces: 4 — click it and choose Indent Using Spaces: 4
  • To convert an entire file: Ctrl+Shift+P → Convert Indentation to Spaces
NameError: name ‘x’ is not defined

A variable was used before it was assigned, or it was defined inside a function and referenced outside it (scope issue).

# Common OOP mistake — forgetting self
class BankAccount:
    def deposit(self, amount):
        balance += amount   # NameError — should be self.balance

    def deposit(self, amount):
        self.balance += amount   # Correct

Fix: Check that the variable is assigned before the line that uses it. In classes, instance attributes always need self. prefix.

TypeError — wrong type passed to a function or operation

The most common variant in this course: input() returns a string; passing it directly to math operations fails.

# Wrong
amount = input("Enter amount: ")
total = amount * 1.07   # TypeError: can't multiply str by float

# Fix — convert immediately
amount = float(input("Enter amount: "))
total = amount * 1.07

Always convert input() with int() or float() before using it in calculations.

KeyError — dictionary key not found

You accessed a key that does not exist in the dictionary. Usually a typo, or the key only exists in some records when processing a list of dicts.

record = {"name": "Ana", "score": 95}
print(record["scroe"])   # KeyError — typo

Fix:

  • Print record.keys() to see what is actually there
  • Use .get("key", default) to handle missing keys gracefully: record.get("score", 0)
  • For JSON/API responses, keys may be missing from some records — always use .get()
AttributeError: ‘NoneType’ object has no attribute ‘…’

A function or method returned None and you tried to call an attribute or method on it. In OOP this often means a constructor forgot to set an attribute or a method forgot to return self.

class Cart:
    def add(self, item):
        self.items.append(item)   # AttributeError if self.items was never created

class Cart:
    def __init__(self):
        self.items = []   # Must initialise in __init__

    def add(self, item):
        self.items.append(item)
UnicodeDecodeError when reading a CSV or text file

The file contains characters that the default encoding (often UTF-8) cannot parse — common with files exported from Excel on Windows.

Fix: Specify the encoding explicitly when opening the file:

# Try UTF-8 first
with open("data.csv", encoding="utf-8") as f:
    ...

# If that fails, try latin-1 (covers most Western European characters)
with open("data.csv", encoding="latin-1") as f:
    ...
Git shows a merge conflict — what do I do?

A merge conflict means the same lines were changed in two different commits. Git marks the conflicting sections with <<<<<<<, =======, and >>>>>>>.

Fix:

  • Open the conflicted file in VS Code — it highlights the conflict with Accept/Ignore buttons
  • Choose which version to keep (or write a combined version)
  • Delete all the <<<, ===, >>> marker lines
  • Stage and commit:
git add the_conflicted_file.py
git commit -m "Resolve merge conflict in the_conflicted_file.py"
My virtual environment is broken or pip install keeps failing

The quickest fix is to delete the venv and recreate it. The venv directory is disposable — all the real code is in your source files and requirements.txt.

# Delete the venv
rm -rf .venv

# Recreate it
python3 -m venv .venv
source .venv/bin/activate

# Reinstall all dependencies
pip install -r requirements.txt

Never commit the .venv/ directory to git. Confirm .venv is in your .gitignore.

Streamlit app crashes immediately or shows a blank page

Two common causes: the venv is not activated when running streamlit run, or there is a Python error on startup before any output is shown.

Fix:

  • Always activate the venv before running: source .venv/bin/activate
  • Check the terminal output — the error message appears there even if the browser is blank
  • If the error is ModuleNotFoundError, reinstall requirements: pip install -r requirements.txt
  • If the browser shows a blank page with no terminal error, try a hard reload: Ctrl+Shift+R
