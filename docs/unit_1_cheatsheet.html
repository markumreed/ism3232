file operations
cat file.txtPrint file contents to terminal
cp file.txt copy.txtCopy a file (original stays)
mv file.txt new.txtRename or move (original gone)
head -5 file.txtFirst 5 lines of a file
tail -5 file.txtLast 5 lines of a file
echo 'text' > file.txtWrite to file (overwrites)
echo 'text' >> file.txtAppend to file
✗ rm without the safety ritual
rm file.txt   # no undo — file is gone permanently
✓ rm safety ritual — always three steps first
pwd           # step 1: confirm location
ls            # step 2: confirm what is here
rm file.txt   # step 3: now delete
ls            # confirm it is gone
virtual environment workflow
Create + activate
python3 -m venv .venv
source .venv/bin/activate
# prompt now shows (.venv)
Verify activation
which python3
# must show .venv/bin/python3
Install + freeze
pip install pytest ruff
pip freeze > requirements.txt
Recreate from requirements
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
✗ Commit .venv/
git add .venv/   # 50-100MB, breaks on other machines
✓ .gitignore — create before first git add
echo '.venv/' > .gitignore
echo '__pycache__/' >> .gitignore
echo '*.pyc' >> .gitignore

ruff — linting and formatting
ruff format .Auto-format all .py files in current folder
ruff check .Report style/logic issues (must be clean before push)
ruff check --fix .Auto-fix fixable issues
the pre-submission ritual — nine steps, every time
pwd                              # 1. confirm location
tree -L 3                        # 2. confirm structure
source .venv/bin/activate        # 3. activate venv
ruff format .                    # 4. auto-format
ruff check .                     # 5. must be clean
pytest                           # 6. must be green
git status                       # 7. see what changed
git add .                        # 8. stage
git commit -m "descriptive msg"  # 9. commit + push
