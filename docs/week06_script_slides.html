🗣 🗣 Trace the accumulator

Let me trace this manually. Before the loop: total is 0. Iteration 1: total = 0 + 200 = 200. Iteration 2: total = 200 + 1500 = 1700. Iteration 3: total = 1700 + 400 = 2100. Iteration 4: total = 2100 + 2200 = 4300. Iteration 5: total = 4300 + 850 = 5150. The key: total is initialised to zero BEFORE the loop. If you initialise inside the loop, it resets to zero on every iteration. That is the single most common accumulator bug.
