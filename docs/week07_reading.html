Why functions?

Functions exist for two reasons: to eliminate repeated code and to make logic testable. If you have written the same approval check three times in three different places, a function reduces that to one definition and three calls. If your logic lives in a function with a return value, you can assert against it in pytest.

Before and after — the case for functions
# Before: logic repeated three times
if amount1 > 1000: print('Manager needed')
if amount2 > 1000: print('Manager needed')
if amount3 > 1000: print('Manager needed')

# After: one function, called anywhere
APPROVAL_LIMIT = 1000.00

def requires_review(amount: float) -> bool:
    """Return True if amount requires manager review."""
    return amount > APPROVAL_LIMIT

# Now testable:
assert requires_review(1001) is True
assert requires_review(1000) is False  # boundary: > not >=

Complete function anatomy

All required elements
APPROVAL_LIMIT = 1000.00
TAX_RATE       = 0.07

def calculate_total(unit_price: float, quantity: int,
                    tax_rate: float = TAX_RATE) -> float:
    """Return total cost including tax."""
    subtotal = unit_price * quantity
    return subtotal * (1 + tax_rate)

def get_approver(amount: float) -> str:
    """Return the required approver tier."""
    if amount <= 500:
        return 'Auto'
    elif amount <= 2000:
        return 'Manager'
    else:
        return 'Director'
⚠ return, not print
⚖ print() makes the function untestable
✗ pytest cannot assert what a print() produces
def requires_review(amount: float):
    print(amount > 1000)   # prints True or False to terminal

# In tests:
assert requires_review(1001) is True
# AssertionError — requires_review() returned None, not True
✓ return lets pytest assert the result
def requires_review(amount: float) -> bool:
    return amount > 1000   # caller receives the value

# In tests:
assert requires_review(1001) is True   # passes
assert requires_review(999) is False   # passes

Every business logic function must return its result. A function that only print()s cannot be tested with assert.

A function that only calls print() cannot be tested with assert. A function that returns a value can. All business logic functions must return their result.

Scope

Local variables do not exist outside their function
⚖ Accessing a local variable from outside the function
✗ NameError — 'result' does not exist outside the function
def calc_tax(amount: float) -> float:
    tax = amount * 0.07   # 'tax' is local
    return tax

calc_tax(1000)
print(tax)   # NameError: name 'tax' is not defined
✓ Use the return value
def calc_tax(amount: float) -> float:
    tax = amount * 0.07
    return tax

result = calc_tax(1000)   # capture the returned value
print(result)              # 70.0

Local variables exist only while the function is running. Use return to pass a value out.

def calculate_tax(amount: float) -> float:
    tax = amount * 0.07   # 'tax' is LOCAL to this function
    return tax

result = calculate_tax(1000)
print(result)  # 70.0 — fine, we returned it

print(tax)     # NameError: name 'tax' is not defined
# 'tax' was destroyed when the function returned
✍ Think before reading on

If requires_review() uses print(self.amount > 1000) instead of return, what happens when you write assert requires_review(1001) is True? What does pytest report?

Writing pytest tests

tests/test_business_rules.py — all six required types
from business_rules import requires_review, get_approver

# 1. True case
def test_requires_review_true():
    assert requires_review(1001.00) is True

# 2. False case
def test_requires_review_false():
    assert requires_review(500.00) is False

# 3. Boundary — exactly AT the limit (> not >=)
def test_requires_review_at_limit():
    assert requires_review(1000.00) is False

# 4. Boundary — one cent above
def test_requires_review_one_above():
    assert requires_review(1000.01) is True

# 5. Tier tests
def test_get_approver_auto():
    assert get_approver(500.00) == 'Auto'

def test_get_approver_manager():
    assert get_approver(501.00) == 'Manager'
Run tests
pytest -v
