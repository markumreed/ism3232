Part A — GenAI-Supported Business Feature
🤖
GenAI is a required application feature, not just a dev tool
Students integrate a controlled AI capability into their Streamlit application. The AI does something useful for the business user — summarising a request, classifying urgency, drafting a response — with required disclosure and human review controls built in.

Required GenAI Controls
⚠
All six controls are required — missing any one fails the feature
A GenAI feature without disclosure, human review, and a limitations statement is not a professional implementation. These are graded on the capstone rubric.
The six required controls

1. Disclosure in the UI: AI-generated content must be clearly labelled in the Streamlit interface.

2. Human review before save: AI output is displayed for the user to review before it is written to the database. Never auto-save AI output.

3. No sensitive data sent to the API: Do not send names, IDs, dollar amounts, or PII. Send only anonymised or generic descriptions.

4. Test cases for the feature: At least one pytest test for the AI feature logic, with a mocked API response.

5. Limitations statement in README: What the feature does, what it does not do, and when it might produce incorrect output.

6. Student can explain the feature: During the presentation, the student must explain what the AI does, how it is called, and why the controls are there.

ai_feature.py — minimal summarisation implementation
import anthropic

def summarise_request(category: str, description: str) -> str:
    """
    Return a one-sentence AI summary of a business request.
    Only category and description are sent — no names or amounts.
    """
    client = anthropic.Anthropic()
    prompt = f"Summarise this {category} business request in one sentence:\n{description}"
    message = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=150,
        messages=[{"role": "user", "content": prompt}]
    )
    return message.content[0].text
