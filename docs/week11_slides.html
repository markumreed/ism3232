Composition: Classes That Contain Other Classes

0–20 min  Composition: RequestManager class

🧩 Composition: the manager class pattern

In most business systems, one object holds a collection of other objects. A RequestManager holds many BusinessRequest instances. This is composition — building complex objects from simpler ones by having one class contain instances of another. Composition is more common than…
“In most business systems, one object holds a collection of other objects. A RequestManager holds many BusinessRequest instances. This is composition — building complex objects from simpler ones. It is more common in business applications than inheritance.”

🐍 Code

class RequestManager:
    """Manages a collection of business requests."""

    def __init__(self):
        self.requests = []   # starts empty — instances added later

    def add_request(self, request):
        """Add a BusinessRequest instance to the manager."""
        self.requests.append(request)

    def list_pending(self):
        """Return a list of all requests with status Pending."""
        return [r for r in self.requests if r.status == "Pending"]

    def total_amount(self):
        """Return the total value of all requests."""
        return sum(r.amount for r in self.requests)

    def get_by_status(self, status):
        """Return all requests matching the given status."""
        return [r for r in self.requests if r.status == status]

    def summary_report(self):
        """Print a formatted summary of all requests by status."""

Inheritance: When to Use It

20–40 min  Manager + entity working together

🧩 Inheritance in business applications: use sparingly

Inheritance lets one class inherit attributes and methods from a parent class. In business systems it is useful for modelling hierarchical relationships — a TravelRequest and an EquipmentRequest are both types of BusinessRequest , sharing core attributes but adding their own. Do…
“Use inheritance when there is a genuine is-a relationship. A TravelRequest is a BusinessRequest. An EquipmentRequest is a BusinessRequest. They share core attributes but add their own. Do not use inheritance just to avoid repeating code — that is what composition is for.”

OOP to SQL Mapping

40–55 min  Light inheritance

🧩 Classes, attributes, and instances map directly to SQL tables

Understanding this mapping makes SQL design intuitive. When students design their capstone database schema in Week 13, they will translate their class designs directly into SQL CREATE TABLE statements. OOP concept SQL equivalent Example class table BusinessRequest → requests…

🐍 Code

class TravelRequest(BusinessRequest):  # inherits from BusinessRequest
    """A business request specifically for travel."""

    def __init__(self, request_id, requester, amount, destination):
        super().__init__(request_id, requester, "Travel", amount)
        self.destination = destination   # extra attribute

    def requires_review(self):
        """Travel requests over $500 require review (lower threshold)."""
        return self.amount > 500   # overrides parent method

trip = TravelRequest(201, "Morgan", 850, "Atlanta")
print(trip.destination)        # Atlanta
print(trip.requires_review())  # True (850 > 500, travel threshold)
trip.approve()                 # inherited from BusinessRequest
print(trip.status)             # Approved
