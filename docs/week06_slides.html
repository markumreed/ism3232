Conditionals

0–15 min  Conditionals

🧩 Business rules are conditionals

An approval threshold, an eligibility check, a routing rule — all of these are if/elif/else statements. Every conditional in a business script should map to a real business rule a human decision-maker would recognise. if / elif / else — multi-tier business approval routing…
“Every business rule you have ever seen is a conditional. An approval threshold. An eligibility check. A routing decision. In Python, these are if/elif/else statements. When you write one, you should be able to point to a real business rule it represents.”

amount = 1500
if amount <= 500:
    print("Auto-approved")
elif amount <= 2000:
    print("Manager approval required")
elif amount <= 10000:
    print("Director approval required")
else:
    print("Executive approval required")

Lists and for Loops

15–35 min  Lists + for loops

🧩 Lists hold multiple records of the same kind

In business systems a list holds pending requests, product lines, or daily transactions. A for loop processes every item in order. The accumulator pattern — start a counter at zero, add to it each iteration — is how you compute totals and filtered counts. Lists, for loops, and…
“The accumulator pattern is one of the most important patterns in business programming: start a counter at zero, loop through the data, add to the counter each iteration. This is how you compute totals, filtered counts, and summaries.”

🐍 Code

amounts = [450, 1200, 89, 3500, 250]

total    = 0
flagged  = []
for amount in amounts:
    total += amount
    if amount > 1000:
        flagged.append(amount)

print(f"Total: ${total:,.2f}")
print(f"Flagged ({len(flagged)}): {flagged}")

# Common list operations
amounts.append(600)   # add to end
len(amounts)           # count items
amounts[0]            # first item — index starts at 0
amounts[-1]           # last item
sorted(amounts)        # sorted copy (original unchanged)

Dictionaries as Business Records

35–50 min  Dictionaries

🧩 A dictionary is a single business record

A dictionary stores data as key–value pairs — like one row in a spreadsheet. A list of dictionaries is a collection of records, mirroring a full database table. This is the data pattern you will use in every remaining assignment and in the capstone. Dictionary operations and the…
“A list of dictionaries is a collection of records. It mirrors a database table. One dict = one row. A list of dicts = the whole table. You will use this pattern for the rest of the course until we replace it with actual SQL in Week 13.”
