The mental model — blueprint and instance

Class = blank form. Instance = filled-out form.

A class defines what every object of that type has and can do. An instance is one specific object built from that blueprint. BusinessRequest is the blank purchase form. req_101 is Taylor's filled-out request.

Writing the BusinessRequest class — every line explained
APPROVAL_LIMIT = 1000.00

class BusinessRequest:
    """Represents a single purchase approval request."""

    def __init__(self, req_id: int, requester: str,
                 category: str, amount: float) -> None:
        """Initialise a new request with Pending status."""
        self.req_id    = req_id
        self.requester = requester
        self.category  = category
        self.amount    = amount
        self.status    = 'Pending'   # default — not a parameter

    def approve(self) -> None:
        """Mark this request as Approved."""
        self.status = 'Approved'

    def reject(self) -> None:
        """Mark this request as Rejected."""
        self.status = 'Rejected'

    def requires_review(self) -> bool:
        """Return True if amount exceeds the approval limit."""
        return self.amount > APPROVAL_LIMIT

    def __repr__(self) -> str:
        return (f'BusinessRequest(id={self.req_id}, '
                f'requester={self.requester!r}, '
                f'amount=${self.amount:,.2f}, '
                f'status={self.status!r})')
⚠ self is the instance
⚖ Two mistakes in one method — both silent failures
✗ Missing self parameter + local variable instead of self.attribute
class BusinessRequest:
    def approve():            # missing self → TypeError
        status = 'Approved'   # local variable — vanishes immediately

req = BusinessRequest(1, 'T', 'E', 500)
req.approve()   # TypeError: approve() takes 0 positional arguments
✓ self as first parameter + self.attribute to persist the change
class BusinessRequest:
    def approve(self) -> None:
        self.status = 'Approved'   # changes THIS instance

req = BusinessRequest(1, 'T', 'E', 500)
req.approve()
print(req.status)   # 'Approved'

Every method needs self as its first parameter. Every attribute change needs self.attribute, not a bare variable name.

When you call req.approve(), Python passes req as self automatically. self.status = 'Approved' changes the status of that specific instance. Forgetting self causes TypeError. Writing status = 'Approved' without self creates a local variable that vanishes immediately.

Instance independence — the most important demo

Approving one instance must not affect others
req1 = BusinessRequest(1, 'Taylor', 'Equipment', 1200.00)
req2 = BusinessRequest(2, 'Jordan', 'Travel',     350.00)

req1.approve()

print(req1.status)  # Approved
print(req2.status)  # Pending ← still Pending!

# Each instance has its own copy of every attribute.

The six required test types

tests/test_models.py
from models import BusinessRequest

def test_default_status_is_pending():
    req = BusinessRequest(1, 'T', 'Equipment', 500.00)
    assert req.status == 'Pending'

def test_approve_sets_status():
    req = BusinessRequest(2, 'T', 'Equipment', 500.00)
    req.approve()
    assert req.status == 'Approved'

def test_requires_review_true():
    req = BusinessRequest(3, 'T', 'Equipment', 1200.00)
    assert req.requires_review() is True

def test_requires_review_false():
    req = BusinessRequest(4, 'T', 'Equipment', 500.00)
    assert req.requires_review() is False

def test_boundary_at_limit():
    req = BusinessRequest(5, 'T', 'Equipment', 1000.00)
    assert req.requires_review() is False   # > not >=

def test_instance_independence():
    req1 = BusinessRequest(6, 'A', 'Equipment', 500.00)
    req2 = BusinessRequest(7, 'B', 'Travel',    500.00)
    req1.approve()
    assert req1.status == 'Approved'
    assert req2.status == 'Pending'   # unchanged

Key terms this week

classA blueprint that defines what objects of that type have (attributes) and can do (methods).
instance / objectOne specific thing built from a class.
__init__The constructor — runs once when an instance is created.
selfThe specific instance the method is being called on.
instance attributeA variable stored on a specific instance. Accessed with self.name.
methodA function defined inside a class. Always takes self as the first parameter.
__repr__Returns a readable string representation of the object. Used by print().
encapsulationBundling data and the methods that operate on it inside one class.
