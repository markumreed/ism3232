Section 1 — Conditionals
Business rules are conditionals

An approval threshold, an eligibility check, a routing rule — all of these are if/elif/else statements. Every conditional in a business script should map to a real business rule a human decision-maker would recognise.

if / elif / else — multi-tier business approval routing
amount = 1500
if amount <= 500:
    print("Auto-approved")
elif amount <= 2000:
    print("Manager approval required")
elif amount <= 10000:
    print("Director approval required")
else:
    print("Executive approval required")

# Combined conditions
category = "Travel"
if amount > 500 and category == "Travel":
    print("Requires travel policy acknowledgment")
Section 2 — Lists and for Loops
Lists hold multiple records of the same kind

In business systems a list holds pending requests, product lines, or daily transactions. A for loop processes every item in order. The accumulator pattern — start a counter at zero, add to it each iteration — is how you compute totals and filtered counts.

Lists, for loops, and the accumulator pattern
amounts = [450, 1200, 89, 3500, 250]

total    = 0
flagged  = []
for amount in amounts:
    total += amount
    if amount > 1000:
        flagged.append(amount)

print(f"Total: ${total:,.2f}")
print(f"Flagged ({len(flagged)}): {flagged}")

# Common list operations
amounts.append(600)   # add to end
len(amounts)           # count items
amounts[0]            # first item — index starts at 0
amounts[-1]           # last item
sorted(amounts)        # sorted copy (original unchanged)
Section 3 — Dictionaries as Business Records
A dictionary is a single business record

A dictionary stores data as key–value pairs — like one row in a spreadsheet. A list of dictionaries is a collection of records, mirroring a full database table. This is the data pattern you will use in every remaining assignment and in the capstone.

Dictionary operations and the list-of-dicts pattern
request = {
    "id":        101,
    "requester": "Taylor",
    "category":  "Travel",
    "amount":    1200.00,
    "status":    "Pending"
}
print(request["requester"])       # Taylor
request["status"] = "Approved"  # update a value

# List of dicts = table of records
requests = [
    {"id": 101, "requester": "Taylor", "amount": 1200, "status": "Pending"},
    {"id": 102, "requester": "Jordan", "amount": 450,  "status": "Pending"},
    {"id": 103, "requester": "Morgan", "amount": 3500, "status": "Approved"},
]
for req in requests:
    if req["status"] == "Pending":
        print(f"Pending: {req['requester']} — ${req['amount']:,.2f}")
Section 4 — Live Coding: Business Record Processor
business_record_processor.py — type together, students extend for assignment
requests = [
    {"id": 101, "requester": "Taylor", "amount": 1200, "status": "Pending"},
    {"id": 102, "requester": "Jordan", "amount": 450,  "status": "Pending"},
    {"id": 103, "requester": "Morgan", "amount": 3500, "status": "Approved"},
    {"id": 104, "requester": "Riley",  "amount": 89,   "status": "Pending"},
]
APPROVAL_LIMIT = 1000
total, flagged = 0, []
for req in requests:
    if req["status"] == "Pending":
        total += req["amount"]
        if req["amount"] > APPROVAL_LIMIT:
            flagged.append(req)
print(f"Total pending: ${total:,.2f}")
print(f"Needing review: {len(flagged)}")
with open("output/summary.txt", "w") as f:
    f.write(f"Total: ${total:,.2f}\nFlagged: {len(flagged)}\n")
