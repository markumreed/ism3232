Section 1 — The zsh Prompt
What the prompt tells you

The zsh prompt is not decoration. Every character is information: who you are, what machine you are on, and — most critically — where you are in the filesystem. Before typing any command, students must be able to read their prompt.

Anatomy of the zsh prompt
# Typical prompt:
yourname@MacBook-Pro ~ %

yourname     # your username
@MacBook-Pro # machine name
~            # current directory (~ = home directory)
%            # prompt character — zsh uses %, bash uses $
💡
Instructor tip
Ask: "What does the ~ mean?" Have students run pwd to confirm it matches their home directory path. This connection — that ~ is just a shortcut for /Users/yourname — is one of the most useful mental models to cement early.
Section 2 — Navigation Commands
The golden rule: you are always somewhere

Every command runs relative to your current location unless you specify otherwise. Knowing where you are — and how to move — is the skill that makes everything else work. Run pwd compulsively until it becomes habit.

Required navigation commands — type each one and read the output
pwd               # Print Working Directory — where am I?
ls                # list visible files and folders
ls -la            # list ALL files including hidden (starts with .)
cd Documents      # enter Documents (relative path)
cd ..             # go up one directory level
cd ~              # jump home from anywhere
cd ~/ism3232      # go to course folder using path from home
mkdir new_folder  # create a directory
touch file.txt    # create an empty file
clear             # clear the terminal screen
tree              # show directory structure as a tree
tree -L 2         # tree limited to 2 levels deep
CommandWhat it doesCommon mistake
pwdShows full path of current locationForgetting to run this before rm

Section 3 — File Operations
⚠
Safety rule — rm is permanent
Always run pwd and ls before rm. There is no Trash. There is no undo. Deleted files are gone immediately. Demonstrate this safety habit every time you use rm. Require it in assignment screenshots throughout the course.
Required file operation commands
cp file.txt copy.txt           # copy a file
cp file.txt ../other/          # copy into another folder
mv old.txt new.txt             # rename a file
mv file.txt ../other/          # move to another folder
cat file.txt                   # print entire file to terminal
head -5 file.txt               # first 5 lines
tail -5 file.txt               # last 5 lines
less file.txt                  # scrollable view (q to quit)
code .                         # open current folder in VS Code
code hello.py                  # open specific file in VS Code

# ALWAYS before rm:
pwd                            # confirm your location
ls                             # confirm what is here
rm file.txt                    # delete — permanent, no undo
rm -r folder/                  # delete folder recursively
