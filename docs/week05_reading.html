The four core data types

Every value in Python has a type. The type determines what operations are legal. These four cover nearly all business data.

Business examples of all four types
# str — text
product_name = 'Ergonomic Office Chair'
category     = 'Furniture'
status       = 'Pending'

# int — whole numbers
quantity    = 12
employee_id = 1047

# float — decimal numbers
unit_price = 449.99
tax_rate   = 0.07
discount   = 0.15

# bool — True or False, always DERIVED from a comparison
total          = unit_price * quantity
requires_review = total > 1000    # True (5399.88 > 1000)
over_budget     = total > 5000    # True

# Inspect any type
print(type(product_name))    # <class 'str'>
print(type(quantity))        # <class 'int'>
print(type(unit_price))      # <class 'float'>
print(type(requires_review)) # <class 'bool'>
⚠ Booleans must be computed

Writing is_approved = True is hardcoding — it reflects no real business rule. Always derive booleans from a comparison: is_approved = amount <= LIMIT.

Operators

All operators with business examples
price = 449.99
qty   = 17

# Arithmetic
subtotal   = price * qty           # 7649.83
tax        = subtotal * 0.07       # 535.49
total      = subtotal + tax        # 8185.32

# Floor division — whole boxes only
boxes    = qty // 4               # 4 (17 ÷ 4, drop the remainder)
leftover = qty % 4               # 1 (remainder of 17 ÷ 4)

# Comparison — always returns a bool
print(total > 5000)    # True
print(qty == 20)       # False
print(price != 0)      # True

# Logical — combine conditions
needs_approval = total > 1000 and discount < 0.20
print(needs_approval)  # True
⚠ // and % are not / and percent

17 // 5 is 3, not 3.4 (floor division drops the decimal). 17 % 5 is 2 (the remainder), not a percentage. These two operators trip students up every week.

input(), print(), and f-strings

The most important bug to understand this week
# This crashes:
amount = input('Enter amount: ')   # input() returns a STRING
total  = amount + 100              # TypeError: can't add str and int

# The fix — wrap input() immediately
amount = float(input('Enter amount: '))
total  = amount + 100              # works: float + int = float
input() always returns str

Every single time. Even when the user types a number. Wrap it immediately: float(input(...)) or int(input(...)). This prevents the most common Week 5 TypeError.

⚖ input() returns str — always
✗ This crashes on every run
amount = input('Enter amount: ')  # returns '1200' not 1200
total  = amount * 1.07
# TypeError: can't multiply str by float
✓ Wrap input() immediately — before any arithmetic
amount = float(input('Enter amount: '))  # convert immediately
total  = amount * 1.07
# Works: 1284.0

The rule: input() returns a string every single time. Wrap it before you store it.

f-string formatting for business output
product = 'Standing Desk'
price   = 1249.50
qty     = 7
total   = price * qty

print(f'Product:  {product}')
print(f'Price:    ${price:.2f}')        # 2 decimal places
print(f'Total:    ${total:,.2f}')        # comma separator + 2 decimals
print(f'Tax rate: {0.072:.1%}')          # percentage: 7.2%
print(f'Boxes:    {qty // 4}')           # expression inside { }

Key terms this week

strString — text enclosed in quotes.
intInteger — a whole number with no decimal.
floatFloating-point — a number with a decimal component.
boolBoolean — True or False. Always derived from a comparison.
type()Returns the data type of any value.
//Floor division — divides and rounds down to the nearest integer.
%Modulo — returns the remainder after division.
f-stringA string prefixed with f that evaluates expressions inside {curly braces}.
:.2ff-string format spec: 2 decimal places.
:,.2ff-string format spec: comma separator and 2 decimal places.

Module Quiz ★ Midterm-Eligible

Quiz questions repeat on the exam verbatim or with minor variation. Click each answer to reveal it.

Q1 · Multiple Choice
What does the // operator return in Python?
  • a) Floating-point division
  • b) The remainder after division
  • c) Integer (floor) division
  • d) Exponentiation
Answer
7 // 2 is 3, not 3.5. The result is always rounded down to the nearest integer.
Q2 · Multiple Choice
What is the output of: type(3.0)?
  • a) <class 'int'>
  • b) <class 'float'>
  • c) <class 'str'>
  • d) <class 'num'>
Answer
The decimal point makes 3.0 a float, even though its value is a whole number.
Q3 · Multiple Choice
Which f-string format specifier displays 1234567.89 as 1,234,567.89?
  • a) :.2f
  • b) :,.2f
  • c) :>2f
  • d) :.2d
Answer
The comma adds thousands separators; .2 sets decimal places; f specifies float formatting.
Q4 · Multiple Choice
Which expression converts the string "42" to the integer 42?
  • a) str(42)
  • b) float("42")
  • c) int("42")
  • d) bool("42")
Answer
int() converts a string or float to an integer. str() goes the other direction.
Q5 · Code Trace
What is the output?
price = 49.99
qty = 3
total = price * qty
print(f'Total: ${total:,.2f}')
Answer
Total: $149.97 — price * qty = 149.97, formatted with comma separator and two decimal places.
Q6 · Code Trace
What is printed?
x = 17
print(x // 5)
print(x % 5)
Answer
3 2 — 17 // 5 = 3 (floor division); 17 % 5 = 2 (remainder).
Q7 · True / False
In Python, "3" + 3 raises a TypeError.
Answer
True — Python does not auto-convert types. You cannot add a string and an integer. Convert first: int("3") + 3, or "3" + str(3).
Q8 · True / False
The % operator returns a percentage as a float.
Answer
False — % is the modulus operator; it returns the remainder after integer division. 10 % 3 is 1. To get a percentage you multiply by 100: rate * 100.
Q9 · Short Answer
What is the difference between = and == in Python?
Answer
= is assignment — it stores a value in a variable (x = 5). == is comparison — it tests equality and returns True or False (x == 5 → True). Using = where == is expected is one of the most common Python bugs.
