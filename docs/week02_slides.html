Always run pwd and ls before rm. There is no Trash. There is no undo. Deleted files are gone immediately. Demonstrate this safety habit every time you use rm. Require it in assignment screenshots throughout the course.

The zsh Prompt

0–15 min  zsh mental model

🧩 What the prompt tells you

The zsh prompt is not decoration. Every character is information: who you are, what machine you are on, and — most critically — where you are in the filesystem. Before typing any command, students must be able to read their prompt. Anatomy of the zsh prompt # Typical…
“ You are always somewhere. Every command you run executes relative to your current location in the filesystem. Knowing where you are — and how to move — is the skill that makes everything else work. We will reinforce this every class.”

🐍 Code

# Typical prompt:
yourname@MacBook-Pro ~ %

yourname     # your username
@MacBook-Pro # machine name
~            # current directory (~ = home directory)
%            # prompt character — zsh uses %, bash uses $
# Typical prompt: yourname@MacBook-Pro ~ % yourname # your username @MacBook-Pro # machine name ~ # current directory (~ = home directory) % # prompt character — zsh uses %, bash uses $

🧩 The golden rule: you are always somewhere

Every command runs relative to your current location unless you specify otherwise. Knowing where you are — and how to move — is the skill that makes everything else work. Run pwd compulsively until it becomes habit. Required navigation commands — type each one and read the…
“Unlike the Trash or Recycle Bin, rm deletes immediately with no recovery. There is no undo. Demonstrate this — create a test file, show it with ls, rm it, show it is gone.”

💻 Terminal

pwd               # Print Working Directory — where am I?
ls                # list visible files and folders
ls -la            # list ALL files including hidden (starts with .)
cd Documents      # enter Documents (relative path)
cd ..             # go up one directory level
cd ~              # jump home from anywhere
cd ~/ism3232      # go to course folder using path from home
mkdir new_folder  # create a directory
touch file.txt    # create an empty file
clear             # clear the terminal screen
tree              # show directory structure as a tree
tree -L 2         # tree limited to 2 levels deep
pwd # Print Working Directory — where am I? ls # list visible files and folders ls -la # list ALL files including hidden (starts with .) cd Documents # enter Documents (relative path) cd .. # go up one directory level cd ~ # jump
