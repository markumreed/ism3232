Course Glossary

Every key term introduced across all 16 weeks, organized alphabetically. Each entry shows which week introduced the concept. Use the unit filters to focus on a specific topic area.

# ABCD EFG ILMN PRST VWZ
#
.. (dot dot)Week 1 · Shell
The parent directory. cd .. moves you up one level in the file tree.
~ (tilde)Week 1 · Shell
Shorthand for your home directory. cd ~ takes you home from anywhere.
%Week 5 · Python
Modulo — returns the remainder after division. 7 % 3 evaluates to 1.
//Week 5 · Python
Floor division — divides and rounds down to the nearest integer. 7 // 3 evaluates to 2.
:,.2fWeek 5 · Python
f-string format spec: comma thousands separator and 2 decimal places. f"${total:,.2f}" → $1,234.56
See also: f-string
:.2fWeek 5 · Python
f-string format spec: 2 decimal places, no comma. f"{price:.2f}" → 9.99
See also: f-string
A
absolute pathWeek 1 · Shell
A path starting from root (/). Works from anywhere regardless of your current directory.
See also: relative path
accumulator patternWeek 6 · Python
Initialise a variable before the loop; update it inside. Produces a running total or collected list with each iteration.
total = 0
for item in items:
    total += item["price"]
add_request(req)Week 11 · OOP
A method that maps to INSERT INTO requests VALUES (...) in SQL. Encapsulates the database write behind a Python method call.
See also: class BusinessRequest
aliasWeek 3 · Dev Tools
A shortcut that expands to a longer command when typed. Defined in .zshrc and loaded on every new terminal session.
See also: .zshrc
append()Week 6 · Python
Adds one item to the end of a list in place. records.append(new_item)
B
boolWeek 5 · Python
Boolean — True or False. Always derived from a comparison expression or boolean context.
See also: if / elif / else
C
catWeek 2 · Shell
Print the full contents of a file to the terminal. Useful for quickly checking small text files.
cdWeek 2 · Shell
Change Directory — move to a different folder. cd .. goes up one level; cd ~ goes to home.
See also: .. (dot dot), ~ (tilde)
classWeek 10 · OOP
A blueprint that defines what objects of that type have (attributes) and can do (methods). Defined with the class keyword.
class BusinessRequest:
    def __init__(self, title, amount):
        self.title = title
        self.amount = amount
See also: __init__, instance / object
class BusinessRequestWeek 11 · OOP
Maps to CREATE TABLE requests (...) in SQL. The class definition is the schema; each instance is one row.
See also: instance req_101
code .Week 2 · Shell
Open the current folder as a VS Code workspace. The code CLI must be installed via VS Code's Command Palette (Shell Command: Install 'code').
cpWeek 2 · Shell
Copy a file — the original stays in place. cp source.txt dest.txt
See also: mv
D
DEFAULT 'Pending'Week 13 · SQLite
A column constraint used automatically when no value is provided on INSERT. Example: status TEXT DEFAULT 'Pending'
dict key accessWeek 6 · Python
record['key'] — retrieves the value for that key. Raises KeyError if missing; use .get('key') for safe access.
See also: list-of-dicts
directoryWeek 1 · Shell
A folder. Directories contain files and other directories, forming a tree starting from root (/ on Mac/Linux).
See also: absolute path, relative path
E
encapsulationWeek 10 · OOP
Bundling data and the methods that operate on it inside one class. External code interacts through defined methods rather than touching internal state directly.
See also: class, method
F
f-stringWeek 5 · Python
A string prefixed with f that evaluates Python expressions inside {curly braces}. The standard way to embed variables in strings (Python 3.6+).
name = "Alice"
print(f"Hello, {name}!")        # Hello, Alice!
print(f"Total: ${amount:,.2f}") # Total: $1,234.56
See also: :,.2f, :.2f
floatWeek 5 · Python
Floating-point — a number with a decimal component. May have tiny rounding errors; use round() for display.
See also: int
for loopWeek 6 · Python
Iterates over every item in a sequence, executing the loop body once per item. Works on any iterable: lists, dicts, strings, ranges.
for record in records:
    print(record["title"])
G
get by idWeek 11 · OOP
A method that maps to SELECT * WHERE id = ?. Returns a single object (row) by its unique primary key.
git add .Week 4 · Git
Stages all changes in the current directory for the next commit. Prefer staging specific files (git add file.py) to avoid accidentally including .venv/ or secrets.
git commit -mWeek 4 · Git
Creates a snapshot of staged changes with a message. Write messages that describe why, not just what changed.
git initWeek 4 · Git
Initialises a new Git repository in the current folder, creating a hidden .git/ directory that stores all history.
git log --onelineWeek 4 · Git
Shows a compact history of all commits — one line each with hash and message. Add --graph to visualize branches.
git pushWeek 4 · Git
Sends local commits to the remote repository on GitHub. First push requires git push -u origin main to set the upstream branch.
git statusWeek 4 · Git
Shows which files have changed since the last commit — staged (green) and unstaged (red). Run before every commit.
.gitignoreWeek 3 · Dev Tools
A file telling Git which files and folders to never track. .venv/ must always be listed — never commit your virtual environment.
See also: venv
I
if / elif / elseWeek 6 · Python
Branching — runs different code depending on which condition is True. Only one branch executes. elif tests additional conditions; else is the fallback.
if status == "Approved":
    process(req)
elif status == "Pending":
    queue(req)
else:
    reject(req)
__init__Week 10 · OOP
The constructor — runs automatically when an instance is created. Sets initial attribute values on self.
See also: self, instance attribute
instance / objectWeek 10 · OOP
One specific thing built from a class. If BusinessRequest is the class (blueprint), then req = BusinessRequest(...) creates an instance.
See also: class
instance attributeWeek 10 · OOP
A variable stored on a specific instance. Accessed with self.name inside methods, obj.name from outside. Each instance has its own copy.
instance req_101Week 11 · OOP
Maps to one row in the requests table. The instance's attributes correspond directly to the row's column values.
See also: class BusinessRequest
intWeek 5 · Python
Integer — a whole number with no decimal. Division with / always returns a float; use // for an integer result.
See also: //, float
INTEGERWeek 13 · SQLite
SQLite column type for IDs, counts, and whole numbers. Do not use for money — use REAL instead.
See also: REAL, PRIMARY KEY AUTOINCREMENT
L
list-of-dictsWeek 6 · Python
One dict per record; the list holds all records. The in-memory equivalent of a database table: one dict = one row, the list = the table.
records = [
    {"id": 1, "title": "Laptop",  "amount": 1200.0},
    {"id": 2, "title": "Monitor", "amount":  450.0},
]
list_by_status('Pending')Week 11 · OOP
A method that maps to SELECT * WHERE status = 'Pending'. Filters all records by a field value and returns matching instances.
lsWeek 2 · Shell
List — shows the contents of the current directory. Add -la to include hidden files with full details.
ls -laWeek 2 · Shell
List all, including hidden files (those starting with .), in long format with permissions, size, and modification date.
M
methodWeek 10 · OOP
A function defined inside a class. Always takes self as the first parameter, giving it access to the instance's own attributes.
See also: self, class
mkdirWeek 2 · Shell
Make Directory — create a new folder. mkdir -p path/to/dir creates all intermediate directories at once.
mvWeek 2 · Shell
Move or rename a file — the original is gone from the source. mv old.py new.py renames; mv file.py dir/ moves it.
See also: cp
N
NOT NULLWeek 13 · SQLite
A column constraint meaning the field is required — INSERT fails if the value is omitted. Enforces data integrity at the database level.
P
pip freezeWeek 3 · Dev Tools
Lists all installed packages with exact version numbers. Pipe to create a requirements file: pip freeze > requirements.txt
See also: requirements.txt
PRIMARY KEY AUTOINCREMENTWeek 13 · SQLite
SQLite assigns a unique integer ID automatically. Never provide this value on INSERT — leave it out and let the database handle it.
pwdWeek 2 · Shell
Print Working Directory — shows your current location in the file system. Run this any time you're unsure where you are.
R
REALWeek 13 · SQLite
SQLite column type for prices, rates, and decimal numbers. Use for all money values.
See also: INTEGER
relative pathWeek 1 · Shell
A path starting from your current location. Shorter and portable within a project, but breaks if you run the script from a different directory.
See also: absolute path
__repr__Week 10 · OOP
Returns a readable string representation of the object. Called automatically by print(). Show key attributes so you can debug instances easily.
def __repr__(self):
    return f"BusinessRequest({self.title!r}, ${self.amount:.2f})"
requirements.txtWeek 3 · Dev Tools
A text file listing pinned package versions so others can recreate your environment exactly. Generate: pip freeze > requirements.txt. Install: pip install -r requirements.txt
ripgrep (rg)Week 4 · Git
A fast content search tool that respects .gitignore. Much faster than grep for large codebases. rg "BusinessRequest" finds all usages instantly.
rmWeek 2 · Shell
Remove — permanently deletes a file. No undo, no Trash. rm -r folder/ deletes an entire directory recursively.
ruff check .Week 4 · Git
Reports style and logic issues without modifying files. Run before every commit to catch bugs and style violations early.
See also: ruff format .
ruff format .Week 4 · Git
Auto-formats all Python files to a consistent style. Safe to run anytime — changes only whitespace and layout, never logic.
See also: ruff check .
S
selfWeek 10 · OOP
The specific instance a method is being called on. Always the first parameter of any method. self.amount reads that instance's own amount attribute.
See also: instance attribute, method
self.amount — attributeWeek 11 · OOP
Maps to the amount REAL column in SQLite. An instance attribute corresponds directly to a column in the database table.
shellWeek 1 · Shell
The program that reads your typed commands and executes them. This course uses zsh (Z shell), the default on macOS and configured on Linux/WSL.
See also: .zshrc
source .venv/bin/activateWeek 3 · Dev Tools
Activates the virtual environment for the current terminal session. The (.venv) prefix in your prompt confirms it is active. Must be run in every new terminal.
See also: venv
strWeek 5 · Python
String — text enclosed in quotes. Strings are immutable: methods like .upper() return a new string rather than modifying the original.
T
terminal / terminal emulatorWeek 1 · Shell
The window you type commands into. VS Code has one built in (Ctrl+`). On Mac: Terminal.app or iTerm2. On Windows: Windows Terminal running WSL.
See also: shell
TEXTWeek 13 · SQLite
SQLite column type for names, categories, statuses — any string value. SQLite has no fixed-length CHAR; always use TEXT.
total_amount()Week 11 · OOP
A method that maps to SELECT SUM(amount) FROM requests. Aggregates a numeric field across all records in the collection.
touchWeek 2 · Shell
Create an empty file (or update its timestamp if it exists). touch app.py creates an empty app.py ready to edit.
type()Week 5 · Python
Returns the data type of any value. type(42) → <class 'int'>. Essential for debugging type errors.
V
venvWeek 3 · Dev Tools
Virtual environment — an isolated Python + packages installation for one project. Created with python3 -m venv .venv. Prevents project dependencies from conflicting.
See also: source .venv/bin/activate, .gitignore
W
which python3Week 3 · Dev Tools
Shows which Python executable is currently active. Must show .venv/bin/python3 when the venv is activated. If it shows the system Python, activate your venv first.
See also: venv
Z
.zshrcWeek 3 · Dev Tools
The zsh startup script that runs whenever a new terminal session opens. Aliases, functions, and PATH additions defined here are always available.
See also: alias
