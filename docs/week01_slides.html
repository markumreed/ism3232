Files, Folders, Paths, and Directory Trees

Ask students to find their Downloads folder in Finder/Explorer, then ask: "What is the full path to that folder?" Most will not know. Then show it in the terminal with echo ~/Downloads . The gap between clicking and knowing where something lives is the core lesson of this section.

How your computer organizes everything

  • Every file on a computer lives at a specific address called a path .
  • Draw this on the board and have students copy it — understanding the tree structure is a prerequisite for ever…
  • The complete address from the root.
  • The address from wherever you are right now.
Every file on a computer lives at a specific address called a path . A path is just a sequence of folder names that tells the computer how to find a file, starting from the root of the disk ( / on Mac/Linux, C:\ on Windows) or from wherever you are right now. Draw this on the board and have students copy it — understanding the tree structure is a prerequisite for every terminal command they will run this semester. The complete address from the root. Always works regardless of where you are. Starts with / on Mac/Linux. The address from wherever you are right now. Only works if you are in the right place. Much shorter to type.

Steps — Installing and verifying the required stack

  • 1. VS Code — install and open
  • 2. Windows only — WSL + Ubuntu + zsh
  • 3. Mac/Linux — verify zsh is the default shell
  • 4. Configure VS Code's integrated terminal to use zsh
  • 5. Verify the full required stack
Step 1: VS Code — install and open Download from code.visualstudio.com . Install the Python extension (search "Python" in the Extensions panel, install the Microsoft one). Open VS Code and verify the Extensions panel shows Python as installed. Step 2: Windows only — WSL + Ubuntu + zsh Open PowerShell as Administrator and run wsl --install . Restart when prompted. Open the Ubuntu app, complete the UNIX user setup, then run sudo apt update && sudo apt install zsh -y and chsh -s $(which zsh) . Restart the terminal. This is the most failure-prone step — watch the room carefully. Step 3: Mac/Linux — verify zsh is the default shell Mac Catalina and later use zsh by default. Run echo $SHELL — it should return /bin/zsh . If not, run chsh -s /bin/zsh and restart the terminal. Step 4: Configure VS Code's integrated terminal to use zsh Open VS Code → ⌘, (Mac) or Ctrl+, (Windows) → search "terminal default profile" → set to zsh (Mac/Linux) or WSL (Windows). Close and reopen the terminal panel. The prompt should show a % or your zsh theme. Step 5: Verify the full required stack Run each command and confirm the expected output. Everyone in the room must see all four pass before proceeding.

⌨ Verification commands — every student runs these and screenshots the output

# 1. Confirm zsh is running
echo $SHELL
# Expected: /bin/zsh  (or /usr/bin/zsh on some Linux)

# 2. Confirm zsh version
zsh --version
# Expected: zsh 5.x or higher

# 3. Confirm Python 3 is installed
python3 --version
# Expected: Python 3.10 or higher (3.11+ preferred)

# 4. Confirm Git is installed
git --version
# Expected: git version 2.x

# 5. Show current location
pwd
# Expected: your home directory  e.g. /Users/yourname

# 6. List files in current directory
ls
# Expected: Desktop  Documents  Downloads  etc.

Course Folder Structure

code . opens VS Code with the current folder as the workspace. The dot means "here." This is how developers open projects — they navigate to the project folder first, then open the editor. It is different from double-clicking a single file. Emphasize this: the workspace, not the file, is the unit of work.
