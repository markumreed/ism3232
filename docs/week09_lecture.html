Midterm Structure and Weighting
SectionWeightFormatTime estimate

What Is and Is Not Tested
Tested — all of Units 1 and 2

zsh: pwd, ls, cd, mkdir, touch, rm, cat, tree, code .

venv + pip: create, activate, install, freeze, deactivate

.zshrc: alias, mkcd function, source

Search: rg, find, grep, tree

Submission ritual: all 9 steps in order

Git: init, add, commit, push, log, .gitignore

Python: all data types, operators, conditionals, loops, lists, dicts, functions, modules, pytest, tracebacks

NOT tested

❌ OOP (classes, objects, inheritance)

❌ SQL and SQLite

❌ Streamlit

❌ FastAPI

❌ GenAI feature implementation

These are covered in Units 3 and 4 after the midterm.

Sample Questions — Instructor Reference
Code tracing example — students predict the exact output
requests = [
    {"id": 1, "amount": 800,  "status": "Pending"},
    {"id": 2, "amount": 1500, "status": "Pending"},
    {"id": 3, "amount": 300,  "status": "Approved"},
]
total = 0
for r in requests:
    if r["status"] == "Pending":
        total += r["amount"]
print(total)

# Expected output: 2300
Fix-the-bug example — students identify and correct the error
def calculate_tax(amount):
    return amount * 0.7   # BUG: should be 0.07 (7% tax)

def test_calculate_tax():
    assert calculate_tax(100) == 7.0   # this fails

# Fix: change 0.7 to 0.07
Midterm Logistics — Instructor Checklist
Before the exam

☐ Distribute the broken script for the fix-the-bug section in advance (same session).

☐ Ensure every student has their venv activated and pytest/ruff available.

☐ Confirm GitHub access is not required during the exam (no push required).

☐ Clarify AI use policy for the exam: AI tools are not permitted during the midterm.

☐ Tell students to bring their own computer with the full stack configured.

After the exam — looking ahead

Week 10 begins OOP. Students who struggled on the midterm with functions and scope will face additional difficulty with classes and methods. Offer office hours before Week 10 for students who want to consolidate function fundamentals before OOP begins.
