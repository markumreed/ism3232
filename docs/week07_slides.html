Business rules are repeated in multiple places. Changing a threshold means finding every copy. Easy to miss one. Impossible to unit-test any of them. Each rule has a name, is defined once, and is testable. Changing a threshold takes one edit. Every caller receives the update automatically. You cannot write a pytest test for an anonymous block of code. Naming business logic is what makes it testable. Every function in business_rules.py will have a corresponding test in tests/test_business_rules.py .
Section 2

Defining, Calling, Returning

⌨ Function anatomy — def, parameters, return

def calculate_total(price: float, quantity: int) -> float:
    """Return total cost including 7% tax."""
    subtotal = price * quantity
    tax = subtotal * 0.07
    return subtotal + tax

def requires_review(amount: float) -> bool:
    """Return True if amount exceeds the approval limit."""
    return amount > 1000

def get_approval_tier(amount: float) -> str:
    """Return approval routing tier for a given amount."""
    if amount <= 500:    return "auto"
    elif amount <= 2000: return "manager"
    else:               return "director"

# Calling functions
total = calculate_total(450.00, 3)
print(requires_review(1500))    # True
print(get_approval_tier(750))   # "manager"

Section 3

Scope

Variables defined inside a function are local — they do not exist outside it. This prevents functions from accidentally overwriting each other's state. Always pass data in via parameters and get it back via return . Module-level constants (ALL_CAPS) are the exception.

Section 3 — Scope

  • Variables defined inside a function are local — they do not exist outside it.
Variables defined inside a function are local — they do not exist outside it. This prevents functions from accidentally overwriting each other's state. Always pass data in via parameters and get it back via return . Module-level constants (ALL_CAPS) are the exception.

⌨ Scope demonstration

APPROVAL_LIMIT = 1000   # module-level — accessible everywhere

def check_limit(amount):
    result = amount > APPROVAL_LIMIT   # local to this function
    return result

# print(result)  → NameError: name 'result' is not defined
print(check_limit(1500))   # True — the return value is accessible

Section 4

Modules and Required Project Structure

When a file contains functions, other files can use them with import . This is how professional Python projects separate concerns: business rules in business_rules.py , database code in database.py , the main entry point in main.py . Tests always import from the module they test.

from business_rules import calculate_total, requires_review, get_approval_tier

def test_calculate_total_basic():
    assert calculate_total(100.00, 2) == 214.00   # 200 * 1.07

def test_requires_review_true():
    assert requires_review(1500) is True

def test_requires_review_false():
    assert requires_review(500) is False

def test_approval_tier_boundary():
    assert get_approval_tier(500) == "auto"   # exactly at boundary

def test_approval_tier_director():
    assert get_approval_tier(5000) == "director"
