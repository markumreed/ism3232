01

Why Functions?

0–15 min  Why functions?

🧩 Without functions

Business rules are repeated in multiple places. Changing a threshold means finding every copy. Easy to miss one. Impossible to unit-test any of them. Each rule has a name, is defined once, and is testable. Changing a threshold takes one edit. Every caller receives the update…
“Without functions, every business rule is written out in full every time it is needed. If the approval threshold changes from $1,000 to $2,000, you must find and update every copy. Miss one and you have a bug. With a function, you update it in one place and every caller gets the new behaviour.”

🧩 With functions

You cannot write a pytest test for an anonymous block of code. Naming business logic is what makes it testable. Every function in business_rules.py will have a corresponding test in tests/test_business_rules.py . Section 2 — Defining, Calling, Returning
“You cannot write a pytest test for an anonymous block of code. Naming the logic is what makes it testable. Every function in business_rules.py will have a corresponding test. If you cannot test it, you cannot trust it.”

def calculate_total(price: float, quantity: int) -> float:
    """Return total cost including 7% tax."""
    subtotal = price * quantity
    tax = subtotal * 0.07
    return subtotal + tax

def requires_review(amount: float) -> bool:
    """Return True if amount exceeds the approval limit."""
    return amount > 1000

def get_approval_tier(amount: float) -> str:
    """Return approval routing tier for a given amount."""
    if amount <= 500:    return "auto"
    elif amount <= 2000: return "manager"
    else:               return "director"

Defining, Calling, Returning

15–35 min  Define, call, return

🧩 Functions are the unit of testability

Variables defined inside a function are local — they do not exist outside it. This prevents functions from accidentally overwriting each other's state. Always pass data in via parameters and get it back via return . Module-level constants (ALL_CAPS) are the exception. Scope…
“Notice that none of these functions contain a print(). They return values. main.py calls them and prints the results. This separation — logic in functions, display in main — is how professional code is structured and how tests work.”

🧩 Local vs module-level scope

When a file contains functions, other files can use them with import . This is how professional Python projects separate concerns: business rules in business_rules.py , database code in database.py , the main entry point in main.py . Tests always import from the module they…
“A variable defined inside a function is local — it does not exist outside that function. This is a feature, not a limitation. It means functions cannot accidentally overwrite each other's data. Always pass data in via parameters and get it back via return.”

APPROVAL_LIMIT = 1000   # module-level — accessible everywhere

def check_limit(amount):
    result = amount > APPROVAL_LIMIT   # local to this function
    return result

# print(result)  → NameError: name 'result' is not defined
print(check_limit(1500))   # True — the return value is accessible
