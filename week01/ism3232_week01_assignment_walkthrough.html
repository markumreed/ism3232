Requirement-by-Requirement Walkthrough
1
Verify all six tools from the zsh terminal

Students must run all six commands and show their output in a single screenshot. The terminal must show the % prompt (zsh), not $ (bash) or PS1 (PowerShell).

🔍 What the grader checks
All six commands visible in one screenshot: echo $SHELL, zsh --version, python3 --version, git --version, pwd, ls
Each command shows expected output directly below it (no blank output)
Prompt character is % throughout — not $ or >
Python version is 3.10 or higher; Git version is 2.x or higher
✅ Passing screenshot
All six commands run in sequence with output. Prompt shows %. python3 --version returns Python 3.11.x.
❌ Failing screenshot
Student shows $ prompt (bash). Or only 4 of 6 commands visible. Or screenshot is a phone photo.
⚠ Common mistakes at this step
Phone photo of the screen instead of a digital screenshot — blurry and unreadable
Bash prompt ($ instead of %) — requires the student to switch shell and resubmit
Only some commands shown — must be all six in one continuous session
python --version instead of python3 --version — may show Python 2
2
Create the course folder structure

Student must show VS Code open with the ism3232/ folder as the workspace (not a single file). The Explorer panel must show all eight module subfolders.

🔍 What the grader checks
VS Code is open (not Terminal, not Finder)
Explorer sidebar is visible on the left
All 8 folders visible: module01_setup through module07_final_project, data, screenshots
ism3232 appears as the workspace root in the Explorer header
⚠ Common mistakes at this step
Student opened a single file (code hello_ism3232.py) instead of the folder (code .) — Explorer shows no subfolders
Only some module folders created — check the count
Folders created in the wrong location (Desktop instead of ~/ism3232)
3
Write and run hello_ism3232.py

Script must be in module01_setup/, not the root ism3232/ folder. Output must match exactly — all five lines.

🔍 What the grader checks
Script is at module01_setup/hello_ism3232.py (check path in terminal)
All five print lines output correctly in the terminal
Script was run with python3 hello_ism3232.py (not by clicking Run in VS Code only)
Author comment at top with student name
✅ Correct output
Hello, ISM3232! Course: Business Application Development Expected weekly hours: 6 Environment verified. Week 1 complete.
❌ Wrong output
Only 'Hello, ISM3232!' printed. Or weekly_hours shows wrong value (not 6). Or script in wrong folder.
⚠ Common mistakes at this step
Script placed in ~/ism3232/ instead of ~/ism3232/module01_setup/ — full credit requires correct location
weekly_hours computed incorrectly — check if student typed 2 * 2 instead of credit_hours * 2
Screenshot does not show the run command, only the output
4
Create README.md

Check the uploaded file directly. Must have all four required sections: name/date, version table, description, and AI use statement.

🔍 What the grader checks
Student name and today's date present
Version table has three rows (zsh, Python 3, Git) with actual version numbers filled in — not empty
2-3 sentence description of what was done this week
AI use statement explicitly says 'I did not use AI for this assignment' or similar
⚠ Common mistakes at this step
Version table is empty or has placeholder text — student must fill in their actual versions
AI use statement missing entirely — required for full credit
README.md submitted as .txt or .docx — must be Markdown (.md)
Known Common Mistakes
⚠ Known common mistakes (from the assignment page)
Taking a phone photo of the screen instead of a digital screenshot — use the OS screenshot tool
Submitting a bash or PowerShell screenshot — the prompt must show % (zsh)
hello_ism3232.py placed in the root ism3232/ folder instead of module01_setup/
Missing or empty README.md — every field must be filled in
Script output that does not match — re-run and re-screenshot after fixing

Grading Flow
Open the Canvas submission. Check all three screenshots are present — if missing, stop and award 0 for missing items before reviewing content.
Screenshot 1: Verify % prompt, all 6 commands, expected output. Deduct 3 pts per missing command (max 15). Deduct all 5 terminal points if $ or > prompt.
Screenshot 2: Confirm VS Code open with workspace and all 8 module folders visible in Explorer.
Screenshot 3: Check the 5-line output matches exactly. Verify file is in module01_setup/ (check path in the screenshot).
Open hello_ism3232.py upload — check student name in comment header.
Open README.md upload — check all four sections: name, date, version table (with values), description, AI use statement.
Total and return. Add a comment if zsh was wrong — student needs to fix this before Week 2.
💡
Additional grading notes
Week 1 grades are often the first grades students see in the course. Leave an encouraging comment even for partial submissions. If a student could not complete setup, flag them for office hours immediately — they will fall behind quickly without a working environment.
💬
Rubric discussion
The 15-point screenshot requirement (all six verification commands) is the highest-weight item. Award 12/15 if all six are present but the student is in bash not zsh — and require a resubmission. Award 0/15 if fewer than four commands are visible. The zsh terminal check (5 pts) is binary: either % is visible or it is not.
