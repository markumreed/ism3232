Question-by-question walkthrough
1
Multiple Choice — Auto-graded
What does the % character at the end of your terminal prompt indicate?
A.You are running the bash shell○
⚠ $ is the bash prompt character. Many students arrive with bash habits. If they see $ in their terminal, they are in the wrong shell and need to switch before anything else.
B.You are running the zsh shell✓
C.You need to install Python○
⚠ Python is already installed separately. The prompt character has nothing to do with Python installation status.
D.You are in the root directory○
⚠ The root directory (/) has its own prompt conventions on some systems but the % character specifically identifies zsh, not location.
💬 Discussion — why B is correct

The % is the default prompt character for zsh. Bash uses $. If a student sees $ they are in the wrong shell.

2
Multiple Choice — Auto-graded
Which command shows your current location in the filesystem?
A.ls○
⚠ ls lists the contents of a directory — it tells you WHAT is here, not WHERE you are.
B.cd○
⚠ cd changes your location but does not print it. It produces no output on its own.
C.pwd✓
D.tree○
⚠ tree visualises the directory structure but does not print the current path.
💬 Discussion — why C is correct

pwd stands for Print Working Directory. It shows the full absolute path of your current location.

3
Multiple Choice — Auto-graded
What is the home directory shortcut symbol in zsh?
A./○
⚠ / is the root of the entire filesystem — the top level, not your home directory.
B..○
⚠ . (single dot) refers to the current directory, not the home directory.
C.~✓
D...○
⚠ .. (double dot) refers to the parent directory — one level up.
💬 Discussion — why C is correct

The tilde ~ expands to your full home directory path (e.g. /Users/yourname). It can be used in any path: ~/ism3232

4
Multiple Choice — Auto-graded
What does "code ." do when typed in the terminal?
A.Creates a new file called code○
⚠ There is no file called 'code'. The code command is added to PATH when you install the VS Code CLI.
B.Opens the current folder as a VS Code workspace✓
C.Opens VS Code settings○
⚠ Cmd+P / Ctrl+P opens the Quick Open file search, not settings.
D.Lists all Python files○
⚠ ls lists files — it does not open an editor.
💬 Discussion — why B is correct

The dot . means "current directory". code . opens VS Code with the current folder as the project workspace.

5
Multiple Choice — Auto-graded
Which command would you run to confirm Python 3 is installed and check its version?
A.python --version○
⚠ python --version may point to Python 2 on older systems. Always use python3 explicitly.
B.python3 --version✓
C.pip --version○
⚠ pip --version shows pip's version, not Python's.
D.py --version○
⚠ py is a Windows-specific launcher, not a cross-platform command.
💬 Discussion — why B is correct

python3 --version prints the installed Python 3 version. On many systems "python" may point to Python 2 — always use python3 in this course.

6
Multiple Choice — Auto-graded
What does ls -la show that plain ls does not?
A.More colours○
⚠ ls alone shows visible files without permissions, file sizes, or modification dates.
B.Hidden files (starting with .) and file permissions✓
C.Only Python files○
⚠ ls -la does show .py files but it shows ALL files — the key addition is hidden files and permissions.
D.The directory tree○
⚠ tree shows the directory structure, not file details like permissions.
💬 Discussion — why B is correct

The -la flags add: -l (long format with permissions, size, date) and -a (all files including hidden ones like .venv and .gitignore).

B..python○
C..py✓
