Section 1 — Variables and Data Types
1

Variables and Data Types 0–15 min

Lecture · No typing yet · Business examples throughout

🗣 Say — opening
“We are now in Unit 2. The terminal, venv, and Git are your baseline — use them for everything from here on. Today we start Python. Not syntax for its own sake — Python as a tool for business logic.”
🗣 Say — variables
“A variable is a named container. In business scripts, variables hold things like prices, quantities, statuses, and request amounts. The name should describe what it contains. request_amount tells me something. x tells me nothing.”
✏ Write the four types on board — with business examples
str: product_name, status, category · int: quantity, employee_id · float: price, tax_rate · bool: is_approved, is_active
💡 Common type confusion to address now
Show students that '3' and 3 are different types. Run type('3') and type(3). Then try '3' + 3 — it throws a TypeError. This is the error they will hit most in Week 5 if they do not understand types.
Section 2 — Operators
2

Operators 15–35 min

Live demo · Students type along

🗣 Say — framing
“Operators let you compute and compare. Every business rule you will ever write uses comparison and logical operators. An approval threshold is a comparison. An eligibility check is a comparison combined with a logical operator.”
Demo each category with business context
# Arithmetic
total = price * quantity
tax   = total * 0.07
avg   = total / quantity   # always float
whole = 105 // 10          # floor division -> 10
rem   = 105 % 10           # modulo -> 5

# Comparison (result is always bool)
total > 1000
status == 'Pending'
quantity != 0

# Logical
total > 500 and is_approved
total > 1000 or quantity > 10
not is_approved
💬 Pause and ask
“What type does a comparison operator always return?” Look for: bool. Then ask: “What is 3 > 2?” → True. “What is 3 > 5?” → False. Quick but important.
Section 3 — Input, Output, and f-strings
3

Input, Output, and f-strings 35–55 min

Live demo · Show the input() type bug explicitly

🗣 Say — f-strings
“An f-string lets you embed variable values directly in a string. The f before the opening quote tells Python to evaluate anything inside curly braces. The :.2f format spec gives you exactly two decimal places — essential for displaying currency.”
⚠
Demo the input() type bug — do not skip this
This is the most common Week 5 bug. Run: amount = input('Enter amount: '), type 1200, then try amount + 100. It throws a TypeError. Say: “input() always returns a string. Always. You must convert it: float(input(...)).” Show the fix.
🗣 Say — after showing the bug
“Every time you call input(), ask yourself: do I need a number? If yes, wrap it in int() or float() immediately. Do not store the raw string and convert it later — convert it on the same line.”
Section 4 — Live Coding — Business Data Script
4

Live Coding — Business Data Script 55–70 min

Type together · No copy-paste

🗣 Say before starting
“We are going to write a business script together. Every line is deliberate. I will explain what each section does before we type it.”
week5_business_data.py — type together
requester      = 'Jordan'
category       = 'Equipment'
unit_price     = 450.00
quantity       = 3
approval_limit = 1000.00

subtotal       = unit_price * quantity
tax            = subtotal * 0.07
total          = subtotal + tax
over_limit     = total > approval_limit

print('=== Purchase Request Summary ===')
print(f'Requester:  {requester}')
print(f'Total:      ${total:.2f}')
print(f'Needs review: {over_limit}')
💬 Debrief discussion after running
Ask: “What type is over_limit? How do you know? What would happen if total were exactly 1000.00?” Students should reason about comparison operator output and boundary conditions — the same thinking they will use for business rules in Week 6.
