Overview
Build a complete business data script for a domain of your choosing. The script must use all four Python data types, compute derived values with arithmetic operators, apply a comparison operator to produce a boolean result, use f-strings with currency formatting, and read at least one value from the user using input() with type conversion. Test your calculations with pytest.
Requirements
1
Choose a business domain and define variables across all four types

Choose any business scenario (purchase requests, employee records, inventory, invoices, etc.). Define variables using all four types:

• str — at least two string variables (name, category, status, etc.)

• int — at least one integer (quantity, count, ID, etc.)

• float — at least one decimal (price, rate, discount, etc.)

• bool — at least one boolean derived from a comparison (e.g. total > 1000)

2
Compute at least three derived values using arithmetic operators

At minimum, compute: a multiplication, an addition, and a percentage (multiplication by a rate). For example: subtotal, tax, and total. Use :.2f formatting in all f-strings that display currency.

3
Display a formatted business summary using f-strings

Print a labelled, formatted summary of your business record. Every currency value must be formatted to two decimal places. The summary must include at least four lines of output.

4
Read one value from the user with input() and type convert it

Use input() to read at least one value from the user. Convert it immediately using int() or float(). Use the converted value in a calculation and print the result.

5
Write at least three pytest tests

In module04_programming/tests/test_week5.py, write at least three tests verifying your calculations. Tests must use assert statements and import nothing from your script (write the calculations directly in the test functions).

6

⚠ Common mistakes
  • Using input() without converting: age = input(...) then age + 1 — this crashes with a TypeError
  • Boolean that is always True: is_approved = True — it must be derived from a comparison like total > 1000
  • Currency displayed without :.2f: printing $1450.6 instead of $1,450.60
  • Tests that assert obvious constants: assert 1 == 1 — tests must verify your actual calculations
