The four core data types

Every value in Python has a type. The type determines what operations are legal. These four cover nearly all business data.

Business examples of all four types
# str — text
product_name = 'Ergonomic Office Chair'
category     = 'Furniture'
status       = 'Pending'

# int — whole numbers
quantity    = 12
employee_id = 1047

# float — decimal numbers
unit_price = 449.99
tax_rate   = 0.07
discount   = 0.15

# bool — True or False, always DERIVED from a comparison
total          = unit_price * quantity
requires_review = total > 1000    # True (5399.88 > 1000)
over_budget     = total > 5000    # True

# Inspect any type
print(type(product_name))    # <class 'str'>
print(type(quantity))        # <class 'int'>
print(type(unit_price))      # <class 'float'>
print(type(requires_review)) # <class 'bool'>
⚠ Booleans must be computed

Writing is_approved = True is hardcoding — it reflects no real business rule. Always derive booleans from a comparison: is_approved = amount <= LIMIT.

Operators

All operators with business examples
price = 449.99
qty   = 17

# Arithmetic
subtotal   = price * qty           # 7649.83
tax        = subtotal * 0.07       # 535.49
total      = subtotal + tax        # 8185.32

# Floor division — whole boxes only
boxes    = qty // 4               # 4 (17 ÷ 4, drop the remainder)
leftover = qty % 4               # 1 (remainder of 17 ÷ 4)

# Comparison — always returns a bool
print(total > 5000)    # True
print(qty == 20)       # False
print(price != 0)      # True

# Logical — combine conditions
needs_approval = total > 1000 and discount < 0.20
print(needs_approval)  # True
⚠ // and % are not / and percent

17 // 5 is 3, not 3.4 (floor division drops the decimal). 17 % 5 is 2 (the remainder), not a percentage. These two operators trip students up every week.

input(), print(), and f-strings

The most important bug to understand this week
# This crashes:
amount = input('Enter amount: ')   # input() returns a STRING
total  = amount + 100              # TypeError: can't add str and int

# The fix — wrap input() immediately
amount = float(input('Enter amount: '))
total  = amount + 100              # works: float + int = float
input() always returns str

Every single time. Even when the user types a number. Wrap it immediately: float(input(...)) or int(input(...)). This prevents the most common Week 5 TypeError.

f-string formatting for business output
product = 'Standing Desk'
price   = 1249.50
qty     = 7
total   = price * qty

print(f'Product:  {product}')
print(f'Price:    ${price:.2f}')        # 2 decimal places
print(f'Total:    ${total:,.2f}')        # comma separator + 2 decimals
print(f'Tax rate: {0.072:.1%}')          # percentage: 7.2%
print(f'Boxes:    {qty // 4}')           # expression inside { }

Key terms this week

strString — text enclosed in quotes.
intInteger — a whole number with no decimal.
floatFloating-point — a number with a decimal component.
boolBoolean — True or False. Always derived from a comparison.
type()Returns the data type of any value.
//Floor division — divides and rounds down to the nearest integer.
%Modulo — returns the remainder after division.
f-stringA string prefixed with f that evaluates expressions inside {curly braces}.
:.2ff-string format spec: 2 decimal places.
:,.2ff-string format spec: comma separator and 2 decimal places.
