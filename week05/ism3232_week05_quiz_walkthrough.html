Question-by-question walkthrough
1
Multiple Choice — Auto-graded
What is the data type of: is_approved = False
A.str○
⚠ str is for text like names and categories. False is not text — it is a boolean keyword.
B.int○
⚠ int is for whole numbers. False is not a number, though Python treats True/False as 1/0 internally.
C.float○
⚠ float is for decimal numbers. False is a boolean literal.
D.bool✓
💬 Discussion — why D is correct

False (capital F) is a boolean literal. Python booleans are True and False. Lowercase true would be a NameError.

2
Multiple Choice — Auto-graded
What type does Python assign to: price = 49.99
A.int○
⚠ int is for whole numbers with no decimal. 49.99 has a decimal point, making it a float.
B.str○
⚠ str would require quotes: '49.99'. Without quotes, Python reads it as a number.
C.float✓
D.double○
⚠ double is a type in Java and C but does not exist in Python. Python uses float for all decimals.
💬 Discussion — why C is correct

Any number with a decimal point is a float. double does not exist in Python. int is for whole numbers only.

3
Multiple Choice — Auto-graded
What type does input() ALWAYS return, regardless of what the user types?
A.int○
⚠ int would require explicit conversion: int(input()). Raw input() never returns int.
B.float○
⚠ float would require explicit conversion: float(input()). Raw input() never returns float.
C.str✓
D.depends on what the user types○
⚠ This is the most dangerous misconception — input() does NOT adapt to what the user types. It always returns str.
💬 Discussion — why C is correct

input() ALWAYS returns a string. Even if the user types 42, Python receives the string "42". You must convert it explicitly with int() or float().

4
Multiple Choice — Auto-graded
What is the result of 17 // 5?
A.3.4○
⚠ 3.4 would be the result of 17 / 5 (true division), not 17 // 5 (floor division).
B.3✓
C.2○
⚠ 2 is the remainder (modulo), not the quotient. That would be 17 % 5.
D.4○
⚠ 4 would require rounding up — floor division always rounds DOWN.
💬 Discussion — why B is correct

// is floor (integer) division. 17 // 5 = 3 with remainder 2. The decimal is discarded (floored), not rounded.

5
Multiple Choice — Auto-graded
What is the result of 17 % 5?
A.3.4○
⚠ 3.4 is the true division result (17/5). Modulo gives the remainder, not the quotient.
B.3○
⚠ 3 is the floor division result (17//5). Modulo gives the remainder.
C.2✓
D.4○
⚠ 4 is not achievable from either operation on 17 and 5.
💬 Discussion — why C is correct

% is the modulo operator — it returns the remainder after division. 17 = 5*3 + 2, so the remainder is 2.

6
Multiple Choice — Auto-graded
What does :.2f do inside an f-string?
A.Rounds to 2 significant figures○
⚠ Significant figures would require round() with a different format specifier.
B.Formats the number with exactly 2 decimal places✓
C.Converts to a 2-digit integer○
⚠ :.2f does not convert to integer — it keeps the decimal format.
D.Adds a currency symbol○
⚠ :.2f does not add a currency symbol — you must add $ explicitly before the value.
💬 Discussion — why B is correct

:.2f is a format specifier meaning "fixed-point notation with 2 decimal places". f"{1450.0:.2f}" produces "1450.00".

7
Multiple Choice — Auto-graded
What is the value of over_limit after: total = 450 * 3 and over_limit = total > 1000
A.1350○
⚠ 1350 is the raw value of total — the comparison has not been evaluated yet at that point.
B.True✓
C.False○
⚠ False would require total <= 1000. Since total is 1350, the comparison is True.
D."True"○
⚠ 'True' (with quotes) would be a string. Comparison operators return a boolean, not a string.
💬 Discussion — why B is correct

total = 450 * 3 = 1350. 1350 > 1000 is True. A comparison operator always returns a boolean.

8
Multiple Choice — Auto-graded
Which line correctly reads a number from the user and stores it as a float?
A.amount = input("Enter amount: ")○
⚠ amount = input() returns a string. Passing a string to float() later is acceptable but the issue is the conversion must happen BEFORE arithmetic.
B.amount = float("Enter amount: ")○
⚠ float('Enter amount: ') would try to convert the prompt string to a float — that is a ValueError.
C.amount = float(input("Enter amount: "))✓
D.amount = int(float(input("Enter amount: ")))○
⚠ int(float(input())) double-converts — works but unnecessarily complex. float(input()) is sufficient for currency amounts.
💬 Discussion — why C is correct

float(input(...)) converts the string returned by input() to a float on the same line. This is the required pattern for numeric input.
