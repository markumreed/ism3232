Requirement-by-Requirement Walkthrough
1
Create and activate the virtual environment

Screenshot must show the (.venv) prompt prefix AND the output of which python3 pointing inside the .venv folder.

🔍 What the grader checks
(.venv) visible in the prompt — this is the confirmation of activation
which python3 output includes .venv/bin/python3 in the path
Environment is named .venv (dot prefix — hidden folder)
✅ Correct
(.venv) student@MacBook module02_zsh % which python3 /Users/student/ism3232/module02_zsh/.venv/bin/python3
❌ Wrong
Prompt shows plain % with no (.venv) prefix — venv was created but not activated. Or which python3 points to /usr/bin/python3 (system Python).
⚠ Common mistakes at this step
Venv named 'venv' or '.env' instead of '.venv' — partial credit; note the naming convention
(.venv) visible in prompt but which python3 still points to system — venv not correctly activated
Student shows pip list from the system Python (many packages) not the venv (minimal)
2
Install packages and freeze

Two pieces must be visible: the pip install output and the cat requirements.txt showing actual content.

🔍 What the grader checks
pip install pytest ruff output visible (or equivalent packages listed)
pip freeze > requirements.txt command run
cat requirements.txt shows content — not empty
requirements.txt has both pytest and ruff (and their dependencies) pinned with ==
⚠ Common mistakes at this step
pip freeze run without activating the venv — requirements.txt will contain system packages, not just the project ones
requirements.txt is empty — student ran pip freeze before pip install
pip install run as sudo or with --break-system-packages without venv — this installs globally
3
Create .gitignore

Open the uploaded .gitignore file and verify content. Also check the ls -la screenshot.

🔍 What the grader checks
.venv/ on its own line (with trailing slash)
__pycache__/ on its own line
ls -la screenshot shows both .venv and .gitignore as hidden items
⚠ Common mistakes at this step
.gitignore has 'venv/' without the dot — misses the actual hidden folder
.gitignore has the right content but was not committed — check the screenshot
Student created .gitignore.txt — the extension must be absent
4
Add aliases to .zshrc

All nine aliases and the mkcd function must be in .zshrc AND tested. The screenshot must show them working after source.

🔍 What the grader checks
All 9 aliases present: ll, c, py, gs, ga, gcmsg, gp, gl, tree2
mkcd() function defined
source ~/.zshrc run before testing
At least ll and mkcd demonstrated working in the terminal
💡 Spot-check method
Ask: does typing ll show ls -la output? Does mkcd testdir create and enter a directory? If yes, award full marks without checking every alias individually.
⚠ Common mistakes at this step
Aliases added but source ~/.zshrc not run — they will not work in the current session
mkcd function missing — it is required separately from the aliases
Only some aliases present — partial credit; count what is there
5
Deactivate cleanly

Check that the student ran deactivate and the prompt returned to normal (no (.venv) prefix).

🔍 What the grader checks
deactivate command visible in terminal
Prompt no longer shows (.venv) after deactivation
which python3 after deactivation points back to system Python
Known Common Mistakes
⚠ Known common mistakes (from the assignment page)
Naming the venv anything other than .venv — the dot prefix is required
Running pip install before activating the venv — always activate first
Committing .venv/ to Git — it must be in .gitignore before any push
Not sourcing after editing .zshrc — aliases only work after source ~/.zshrc

Rubric discussion
The venv activation screenshot (15 pts) is the highest-value item. A student who created the venv but never activated it has not demonstrated the skill — award 8/15 maximum. The aliases section (15 pts) can be verified by spot-checking ll and mkcd — do not check all nine individually.
