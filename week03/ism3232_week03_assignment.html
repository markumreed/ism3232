Overview
This assignment establishes the environment management habits you will use for every remaining assignment. You must create a virtual environment, install the required tools, freeze dependencies, configure your shell, and verify that nothing is tracked by Git that should not be.
Requirements
1
Create, activate, and verify a virtual environment

Inside module02_zsh/, create a virtual environment named .venv. Activate it. Verify you are inside it with which python3. The prompt must show (.venv).

cd ~/ism3232/module02_zsh
python3 -m venv .venv
source .venv/bin/activate
which python3   # must point inside .venv/

Screenshot required: the terminal showing (.venv) in the prompt and the which python3 output.

2
Install pytest and ruff, then freeze
pip install pytest ruff
pip list                      # confirm both appear
pip freeze > requirements.txt
cat requirements.txt          # verify it was written

Screenshot required: pip install output and cat requirements.txt output.

3
Create .gitignore

Create module02_zsh/.gitignore containing at minimum:

.venv/
__pycache__/
*.pyc

Verify with ls -la that both .venv/ and .gitignore are visible. Screenshot required.

4
Add all required aliases to ~/.zshrc

Open ~/.zshrc and add the nine required aliases and the mkcd() function. After saving, run source ~/.zshrc. Test each alias:

Required aliases
alias ll='ls -la'
alias c='clear'
alias py='python3'
alias gs='git status'
alias ga='git add .'
alias gcmsg='git commit -m'
alias gp='git push'
alias gl='git log --oneline'
alias tree2='tree -L 2'

mkcd() { mkdir -p "$1" && cd "$1" }

Screenshot required: each alias and mkcd tested in the terminal after source ~/.zshrc.

5
Deactivate cleanly

Run deactivate. Confirm the (.venv) prefix is gone. Run which python3 to confirm you are back to the system Python.

⚠ Common mistakes
  • Naming the venv anything other than .venv — the dot prefix is required
  • Running pip install before activating the venv — always activate first
  • Committing .venv/ to Git — it must be in .gitignore before any push
  • Not sourcing after editing .zshrc — aliases only work after source ~/.zshrc
