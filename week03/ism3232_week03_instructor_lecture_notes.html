Section 1 — Why Virtual Environments?
1

Why Virtual Environments? 0–15 min

Concept lecture · No typing yet · Use the private room analogy

🗣 Say — the problem
“Without a virtual environment, every Python package you install goes into one shared global installation. This works fine for one project. It breaks when Project A needs requests==2.28 and Project B needs requests==2.31. They cannot both be installed globally at once. One of them breaks.”
✏ Draw on board — the private room mental model
[ Global Python ] ← fragile, shared, conflicts possible
[ Project A: .venv ] ← private room, isolated packages
[ Project B: .venv ] ← separate private room, no conflict
🗣 Say — the mental model
“A venv is like a private room for each project. The room has its own Python and its own shelves of packages. When you activate it, you walk in. When you deactivate, you walk out. Other rooms are completely unaffected.”
🗣 Say — this course expectation
“In this course: every assignment uses a venv. The pre-submission ritual requires it to be active before you run ruff or pytest. Starting now, this is non-negotiable.”
Section 2 — Creating and Managing Virtual Environments
2

Creating and Managing Virtual Environments 15–40 min

Live demo · Students follow along step by step

🗣 Say before starting
“Watch me do the full workflow once. Then you repeat every step in your own terminal.”
1
Create the environment
Navigate to module02_zsh/ first. Then: python3 -m venv .venv. The dot in .venv makes it a hidden folder — it will not clutter your ls output but it will show in ls -la.
2
Activate it
source .venv/bin/activate. The prompt changes to show (.venv). Point to this: “That label is your confirmation. If you do not see it, you are not inside the venv.”
3
Verify you are inside
which python3 — should point inside .venv/bin/. pip list — should show only a few base packages.
4
Install required tools
pip install pytest ruff. These are required for every assignment from here on.
5
Freeze dependencies
pip freeze > requirements.txt. Then cat requirements.txt to verify it was written.
6
Deactivate
deactivate. Prompt returns to normal. Run which python3 again — now points back to system Python.
⚠
Never commit .venv to GitHub
This is the single most common Git mistake in this course. Show students how to create .gitignore with .venv/ on the first line right now, before they push anything. We will cover Git properly in Week 4 but plant the seed today.
📸
Screenshot moment — tell students: “Take a screenshot showing venv activation (prompt with (.venv)), pip install output, and pip freeze > requirements.txt. This is Assignment 3 deliverable #1 and #2.”
Section 3 — .zshrc Customisation — Aliases and Functions
3

.zshrc Customisation — Aliases and Functions 40–55 min

Live demo · Students edit their own .zshrc

🗣 Say — what .zshrc is
“.zshrc is a hidden configuration file in your home directory. zsh reads it every time a new terminal opens. Think of it as a startup script. Anything defined inside it — aliases, functions — is automatically available in every session.”
Open and edit — demo each alias
code ~/.zshrc

# Add these lines:
alias ll='ls -la'
alias c='clear'
alias py='python3'
alias gs='git status'
alias ga='git add .'
alias gcmsg='git commit -m'
alias gp='git push'
alias gl='git log --oneline'
alias tree2='tree -L 2'

mkcd() { mkdir -p "$1" && cd "$1" }

# Apply without restarting:
source ~/.zshrc
🗣 Say after sourcing
“Test each alias right now. Type ll. Type mkcd testdir and watch it create and enter the folder in one command. These are not just shortcuts — they are muscle memory you will build over the next 14 weeks.”
💡 Common mistake
Students add aliases with double quotes instead of single quotes inside the alias value. alias ll="ls -la" works but alias ll=ls -la (no quotes) does not. Show the error if it comes up.
⚠
Assignment 3 walkthrough
Three screenshots: venv activation + pip install, pip freeze output, aliases tested. Two files: .gitignore (with .venv/ and __pycache__/) and requirements.txt. Remind students that requirements.txt must be generated from inside an active venv, not from the global Python environment.
