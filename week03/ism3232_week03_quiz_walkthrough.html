How to use this walkthrough
Week 3 introduces two concepts that recur in every remaining assignment: virtual environments and shell customisation. This walkthrough gives detailed explanations for every question. The venv activation question (Q3) and the gitignore question (Q6) are particularly important — students who miss these will make mistakes on every subsequent submission.
Question-by-question walkthrough
1
Multiple Choice — Auto-graded
What is the purpose of a Python virtual environment?
A.To make Python run faster○
⚠ venvs do not make Python faster — they isolate packages, not the interpreter.
B.To give each project its own isolated set of packages✓
C.To encrypt your code○
⚠ venvs do not encrypt code. That would be a different tool entirely.
D.To compress Python files○
⚠ venvs do not compress files. They create a folder structure with a Python interpreter and package directory.
💬 Discussion — why B is correct

A venv isolates each project's packages from the global installation and from other projects, preventing version conflicts.

2
Multiple Choice — Auto-graded
What command creates a virtual environment named .venv?
A.venv create .venv○
⚠ 'venv create' is not valid Python syntax.
B.pip install venv○
⚠ pip install venv would try to install a package named venv, not create an environment. venv is a built-in module.
C.python3 -m venv .venv✓
D.make venv .venv○
⚠ 'make venv' is not a Python command.
💬 Discussion — why C is correct

python3 -m venv .venv uses the built-in venv module (-m) to create a new environment in a folder named .venv.

3
Multiple Choice — Auto-graded
Which command activates the virtual environment on Mac/Linux?
A.activate .venv○
⚠ 'activate .venv' is not a valid command. The activation script is at .venv/bin/activate.
B.source .venv/bin/activate✓
C.venv start○
⚠ venv does not have a 'start' subcommand.
D.python3 activate○
⚠ python3 activate is not a command. Python runs .py files, not shell scripts.
💬 Discussion — why B is correct

source .venv/bin/activate runs the activation script in the current shell. After activation the prompt changes to show (.venv).

4
Multiple Choice — Auto-graded
How do you confirm the venv is active?
A.Run python3 --version○
⚠ python3 --version shows the Python version but does not tell you which Python — inside or outside the venv.
B.Check that (.venv) appears in the terminal prompt✓
C.Run pip --check○
⚠ 'pip --check' checks for broken packages, not venv status.
D.Open VS Code○
⚠ VS Code's display does not change when a venv is activated in the terminal.
💬 Discussion — why B is correct

The (.venv) prefix in the prompt is the visual confirmation. You can also run "which python3" and confirm it points inside .venv/bin/.

5
Multiple Choice — Auto-graded
Why should .venv/ always be in .gitignore?
A.To keep the repo small and platform-independent — requirements.txt recreates it✓
B.Because .venv is a secret○
⚠ .venv does not contain secrets.
C.Because GitHub does not support Python○
⚠ GitHub supports Python files perfectly well. The issue is repository size and portability.
D.Because VS Code cannot read it○
⚠ VS Code can read .venv files fine. The gitignore rule is about collaboration and portability.
💬 Discussion — why A is correct

The .venv folder is large, platform-specific, and unnecessary to share. requirements.txt is all that is needed to recreate it on any machine.

6
Multiple Choice — Auto-graded
What does running "source ~/.zshrc" do?
A.Deletes the .zshrc file○
⚠ 'source' runs the file; it does not delete it.
B.Applies changes made to .zshrc to the current terminal session✓
C.Creates a backup of .zshrc○
⚠ 'source' does not create a backup.
D.Opens .zshrc in VS Code○
⚠ 'source' runs the file — it does not open it in an editor (that would be 'code ~/.zshrc').
💬 Discussion — why B is correct

source runs the file in the current shell session. Without sourcing, changes to .zshrc only take effect in new terminal windows.

7
Multiple Choice — Auto-graded
What does the alias "alias ll='ls -la'" do?
A.Renames the ls command permanently○
⚠ Aliases are shortcuts — they do not rename or remove existing commands. ls still works after defining the ll alias.
B.Creates a shortcut so typing ll runs ls -la✓
C.Makes ll a variable○
⚠ Aliases are not variables. They are command shortcuts evaluated by the shell.
D.Deletes the ls command○
⚠ Defining an alias does not affect the underlying command.
💬 Discussion — why B is correct

An alias creates a shortcut name for a longer command. After sourcing ~/.zshrc, typing ll is equivalent to typing ls -la.

8
Multiple Choice — Auto-graded
A colleague clones your repo on a new machine. What command recreates your exact dependencies?
A.pip install --all○
⚠ 'pip install --all' is not a valid pip command.
B.pip sync○
⚠ 'pip sync' is from pip-tools, not standard pip.
C.pip install -r requirements.txt✓
D.pip freeze○
⚠ pip freeze GENERATES requirements.txt — it does not install from it.
💬 Discussion — why C is correct

pip install -r requirements.txt installs all packages listed in requirements.txt with their exact versions.

9
Multiple Choice — Auto-graded
What does the mkcd() function do?
A.Makes a file and copies it○
⚠ mkcd copies nothing. It creates and enters a directory.
B.Creates a directory and immediately enters it✓
C.Lists files and sorts by date○
⚠ mkcd does not list files or sort by date.
D.Copies and renames a file○
⚠ mkcd does not copy anything.
💬 Discussion — why B is correct

mkcd() { mkdir -p "$1" && cd "$1" } combines mkdir and cd. The && means: run cd only if mkdir succeeded.

14
Multiple Choice — Auto-graded
Which file stores shell aliases and functions that persist across all terminal sessions?
A..bash_profile○
B.config.zsh○
C.~/.zshrc✓
D..aliases○
💬 Discussion — why C is correct

~/.zshrc is the zsh startup configuration file. zsh reads it every time a new terminal opens.

15
Multiple Choice — Auto-graded
What is the difference between > and >> in the shell?
A.> appends; >> overwrites○
B.> overwrites the file; >> appends to it✓
C.They are identical○
D.> creates; >> deletes○
💬 Discussion — why B is correct

> redirects output and overwrites. >> redirects output and appends. Use >> for logs and multi-step file building.
