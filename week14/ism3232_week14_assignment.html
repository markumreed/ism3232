Overview
Build the data access layer for your capstone. All five functions must use parameterised queries (never f-strings in SQL), use sqlite3.Row and dict() for results, and accept an optional db_file parameter so tests can use isolated temporary databases.
Requirements
1
database.py with all five required functions

Each function must:

• Accept an optional db_file=DB_FILE parameter

• Use a with sqlite3.connect(db_file) as conn: context manager

• Use ? placeholders in all SQL — never f-strings or string concatenation

• Have a docstring

The five required functions:

1. create_table(db_file) — CREATE TABLE IF NOT EXISTS, runs without error if table already exists

2. add_record(..., db_file) — inserts one record; accepts all non-auto fields as parameters

3. get_all_records(db_file) — returns a list of dicts using row_factory = sqlite3.Row

4. update_status(record_id, new_status, db_file) — updates one record by ID

5. get_status_report(db_file) — returns GROUP BY status aggregate (count + sum)

2
Never use f-strings or concatenation in SQL

This is a SQL injection rule. Every value in a SQL statement must be passed as a tuple via the second argument to conn.execute():

# CORRECT
conn.execute('INSERT INTO records (name, amount) VALUES (?, ?)', (name, amount))

# WRONG -- never do this
conn.execute(f'INSERT INTO records (name, amount) VALUES ({name}, {amount})')
3
get_all_records returns a list of dicts

Use conn.row_factory = sqlite3.Row and convert each row with dict(row):

def get_all_records(db_file=DB_FILE):
    with sqlite3.connect(db_file) as conn:
        conn.row_factory = sqlite3.Row
        cur = conn.execute('SELECT * FROM ... ORDER BY id DESC')
        return [dict(row) for row in cur.fetchall()]
4
At least five pytest tests using the tmp_path fixture

Use pytest’s built-in tmp_path fixture to create a fresh database for each test:

def test_add_and_retrieve(tmp_path):
    db = tmp_path / 'test.db'
    create_table(db)
    add_record('Taylor', 500.00, db_file=db)
    records = get_all_records(db_file=db)
    assert len(records) == 1
    assert records[0]['amount'] == 500.00

Required tests: add + retrieve, default status check, update_status, multiple records, status report grouping.

5
A test script demonstrating all five functions end-to-end

Write a short test_script.py (not a pytest file) that calls all five functions in sequence with realistic data and prints the results. This should produce readable output confirming everything works before pushing.
