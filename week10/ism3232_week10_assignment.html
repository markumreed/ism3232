Overview
Choose a business domain and build a class that represents a single business entity. The class must have an __init__ constructor with at least four attributes, at least three business logic methods, and a __repr__ method. Create multiple instances and verify they are independent.
Requirements
1
A business entity class in models.py

Choose a domain (purchase requests, leave requests, support tickets, inventory items, etc.). Your class must have:

• __init__ with at least 4 parameters (plus self)

• At least one attribute with a default value set inside __init__ (not passed in) — typically self.status = 'Pending'

• At least three methods with docstrings (not counting __init__ and __repr__)

• At least one method that returns a bool (a business rule check)

• At least one method that changes an attribute (like approve() or activate())

• __repr__ returning a meaningful string showing key attributes

2
main.py: create at least three instances and call every method

Create at least three instances with different data. Call every method on at least one instance. Print the results. Verify that calling a method on one instance does not affect another — demonstrate this explicitly in your output.

3
tests/test_models.py: at least six pytest tests

Required test coverage:

• Default attribute value on a newly created instance

• Each status-changing method (at least 2 methods tested)

• The boolean rule method: one test where it returns True, one where it returns False

• Boundary test for the boolean method (the exact threshold value)

• Instance independence test: changing one instance does not affect another

4
README.md with class design explanation

Write a README that includes: the class name and business domain, a description of each attribute (name, type, purpose), a description of each method (what it does, what it returns), and why you chose this domain for your capstone.

⚠ Common mistakes
  • Forgetting self as the first parameter in every method definition
  • Writing status = 'Approved' inside a method instead of self.status = 'Approved'
  • Not testing instance independence — this is the most important OOP concept to demonstrate
  • Methods that print instead of return — approve() should set self.status and return nothing; tests then check instance.status
