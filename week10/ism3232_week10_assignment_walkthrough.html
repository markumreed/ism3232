Requirement-by-Requirement Walkthrough
1
Business entity class in models.py

Open models.py. Audit the class structure line by line against all structural requirements.

🔍 What the grader checks
__init__ with at least 4 parameters (plus self) — and at least one attribute with a default value set inside __init__ (not passed in) typically self.status = 'Pending'
At least 3 methods beyond __init__ and __repr__, each with a docstring
At least one method returns bool (business rule check)
At least one method changes an attribute (approve(), activate(), close())
__repr__ method present and returns a formatted string showing key attributes
No print() at module level inside the class methods (methods should return, not print)
✅ Full credit class
class BusinessRequest: def __init__(self, req_id, requester, category, amount): self.req_id = req_id self.requester = requester self.category = category self.amount = amount self.status = 'Pending' # default def approve(self): """Mark as approved.""" self.status = 'Approved' def requires_review(self) -> bool: """Return True if above threshold.""" return self.amount > 1000
❌ Common structural failures
def approve(): # missing self self.status = 'Approved' # AttributeError # or: def check(self): print(self.amount > 1000) # print not return
⚠ Common mistakes at this step
self missing on methods — causes TypeError when called
status = 'Approved' inside method instead of self.status — silent bug, attribute unchanged
No __repr__ — print(instance) shows unhelpful memory address
Only 2 methods — need 3+; partial credit 15/25
2
main.py with three instances and instance independence demonstrated

Open main.py. Verify that instance independence is explicitly demonstrated — not just implied.

🔍 What the grader checks
At least 3 instances created with different data
Every method called on at least one instance
Instance independence shown: approve() called on instance 1, then instance 2 status printed and still 'Pending'
Output labelled so the demonstration is readable
💡 What instance independence looks like in code
req1.approve() print(req1.status) # Approved print(req2.status) # Pending — this line is the independence demonstration
⚠ Common mistakes at this step
Three instances created but all the same data — not a useful demonstration
Instance independence not shown — student calls methods but does not print status of the unaffected instance
Only 2 instances — need 3+
3
Six pytest tests covering required types

Open tests/test_models.py. Verify the six required test types are present — not just any six tests.

🔍 What the grader checks
Default attribute value test (req = BusinessRequest(...); assert req.status == 'Pending')
Each status-changing method tested (at least 2 method tests)
Boolean rule method tested True: assert req.requires_review() is True
Boolean rule method tested False: assert req.requires_review() is False
Boundary test for the boolean method (exact threshold value)
Instance independence test (change req1, assert req2 unchanged)
💡 Grading the boundary test
If the approval threshold is 1000, the boundary test must use exactly 1000. assert requires_review(1000) is False (because > not >=). Testing 999 and 1001 is also acceptable as boundary evidence.
⚠ Common mistakes at this step
6 tests present but all test the same method — need coverage of all required types
Independence test missing — this is the most commonly omitted required test (deduct 5 pts)
Boundary test is just another typical case (e.g. amount=500, amount=2000) not the exact threshold
Tests import the wrong thing (import models instead of from models import BusinessRequest)
4
README with class design explanation

Open README.md. Check that attribute descriptions and method descriptions are present.

🔍 What the grader checks
Class name and business domain stated
Every attribute listed with its type and purpose
Every method listed with what it does and what it returns
Brief explanation of why this domain was chosen for the capstone
Known Common Mistakes
⚠ Known common mistakes (from the assignment page)
Forgetting self as the first parameter in every method definition
Writing status = 'Approved' inside a method instead of self.status = 'Approved'
Not testing instance independence — this is the most important OOP concept to demonstrate
Methods that print instead of return — approve() should set self.status and return nothing; tests then check instance.status

💬
Rubric discussion
The test suite (30 pts) is the most heavily weighted item. Six tests that cover only approve() and reject() earn 15/30 maximum — required test types must each be present. The boundary test (5 pts of the 30) is where students most often lose points without realising it — testing 500 and 1500 around a 1000 threshold does not constitute boundary testing.
