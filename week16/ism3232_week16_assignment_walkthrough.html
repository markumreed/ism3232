Requirement-by-Requirement Walkthrough
1
ai_feature.py — function with docstring, no PII, correct model

Open ai_feature.py. Audit the function against all technical requirements.

🔍 What the grader checks
At least one function defined with a clear business purpose
Docstring explains what it does, what is sent to the API, and what it returns
No PII in the prompt — no names, IDs, or amounts passed to the API
Model: claude-haiku-4-5-20251001 (not claude-opus or claude-sonnet)
Returns a string (the AI response text)
client.messages.create() called correctly with role/content structure
✅ Full credit function
def summarise_request(category: str, description: str) -> str: """Return a one-sentence summary. Only category and description sent — no PII.""" client = anthropic.Anthropic() message = client.messages.create( model='claude-haiku-4-5-20251001', max_tokens=150, messages=[{'role': 'user', 'content': f'Summarise this {category} request: {description}'}] ) return message.content[0].text
❌ PII violation
def summarise(requester, amount, description): prompt = f'Summarise request from {requester} for ${amount}: {description}' # PII sent
⚠ Common mistakes at this step
PII in the prompt (name, ID, amount) — deduct 10 pts and note the violation explicitly
Wrong model used — claude-sonnet or claude-opus is more expensive and not required
Function does not return the text — returns the whole message object
2
All six GenAI controls present

Check each of the six required controls explicitly. Missing any one fails this requirement.

🔍 What the grader checks
Control 1 — Disclosure label: AI-generated content visibly labelled in the UI (st.info with disclosure text)
Control 2 — Human review gate: a second button or confirmation step before saving AI output
Control 3 — No PII: verified in Req 1 above
Control 4 — Mocked test: tests/test_ai_feature.py uses unittest.mock.patch
Control 5 — Limitations in README: at least two specific limitations written
Control 6 — Student can explain: verified during presentation
💡 Checking the review gate
Look for two separate st.button() calls: one to generate the AI output (showing it), and a second to confirm saving it. Auto-save after generation is a violation.
⚠ Common mistakes at this step
AI output auto-saved without review gate — Control 2 violated; deduct 10 pts
Mocked test missing entirely — Control 4 violated; deduct 5 pts
Limitations in README say 'may make mistakes' but nothing specific — require specifics (e.g. 'AI may miscategorise technical equipment requests as travel')

💡
Additional grading notes
The final capstone grade represents 30% of the course. It is worth taking 20-30 minutes per student. The most common reasons for low scores: (1) GenAI review gate missing or auto-saving, (2) hardcoded IDs in the Update tab, (3) missing boundary tests in test suite, (4) vague README setup instructions. Leave specific, actionable feedback even on the final submission — students may revisit this code for their portfolios.
💬
Rubric discussion
The presentation (40 of 150 pts) is the most subjective grading element. Use the 8-topic checklist strictly — each covered topic earns 5 pts, each missing topic loses 5 pts. A student who demos all five Streamlit features without explaining the OOP or SQL layer has covered only Topics 5, 7, and 8 (if they mention testing). The GenAI controls (25 pts) are the most nuanced — read each control against the code before grading, not just the README claim.
