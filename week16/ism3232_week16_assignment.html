Overview
The final capstone integrates all units: the professional workflow from Unit 1, the Python programming skills from Unit 2, the OOP system design from Unit 3, and the full-stack application build from Unit 4. You will complete the GenAI feature, finalize your repository, and present your working application covering 8 required topics.
Part A — GenAI Feature (40 points)
1
ai_feature.py with at least one AI function

Write at least one function in ai_feature.py that calls the Anthropic API. The function must:

• Accept content-based parameters only — no names, IDs, or amounts sent to the API

• Have a docstring explaining what it does, what it sends to the API, and what it returns

• Return a string (the AI output)

• Use claude-haiku-4-5-20251001 as the model

Approved feature types: summarisation, classification, drafting, checklist generation, missing-field detection, report explanation.

2
All six required GenAI controls implemented

Every control is required. Missing any one fails this requirement:

• Disclosure label in the Streamlit UI: AI-generated content must be visibly labelled

• Human review gate: AI output is shown to the user for review before it can be saved

• No PII sent to the API: names, IDs, amounts must not appear in the prompt

• Test with mocked API: at least one pytest test using unittest.mock.patch

• Limitations in README: at least two specific limitations of the AI feature

• Student can explain the feature: required in the presentation

Part C — Presentation (50 points)
6
Live presentation covering all 8 required topics

Your presentation must cover all 8 topics. The instructor will note which topics were addressed. Maximum 10 minutes:

8 required presentation topics

1. Business problem + user workflow — what problem does your system solve? walk through a user session

2. Developer workflow — show the terminal: venv active, ruff passing, pytest green, git log visible

3. OOP design — walk through models.py: class names, attributes, methods, relationships

4. SQL schema + key queries — show the CREATE TABLE, a filter query, the GROUP BY report

5. Streamlit demo — live: submit a record, view it, filter, update status, show the report

6. GenAI feature — demo: show the disclosure label and review gate, explain the prompt design

7. Testing + known limitations — show pytest passing, name at least one limitation

8. AI use statement — honest account of what AI tools you used during development

Automation bonus (+10 points)
Students who implement an automation feature earn up to 10 bonus points. Examples: CSV export of the database, Excel report generated with openpyxl, a scheduled daily summary. Submit an automation.py file with documentation in the README explaining what it does and how to run it.
⚠ Common mistakes
  • A single commit called ‘final’ — commit at meaningful milestones throughout the build
  • AI feature that auto-saves output without a human review step — the review gate is required
  • README setup instructions missing — the instructor must be able to clone and run the app from the README alone
  • Presentation that demos only the app without explaining any code — all 8 topics require explanation, not just clicking
