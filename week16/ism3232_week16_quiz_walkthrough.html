How to use this walkthrough
Week 16 closes the course. This walkthrough covers the six required GenAI controls, the final presentation requirements, and the repository checklist. Every question in this quiz has direct capstone rubric implications — Q1 (no PII), Q2 (human review gate), Q3 (mocked tests), Q4 (8 presentation topics), and Q11 (all six controls) are the questions students must answer correctly to receive full capstone credit. Use this walkthrough as a pre-presentation prep session.
Question-by-question walkthrough
1
Multiple Choice — Auto-graded
What is the most important reason NOT to send PII (names, IDs, amounts) to the AI API?
A.It makes the AI slower○
⚠ API response time is not affected by what data is sent in any significant way.
B.PII sent to external APIs may be logged or used for training — sending it violates privacy and data handling responsibilities✓
C.The API does not accept numbers○
⚠ The Anthropic API accepts numbers — the restriction is about privacy and legal responsibility.
D.It costs more tokens○
⚠ Token cost differences from including PII are negligible — the reason is privacy responsibility.
💬 Discussion — why B is correct

Sending personally identifiable information to external APIs creates privacy and legal risks. Prompts should use only generic descriptions — never specific names, IDs, or dollar amounts.

2
Multiple Choice — Auto-graded
What is the purpose of the human review gate in the GenAI feature?
A.To slow down the AI response○
⚠ A review step adds one interaction but prevents wrong data from being stored.
B.To ensure the user reads and approves AI-generated output before it is saved to the database✓
C.To test the AI○
⚠ The review gate is for the student's Streamlit app, not for testing the AI itself.
D.To display the AI model name○
⚠ Displaying the model name is a transparency practice but not the purpose of the review gate.
💬 Discussion — why B is correct

AI output can be wrong or inappropriate. The human review gate (a second confirmation button) ensures a human decides whether to persist AI output. Never auto-save AI output.

3
Multiple Choice — Auto-graded
Why must tests for the AI feature use unittest.mock.patch?
A.pytest requires it○
⚠ pytest does not require mocking — the reason is cost and determinism.
B.To avoid calling the real API during tests — the real API costs money, requires a key, and has network latency✓
C.To make tests run faster only○
⚠ Test speed improvement is a side benefit, not the primary reason.
D.Because Anthropic requires mocking○
⚠ Anthropic does not require mocking — it is a best practice for any external API.
💬 Discussion — why B is correct

Tests must be deterministic and free. Calling the real API in tests costs tokens, requires keys, and fails if network is unavailable. Mock the API call and return a controlled fake response.

Multiple Choice — Auto-graded
What does a meaningful Git commit history demonstrate?
A.That the student pushed frequently○
⚠ Pushing frequently with meaningful messages is better than pushing once — frequency alone is not the measure.
B.That the student worked incrementally with commits at logical checkpoints, not just one final commit✓
C.That the student used Git before this course○
⚠ Prior Git experience does not substitute for demonstrating good workflow in this project.
D.Nothing — only the code matters○
⚠ Code quality matters, but commit history shows process — both are assessed.
💬 Discussion — why B is correct

A history of meaningful commits shows the build process. A single commit called "final" shows nothing about development workflow.

Multiple Choice — Auto-graded
A student's AI feature automatically saves the AI summary without user review. Which control is violated?
A.Disclosure in the UI○
⚠ Disclosure in the UI is about labelling AI output for USERS — the human review gate is the separate control about saving that output.
B.Human review before save✓
C.No PII rule○
⚠ The no-PII rule governs what is sent to the API — the review gate governs what is saved from the API response.
D.Mocked test requirement○
⚠ The mocked test requirement governs test isolation — the review gate governs user confirmation.
💬 Discussion — why B is correct

The human review gate requires the user to actively confirm they want to save the AI output. Automatic saving bypasses this control and is explicitly prohibited.
