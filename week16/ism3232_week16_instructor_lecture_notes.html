GenAI Feature Types and Required Controls 0–15 min

Lecture · The six controls are non-negotiable

🗣 Say — what this feature is
“The GenAI feature is a required part of your application — not just a development tool. It does something useful for the business user: summarising a request, classifying urgency, drafting a response. The AI output appears in the Streamlit interface with disclosure and human review.”
✏ Write the 6 required controls on board
1. AI disclosure label in the UI · 2. Human review before saving · 3. No PII sent to API · 4. Test cases for the feature · 5. Limitations in README · 6. Student can explain the feature
⚠
All six controls are required
Missing any one of the six controls fails the GenAI feature on the capstone rubric. Walk through each control and confirm students understand what it means for their specific implementation.
🗣 Say — the no-PII rule
“Do not send names, IDs, dollar amounts, or any personally identifiable information to the AI API. Send only the category and a generic description. Your prompt should be constructable without any specific record data.”
Section 2 — Build ai_feature.py and Wire to Streamlit
2

Build ai_feature.py and Wire to Streamlit 15–45 min

Live coding · Show disclosure and review gate

ai_feature.py — minimal summarisation
import anthropic

def summarise_request(category: str, description: str) -> str:
    """
    Return a one-sentence AI summary.
    Only category and description sent -- no names or amounts.
    """
    client = anthropic.Anthropic()
    prompt = f'Summarise this {category} business request in one sentence:\n{description}'
    message = client.messages.create(
        model='claude-haiku-4-5-20251001',
        max_tokens=150,
        messages=[{'role': 'user', 'content': prompt}]
    )
    return message.content[0].text
In app.py — disclosure label and human review gate
from ai_feature import summarise_request

description = st.text_area('Request description')

if st.button('Generate AI Summary'):
    with st.spinner('Generating summary...'):
        ai_summary = summarise_request(category, description)
    st.info(f'AI-generated summary (review before saving):\n\n{ai_summary}')
    if st.button('Looks good -- save this summary'):
        add_record(requester, category, amount, notes=ai_summary)
        st.success('Record saved with AI summary.')

Final Demo Presentations 60–80 min

8–10 minutes per student · 8 required topics

🗣 Say — presentation ground rules
“Each presentation covers 8 topics. I will stop you if you skip one. You have 8–10 minutes. Show the running application. Explain the code. Tell me what you would improve.”

💡 Grading the presentation
The capstone grade is 30% of the course. Do not rush through presentations. A student who cannot explain their OOP design or their SQL schema has not met the learning objectives, regardless of whether the application runs.
⚠
Automation bonus — present separately
Students who implemented an automation feature (CSV export, Excel report, scheduled summary) present it after the 8 required topics. Award the +5% if: the feature runs, is documented in the README, and the student can explain how it works.
