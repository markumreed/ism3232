1
Multiple Choice — Auto-graded
What is composition in OOP?
A.Defining a class inside another class○
⚠ Defining a class inside another (nested class) is possible but different from composition.
B.A class that holds instances of another class as an attribute✓
C.A class that inherits from multiple parents○
⚠ Multiple inheritance is possible in Python but it is a different concept from composition.
D.A method that calls itself○
⚠ A method that calls itself is recursion — entirely unrelated to composition.
💬 Discussion — why B is correct

Composition means one class contains instances of another as an attribute. A RequestManager holds a list of BusinessRequest objects.

2
Multiple Choice — Auto-graded
In the manager/entity pattern, which class is responsible for approving a single request?
A.The RequestManager class○
⚠ RequestManager delegates to BusinessRequest for approval. The manager should not implement individual entity behaviour.
B.The BusinessRequest class✓
C.Either class can do it○
⚠ Either class CAN call approve() but in the manager/entity pattern, the entity owns its own behaviour.
D.A separate approver class○
⚠ A separate approver class would add unnecessary complexity. The entity approves itself.
💬 Discussion — why B is correct

The entity class (BusinessRequest) owns the approve() method. The manager holds and filters but delegates status changes to the entity.

3
Multiple Choice — Auto-graded
What does super().__init__() do inside a child class constructor?
A.Calls the grandparent class○
⚠ super() in a child class refers to the immediate parent, not any grandparent.
B.Calls the parent class constructor to initialise inherited attributes✓
C.Creates a second object○
⚠ super().__init__() does not create a second object — it initialises the SAME object being created.
D.Overrides all parent methods○
⚠ super().__init__() runs the parent's constructor — it does not override parent methods.
💬 Discussion — why B is correct

super().__init__() calls the parent class's __init__ to set up inherited attributes. Without it, the child instance would be missing parent-defined attributes.

Multiple Choice — Auto-graded
Which Python syntax defines that TravelRequest is a subclass of BusinessRequest?
A.class TravelRequest extends BusinessRequest:○
⚠ 'class TravelRequest extends BusinessRequest' is Java syntax, not Python.
B.class TravelRequest(BusinessRequest):✓
C.class TravelRequest : BusinessRequest○
⚠ 'class TravelRequest : BusinessRequest' is not valid Python. The colon comes after the closing parenthesis.
D.TravelRequest inherits BusinessRequest○
⚠ 'TravelRequest inherits BusinessRequest' is English description, not Python code.
💬 Discussion — why B is correct

Python uses parentheses in the class definition to indicate inheritance: class Child(Parent):

Multiple Choice — Auto-graded
In the OOP-to-SQL mapping, what does a class correspond to?
A.A row○
⚠ A row is one instance — the class defines the structure of ALL rows.
B.A column○
⚠ A column is one attribute — the class defines all columns together.
C.A table✓
D.A database file○
⚠ A database file corresponds to the entire database, not one class.
💬 Discussion — why C is correct

A class maps to a table. The class name becomes (or suggests) the table name. Each attribute maps to a column.

Multiple Choice — Auto-graded
In the OOP-to-SQL mapping, what does one instance correspond to?
A.A table○
⚠ A table is the class — one instance is one row, not the whole table.
B.A database○
⚠ A database holds multiple tables — it is larger than what one class corresponds to.
C.A row (one record)✓
D.A column○
⚠ A column is one attribute — an instance has ALL its attributes (all columns of one row).
💬 Discussion — why C is correct

One instance is one row in the table. All attributes of that instance become the column values in that row.

7
Multiple Choice — Auto-graded
When should you prefer inheritance over composition?
A.When one class manages a collection of another○
⚠ Composition is the has-a relationship — one class CONTAINS instances of another.
B.When one class IS a specialised version of another (is-a relationship)✓
C.Always○
⚠ Always using composition would eliminate inheritance, which is sometimes the right tool (is-a relationships).
D.When two classes share a name○
⚠ Shared names do not determine the relationship pattern.
💬 Discussion — why B is correct

Use inheritance for is-a relationships (TravelRequest IS a BusinessRequest). Use composition for has-a relationships (Manager HAS a list of Requests).

8
Multiple Choice — Auto-graded
Two manager instances are independent of each other. What does this mean for their requests lists?
A.They share the same list○
⚠ Shared lists only occur if self.requests is defined at the class level (outside __init__). Correct __init__ creates independent lists.
B.Each manager has its own separate list not affected by the other✓
C.The second is a copy of the first○
⚠ The second manager is not a copy of the first — they are completely independent instances.
D.Lists cannot be stored inside objects○
⚠ Lists can absolutely be stored inside objects — this is exactly what the manager pattern does.
💬 Discussion — why B is correct

Each instance has its own copy of every attribute. Adding a request to mgr1 does not affect mgr2's list.
