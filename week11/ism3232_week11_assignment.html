Overview
Add a manager class to models.py that holds a collection of your entity objects and provides add, filter, aggregate, and reporting methods. The manager and entity classes work together — the entity defines what individual records can do; the manager defines what a collection of records can do. Extend your test suite to cover both classes.
Requirements
1
Manager class in models.py (added to your Week 10 file)

The manager class must have:

• __init__ that initialises an empty list

• add(item) — appends an entity instance to the list

• list_by_status(status: str) — returns a filtered list

• total() — returns the sum of a numeric attribute across all instances

• summary_report() — prints a formatted report showing counts and totals by status

• Docstrings on every method

2
main.py: wire both classes together

Create a manager. Add at least four entity instances. Call at least one status-changing method on one instance. Call summary_report() and confirm the output reflects the status change.

3
OOP-to-SQL mapping in README.md

Add a section to README.md that maps your OOP design to SQL. Complete this table:

| OOP concept           | SQL equivalent         | Your system example     |
|-----------------------|------------------------|-------------------------|
| your entity class     | table                  | [your table name]       |
| instance (one object) | row (one record)       | [example values]        |
| key attribute         | column                 | [column name + type]    |
| list_by_status()      | SELECT WHERE status=?  | [your SQL equivalent]   |
| total()               | SELECT SUM(amount)     | [your aggregate query]  |
4
Extend tests to at least ten total (from both weeks)

Your test file must now have at least 10 tests total, including:

• All tests from Week 10 (kept and still passing)

• New tests for the manager class:

   • Manager starts empty

   • add() increases the count

   • list_by_status() returns correct subset

   • total() returns correct sum

   • Two manager instances are independent

⚠ Common mistakes
  • Manager class that re-implements entity logic — the manager should call entity methods, not duplicate them
  • total() that sums the count of records instead of a numeric attribute — it should sum e.g. r.amount
  • OOP-to-SQL table left as placeholder text — every row must have your actual class/table names
  • Week 10 tests removed or broken — the full test suite must still pass
