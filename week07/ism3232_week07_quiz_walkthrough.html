Question-by-question walkthrough
1
Multiple Choice — Auto-graded
What is the purpose of the return statement in a function?
A.Prints the result to the terminal○
⚠ print() sends to the terminal — it does not return a value to the caller. A function that only prints cannot be tested with assert.
B.Sends a value back to the caller✓
C.Ends the script○
⚠ return does end execution of the function, but its purpose is to send a value back — the ending is a side effect.
D.Calls another function○
⚠ Calling another function is done with a function call expression, not return.
💬 Discussion — why B is correct

return sends a value back to wherever the function was called. Without return, the function returns None.

2
Multiple Choice — Auto-graded
What happens when you try to use a variable defined inside a function from outside that function?
A.It works normally○
⚠ Local variables ARE accessible inside the function. It is outside the function where they are inaccessible.
B.You get a NameError — local variables are not accessible outside their function✓
C.It becomes a global variable○
⚠ Variables defined inside a function are explicitly LOCAL. They do not become global automatically.
D.Python copies it to the global scope○
⚠ Python does not copy local variables to global scope. That would break encapsulation entirely.
💬 Discussion — why B is correct

Variables defined inside a function are local. They exist only while the function runs. Accessing them outside raises a NameError.

3
Multiple Choice — Auto-graded
Given a file business_rules.py with a function calculate_total, what is the correct import statement?
A.import calculate_total○
⚠ import calculate_total tries to import a module named calculate_total. The function is inside business_rules.
B.import business_rules.calculate_total○
⚠ import business_rules.calculate_total is not valid Python import syntax.
C.from business_rules import calculate_total✓
D.include business_rules○
⚠ include is not a Python keyword — it is from C/C++.
💬 Discussion — why C is correct

from module import function imports a specific function. This is the pattern used in both main.py and test files.

Multiple Choice — Auto-graded
Which file naming pattern does pytest use to discover test files automatically?
A.Files starting with check_○
⚠ check_ prefix is used in some testing frameworks but not pytest's default discovery.
B.Files ending in _spec.py○
⚠ _spec.py is from RSpec (Ruby). pytest uses test_ prefix.
C.Files starting with test_ or ending in _test.py✓
D.Files named tests.py○
⚠ tests.py is a valid name but pytest discovers by prefix, not exact name.
💬 Discussion — why C is correct

pytest discovers tests in files that start with test_ or end with _test.py. Test functions must also start with test_.

5
Multiple Choice — Auto-graded
Which of the following is the correct way to call a function named get_tier with argument 750?
A.get_tier○
⚠ get_tier alone is just the name — it does not execute the function. You must add parentheses.
B.get_tier()○
⚠ get_tier() calls the function with no argument — this would raise a TypeError if the function expects one.
C.get_tier(750)✓
D.def get_tier(750)○
⚠ def is for DEFINING a function, not calling it. And you cannot put a literal value where a parameter name goes.
💬 Discussion — why C is correct

To call a function, use the name followed by parentheses containing the arguments. def is for defining, not calling.

6
Multiple Choice — Auto-graded
A function that returns True or False based on a business rule is called what?
A.A loop○
⚠ A loop repeats code but does not return a specific type. A predicate is specifically a boolean-returning function.
B.A predicate function✓
C.A void function○
⚠ A void function returns None — it has no return statement or bare return. A predicate always returns bool.
D.A class○
⚠ A class is an object blueprint, not a function that returns bool.
💬 Discussion — why B is correct

A function that returns a boolean is commonly called a predicate. Examples: requires_review(amount), is_eligible(employee).

7
Multiple Choice — Auto-graded
What is the difference between a function parameter and a function argument?
A.They are identical○
⚠ Parameters and arguments are NOT identical — they are related but at different points: definition vs call.
B.A parameter is in the definition; an argument is the value passed at call time✓
C.A parameter is a number; an argument is a string○
⚠ Parameters can be any type, not just numbers. Arguments can be any type, not just strings.
D.Arguments are inside functions; parameters are outside○
⚠ Arguments are passed FROM outside TO inside the function — they are external values, not internal ones.
💬 Discussion — why B is correct

Parameter: the variable name in the definition (def f(price): — price is the parameter). Argument: the actual value passed at call time (f(450.00) — 450.00 is the argument).

8
Multiple Choice — Auto-graded
What does assert do in a pytest test function?
A.Prints the result○
⚠ print() sends to the terminal but the value is not returned and cannot be captured by assert.
B.Raises AssertionError and fails the test if the condition is False✓
C.Defines the expected value○
⚠ Python does allow print() in methods and functions — the issue is testability, not legality.
D.Imports the function○
⚠ Testability is a design issue, not a style preference — it determines whether the function can be verified automatically.
💬 Discussion — why B is correct

assert checks that a condition is True. If False, pytest marks the test FAILED and shows what the actual vs expected values were.
