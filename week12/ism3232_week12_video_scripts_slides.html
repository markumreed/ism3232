🗣 🗣 Why design first

The cognitive load of thinking through what attributes your class needs is different from the cognitive load of getting Python syntax right. If you try to do both at once, you end up with a class that almost does what you want but has the wrong attributes and the wrong methods. Design first. Code second. Commit them in that order.

🗣 🗣 The checklist

Before you submit: One — every method has self as its first parameter. Two — attribute changes use self.attribute = ..., not attribute = ... Three — business logic methods return values, they do not print them. Four — self.requests = [] is inside __init__, not at class level. If you check these four things, you will catch 90% of OOP submission bugs.
