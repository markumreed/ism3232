📋CLASS ANATOMY
class BusinessRequest: """Represents one purchase request.""" def __init__(self, req_id, requester, amount): self.req_id = req_id self.requester = requester self.amount = amount self.status = "Pending" # default value def approve(self): # status-changing method """Mark as approved.""" self.status = "Approved" def requires_review(self) -> bool: # predicate """Return True if above threshold.""" return self.amount > 1000 def __repr__(self): # readable print output return f"Request({self.req_id}, {self.requester}, ${self.amount})"
classBlueprint — defines structure and behaviour
instanceOne specific object created from the class
__init__Constructor — runs when instance is created
selfThe specific instance the method is called on
attributeData stored on an instance: self.amount
methodFunction defined inside a class
__repr__Defines what print(instance) shows
⚠ Every method must have self as its first parameter.
⚠ Use self.status = … not status = … inside methods.
📦INSTANCES & INDEPENDENCE
# Creating instances req1 = BusinessRequest(101, "Taylor", 1200) req2 = BusinessRequest(102, "Jordan", 450) # Calling methods req1.approve() print(req1.status) # → "Approved" print(req2.status) # → "Pending" ← unchanged! # Calling predicates print(req1.requires_review()) # → True (1200 > 1000) print(req2.requires_review()) # → False ( 450 > 1000)
✓ Each instance has its own copy of every attribute.
✓ Changing req1.status does NOT change req2.status.

OOP → SQL MAPPING

REQUIRED TESTS — 6 TYPES

1. Default stateassert req.status == 'Pending'
2. Status changereq.approve(); assert req.status == 'Approved'
3. Bool Trueassert req.requires_review() is True
4. Bool Falseassert req.requires_review() is False
5. Boundaryassert requires_review(1000) is False ← exact threshold
6. Independencereq1.approve(); assert req2.status == 'Pending'
💼COMPOSITION — MANAGER PATTERN
class RequestManager: """Manages a collection of requests.""" def __init__(self): self.requests = [] # MUST be inside __init__ def add_request(self, request): """Add a BusinessRequest.""" self.requests.append(request) def list_pending(self): """Return all Pending requests.""" return [r for r in self.requests if r.status == "Pending"] def total_amount(self): """Return sum of all amounts.""" return sum(r.amount for r in self.requests) def summary_report(self): """Print formatted summary.""" pending = self.list_pending() approved = [r for r in self.requests if r.status == "Approved"] print(f"Total: {len(self.requests)} | ${self.total_amount():,.2f}") print(f"Pending: {len(pending)}") print(f"Approved: {len(approved)}")
⚠ The requests = [] MUST be inside __init__ (as self.requests). If defined at class level, all managers share the same list — a serious bug.

class TravelRequest(BusinessRequest): """Is-a BusinessRequest, lower threshold.""" def __init__(self, req_id, requester, amount, dest): super().__init__(req_id, requester, amount) self.destination = dest def requires_review(self) -> bool: return self.amount > 500 # override: lower
