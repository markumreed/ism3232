Composition — the manager pattern

has-a relationship

Composition means one class holds a collection of another class. RequestManager HAS A list of BusinessRequest objects. The manager coordinates; the entity owns its own methods.

RequestManager — all five required methods
class RequestManager:
    """Manages a collection of BusinessRequest objects."""

    def __init__(self) -> None:
        self.requests = []   # ← instance-level, NOT class-level

    def add_request(self, request) -> None:
        self.requests.append(request)

    def list_by_status(self, status: str) -> list:
        return [r for r in self.requests if r.status == status]

    def total_amount(self) -> float:
        return sum(r.amount for r in self.requests)

    def summary_report(self) -> None:
        pending  = self.list_by_status('Pending')
        approved = self.list_by_status('Approved')
        print(f'Total:    {len(self.requests)} requests')
        print(f'Pending:  {len(pending)}')
        print(f'Approved: {len(approved)}')
        print(f'Value:    ${self.total_amount():,.2f}')
⚠ The class-level list trap

Writing requests = [] outside __init__ makes it class-level — shared by ALL instances. Every RequestManager would see every other manager's requests. Always write self.requests = [] inside __init__.

Light inheritance — the is-a relationship

TravelRequest inherits BusinessRequest
class TravelRequest(BusinessRequest):
    """A travel request with a lower approval threshold."""

    TRAVEL_LIMIT = 500.00

    def __init__(self, req_id, requester, amount, destination):
        super().__init__(req_id, requester, 'Travel', amount)
        self.destination = destination

    def requires_review(self) -> bool:
        return self.amount > self.TRAVEL_LIMIT

is-a test: 'A TravelRequest IS A BusinessRequest.' If that sentence makes sense, inheritance is appropriate. Use super().__init__() to call the parent constructor before adding child-specific attributes.

OOP to SQL mapping

Every OOP concept maps directly to a SQL concept. Understanding this connection makes Week 14 a translation exercise, not a new subject.

class BusinessRequest→ CREATE TABLE requests (...)
instance req_101→ one row in the requests table
self.amount — attribute→ amount REAL column
list_by_status('Pending')→ SELECT * WHERE status = 'Pending'
total_amount()→ SELECT SUM(amount) FROM requests
add_request(req)→ INSERT INTO requests VALUES (...)
get by id→ SELECT * WHERE id = ?
