5
Six-part AI literacy reflection in reflection.md

Open reflection.md and verify all six sections are present and substantive. This is worth 30 of 70 points.

🔍 What the grader checks
Bug 1 section: error type, file/line, hypothesis, print() output, fix
Bug 2 section: expected vs actual, location, fix
AI use section: honest Yes/No, exact prompt if used, what AI explained, what student changed
Can you explain section: Yes/No with proof (explains each fixed line)
What I learned section: 2-3 sentences, specific and not generic
💡 Grading the AI use section
Accept 'No' as a complete answer if the student explains why they did not need AI. For 'Yes' answers: the exact prompt must be present and it must NOT be 'Fix this code for me' — that prompt is explicitly prohibited. If a student used a prohibited prompt, award partial credit and leave a note.
⚠ Common mistakes at this step
reflection.md missing entirely — cap at 40/70 (code fixes only)
All sections present but all answers are one sentence — partial credit proportional to effort
AI use section says 'No' but the code was clearly generated by AI (e.g. completely rewritten style) — note suspicion, do not automatically penalise

Grading Flow
Open GitHub repo. Verify the fixed business_rules.py is present.
Run the provided test file mentally or by cloning — check all tests pass against the submitted code.
Open reflection.md — verify all six major sections are present.
Bug 1 section: check error type named, print() statement shown with output.
Bug 2 section: check expected vs actual documented, fix described.
AI use section: check honesty. If 'Yes' — was the prompt appropriate? If prompt was 'fix this code' — deduct 3-5 pts and note.
Can you explain / What I learned: check for genuine reflection vs one-word answers.
Check ritual screenshot: no DEBUG prints in final code, ruff clean, pytest green.
Total and return. Weight the reflection feedback carefully — this shapes how students approach debugging for the rest of the course.
💡
Additional grading notes
The most common AI policy violation on this assignment is asking AI to 'rewrite' the buggy file. The evidence: the submitted code will have a completely different style than the original, possibly using list comprehensions or different variable names not present in the original. If you see this, require a resubmission with a genuine reflection.
💬
Rubric discussion
Assignment 8 is unique: 30 of 70 points are for the reflection, not the code. A student who fixed both bugs perfectly but submitted no reflection earns 40/70. Grade the reflection generously for honest effort but strictly for completeness — all six sections must be present for full credit. The AI use section in particular rewards honesty: a student who accurately reports using AI and explains how they used it responsibly earns full credit for that section.
