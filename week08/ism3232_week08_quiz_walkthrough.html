How to use this walkthrough
Week 8 is the final Unit 2 week before the midterm. This walkthrough covers traceback reading, print() debugging, the AI use policy, and a direct midterm scope review. Q1 (read bottom-up) and Q5 (permitted AI use) are the most policy-critical questions — they determine how students approach debugging for the rest of the course. Q8 (Debug First, Then Ask workflow) is the most common SA question on the midterm.
Question-by-question walkthrough
1
Multiple Choice — Auto-graded
When reading a traceback, where do you start?
A.The top line (first file mentioned)○
⚠ The top of the traceback shows the outermost call — the least specific information. The error type is always at the bottom.
B.The middle of the output○
⚠ The middle shows intermediate call frames — useful context but not the direct cause.
C.The bottom — the error type and message on the last line✓
D.The line with the most indentation○
⚠ Indentation in a traceback shows call depth, not error severity.
💬 Discussion — why C is correct

Tracebacks are read BOTTOM-UP. The last line tells you exactly what went wrong (error type and message). Lines above show the call chain.

2
Multiple Choice — Auto-graded
What type of error is caused by using a variable before it is defined?
A.TypeError○
⚠ TypeError means the wrong type was used — a name that has not been defined is a NameError.
B.SyntaxError○
⚠ SyntaxError means Python could not parse the code at all — it would not even get to undefined names.
C.NameError✓
D.IndexError○
⚠ IndexError means a list index was out of range — trying to access items[-5] on a 3-item list.
💬 Discussion — why C is correct

NameError: name "x" is not defined occurs when Python encounters a name it has not seen yet. Common causes: typo in variable name, or variable out of scope.

3
Multiple Choice — Auto-graded
A TypeError occurs. Which of these most likely caused it?
A.A list with the wrong number of items○
⚠ A list with wrong number of items would cause an IndexError, not a TypeError.
B.Trying to add a string and an integer together✓
C.A missing colon after if○
⚠ A missing colon after if causes a SyntaxError, caught before the program runs.
D.An empty file○
⚠ An empty file is valid Python — it simply does nothing.
💬 Discussion — why B is correct

TypeError means an operation was applied to the wrong type. Common: "hello" + 5, input() + 100 (input returns str), calling a non-function.

4
Multiple Choice — Auto-graded
What is the purpose of adding print(f"DEBUG: rec = {rec}") inside a loop?
A.To display the loop to the user○
⚠ Display to users should be done with formatted print statements, not DEBUG labels.
B.To inspect the value of rec at each iteration and find where data is wrong✓
C.To count the iterations○
⚠ Counting iterations is done with enumerate() or a separate counter, not f-string inspection.
D.To format the output○
⚠ Formatting output is done in the actual print statements, not debug statements.
💬 Discussion — why B is correct

Strategic print() statements let you see actual values at specific execution points, narrowing down where the bug is.

5
Multiple Choice — Auto-graded
Which of the following is a PERMITTED use of AI in this course?
A.Ask AI to complete the assignment○
⚠ Asking AI to complete an assignment is explicitly prohibited. The submitted work must be the student's own.
B.Ask AI to explain what a traceback error message means✓
C.Ask AI to write all your pytest tests○
⚠ Asking AI to write all tests removes the student's understanding of what the tests verify.
D.Use AI during the midterm exam○
⚠ AI tools are not permitted during the midterm exam.
💬 Discussion — why B is correct

AI may be used to explain error messages, explain what code does, and suggest test cases. It may not generate your submission or be used during the midterm.

Multiple Choice — Auto-graded
The midterm is open notes. What does this mean?
A.Students can use any AI tool○
⚠ AI tools are explicitly not permitted, even during open-note sessions.
B.Students can use their own notes, course slides, and prior assignments✓
C.Students can search the internet○
⚠ Internet searches are not permitted — only the student's own notes and course materials.

⚠ Collaboration is not permitted — the midterm is an individual assessment.

Open notes means personal notes, course materials, and prior assignments are permitted. AI tools, internet searches, and collaboration are not permitted.

8
Multiple Choice — Auto-graded
What must you do BEFORE asking AI for help with a bug?
A.Nothing — ask AI first for speed○
⚠ Nothing — attempting to debug first is always required before any AI use.
B.Read the traceback and attempt to fix with print() debugging first✓
C.Delete the broken code○
⚠ Deleting broken code does not help the student understand what went wrong.
D.Ask a classmate○
⚠ Asking a classmate for help on assignments is an academic integrity issue in most cases.
💬 Discussion — why B is correct

The Debug First, Then Ask workflow: read traceback, attempt fix with print(), document what you tried, THEN ask AI to explain (not fix) the error.
