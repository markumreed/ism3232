This assignment is graded on process

📋
What you are fixing
The instructor provides a broken business_rules.py file (posted to Canvas). It contains at least two bugs — one that causes an immediate crash (traceback-visible error) and one that causes incorrect behaviour (wrong output, no crash). Your job is to find and fix both, using only the traceback and print() debugging.
Requirements
1
Read and document the first traceback

Run the broken script. Copy the traceback. In reflection.md, answer:

• What is the error type on the last line of the traceback?

• Which file and line number is highlighted?

• What does this error type mean in plain English?

• What do you think is wrong before you touch any code?

2
Fix Bug 1 using print() debugging

Add at least one print() statement to inspect the data at the point of failure. In reflection.md:

• Show the print() statement you added

• Show the output it produced

• Explain what the output revealed

• Describe what you changed and why

Remove the print() statement before your final commit.

3
Find and fix Bug 2 (logical error — no crash)

The second bug does not cause a crash but produces wrong output. You must identify it by reading the expected vs actual output carefully. Document in reflection.md:

• What output were you expecting?

• What output did you get?

• Where was the bug?

• What was the fix?

4
All tests pass after your fixes

The instructor provides a test file (tests/test_business_rules.py) alongside the broken script. After fixing both bugs, run pytest -v. All provided tests must pass before you submit.

5
Complete the six-part AI literacy reflection in reflection.md

Required sections in reflection.md:

reflection.md required sections
## Bug 1: Traceback Analysis

[2-3 sentences: what you would do differently next time you hit a bug]
