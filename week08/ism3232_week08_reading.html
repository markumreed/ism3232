Reading tracebacks — always start at the bottom

The rule

The bottom line of a traceback is the most specific information. The top is the least specific. Always read bottom-up: error type → line number → file → work upward only if needed.

Three common tracebacks and what they mean
# KeyError — dict key does not exist (usually a typo)
records = [{'id': 1, 'amount': 1200}]
print(records[0]['amout'])   # typo: 'amout' not 'amount'
# Bottom line: KeyError: 'amout'

# TypeError — wrong type for the operation
amount = input('Enter: ')    # returns str
result = amount + 100        # str + int → crash
# Bottom line: TypeError: can only concatenate str (not 'int') to str

# NameError — variable used before it was defined
def calc(amount):
    result = amount * 1.07
    return result
calc(500)
print(result)    # result is LOCAL — does not exist here
# Bottom line: NameError: name 'result' is not defined

Debugging with print()

Strategic print() debugging
# The buggy script — wrong total, but why?
records = [
    {'id': 101, 'ammount': 1200.00},   # typo in key!
    {'id': 102, 'amount':   350.00},
]
total = 0
for r in records:
    print(f'DEBUG: record = {r}')   # ← add this
    total += r.get('amount', 0)      # .get avoids crash
print(f'Total: ${total:,.2f}')

# The DEBUG line reveals: record 101 has 'ammount' not 'amount'
# Fix the typo, then REMOVE the DEBUG print before committing
⚠ Remove DEBUG prints before committing

Debug prints left in submitted code are visible to graders. They indicate an incomplete submission. Remove them before running the ritual.

Required AI literacy activity — Debug First, Then Ask

This week includes a required structured activity. You will receive a script with two bugs. You must follow the six-step workflow before submitting:

  1. Read the traceback — bottom-up. Write down the error type, file, and line number.
  2. Write your hypothesis — what do you think is wrong? Write it before touching any code.
  3. Add print() statements — inspect variables at the right points. Run and read the output.
  4. If still stuck after 10 minutes — you may ask an AI to explain the error message only. Not to fix the code.
  5. Fix the code yourself — based on your own understanding.
  6. Write the reflection — what was wrong, how you found it, what you learned.
⚠ Reflection is 30 of 70 points

A student who fixes both bugs perfectly but submits no reflection earns 40/70. The reflection is graded on its own and is not optional.
