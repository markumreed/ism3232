Requirement-by-Requirement Walkthrough
1
Search tools — rg and find

Screenshot must show at least two rg commands with output, plus the tree -L 3 output.

🔍 What the grader checks
At least 2 rg commands visible with matching output (not empty results)
find command with *.py or *.md filter visible
tree -L 3 output showing course folder structure
Search run from ~/ism3232/ (not from a subfolder — results should span all modules)
⚠ Common mistakes at this step
rg with no results — student ran it in an empty folder or used the wrong pattern
grep instead of rg — partial credit (5/15); note the course requires ripgrep
find without -name flag — returns everything including directories
2
test_week4.py with three passing tests

Check the repository for the test file. Tests must actually assert something meaningful — not assert True.

🔍 What the grader checks
tests/test_week4.py exists in the repository
tests/__init__.py exists (makes the folder a Python package)
Three test functions present, each starting with test_
Each test has a meaningful assert statement (not assert True)
✅ Passing test
def test_string_upper(): assert 'ism3232'.upper() == 'ISM3232'
❌ Trivial test (no credit)
def test_passes(): assert True
⚠ Common mistakes at this step
Test functions do not start with test_ — pytest will not discover them
No __init__.py in tests/ — pytest may have import issues
Tests have assert statements but they assert constants (1 == 1)
3
Complete pre-submission ritual — all 9 steps

This is the most important deliverable. All 9 steps must be visible in ONE screenshot of a continuous terminal session. Check each step explicitly.

🔍 What the grader checks
pwd — location shown
tree -L 3 — directory structure shown
source .venv/bin/activate — (.venv) prompt appears
ruff format . — runs without error (or shows files reformatted)
ruff check . — runs with no error output (clean)
pytest — shows test results (all passing, green)
git status — shows staged/unstaged state
git add . — runs
git commit -m '...' — commit message visible and descriptive
git push — runs and shows remote push output
✅ Full credit
All 10 steps visible in sequence in one terminal window. Prompt changes to (.venv) after activation. pytest shows 3 passed. git push shows branch pushed.
❌ Partial credit
Only 6 steps visible. Or ruff check shows errors that were not fixed. Or pytest shows failures. Or git push is missing.
⚠ Common mistakes at this step
Ritual run in wrong order (e.g. git add before pytest) — partial credit, note the order requirement
pytest shows failures — submission should not have been pushed; deduct ritual points
ruff check shows errors — formatting was not done or there is a linting error
Commit message is 'final' or 'update' — note professionalism requirement; not a deduction but worth mentioning
4
GitHub repository page visible

Screenshot must show the GitHub repository with files committed. Not just the local terminal — the actual github.com page.

🔍 What the grader checks
github.com URL visible in the screenshot (not localhost)
Repository shows module02_zsh directory structure
tests/ folder and test_week4.py visible in the file listing
.gitignore committed (should not see .venv/ in the listing)
⚠ Common mistakes at this step
Student submitted a terminal screenshot instead of the GitHub page
.venv/ folder visible in the GitHub listing — .gitignore was not created or not committed
Repository is empty — push did not succeed
5
README.md with 15+ commands

Open the README from the GitHub repository. Count the commands. Check descriptions are meaningful.

🔍 What the grader checks
15 or more commands from Weeks 2, 3, and 4 documented
Each command has a description (at least one sentence)
Commands are grouped logically (navigation, file ops, venv, search, git)
Known Common Mistakes
⚠ Known common mistakes (from the assignment page)
Commit message 'final' or 'update' — be descriptive: what did you do this week?
Pushing with .venv/ not in .gitignore — create the gitignore before the first push
Running pytest before ruff — the ritual order is mandatory
Submitting a Canvas upload instead of a GitHub URL — paste the repository URL into the Canvas submission box

Grading Flow
Open the GitHub URL submitted. Verify the repo is accessible and has files.
Check the file structure in GitHub: module02_zsh/, tests/, README.md, .gitignore visible.
Check that .venv/ is NOT visible in the repository (it should be in .gitignore).
Screenshot 1 (search): Verify 2+ rg commands, find, tree -L 3 from ~/ism3232/.
Screenshot 2 (ritual): Walk through all 9 steps — mark each present/absent. This is the most time-consuming check.
Screenshot 3 (GitHub): Verify github.com URL visible, files shown, tests/ folder present.
Clone or browse the repo: check test_week4.py has 3 tests with meaningful assertions.
Check README.md in repo: count commands (need 15+), check descriptions.
Total and return with specific ritual step feedback.
💡
Additional grading notes
Assignment 4 is the first GitHub submission. Some students will have technical issues with remote setup (wrong URL, authentication failures). Award full marks for completed work if a student can demonstrate the push worked even if the URL submission had issues — but require the URL for full submission credit.
💬
Rubric discussion
The ritual screenshot (20 pts) is the highest-weight item and the most important habit of the course. A student who completes 7-8 of 9 steps earns 14-16/20. A student who completes fewer than 5 earns 0-8/20. Do not award partial credit for 'all steps were probably done but not shown' — the screenshot is the evidence.
