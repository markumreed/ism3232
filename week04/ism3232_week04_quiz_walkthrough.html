How to use this walkthrough
Week 4 establishes the professional submission workflow used for every remaining assignment. This walkthrough covers ripgrep, the full 9-step ritual, and the Git basics. Q4 (ritual step order) and Q9 (SA ritual from memory) are the highest-value midterm questions. Q15 (ruff format vs ruff check) is commonly confused and frequently re-tested.
Question-by-question walkthrough
1
Multiple Choice — Auto-graded
What does "rg 'def '" find when run inside a project folder?
A.Files named def○
⚠ rg does not search for files named 'def'. It searches file contents for the pattern 'def '.
B.Every function definition in all Python files✓
C.The definition of rg○
⚠ 'definition of the rg command' — rg is a search tool, not an introspection tool.
D.Only files in the current directory○
⚠ rg searches recursively through all subdirectories by default, not just the current directory.
💬 Discussion — why B is correct

rg (ripgrep) recursively searches for a pattern. "def " matches every Python function definition. rg automatically respects .gitignore.

2
Multiple Choice — Auto-graded
What is the advantage of rg over grep?
A.rg only searches .py files○
⚠ rg searches all files by default, not just .py files. You can add --type py to restrict.
B.rg is much faster and automatically respects .gitignore (skipping .venv/, .git/)✓
C.rg produces coloured output only○
⚠ rg does produce coloured output but that is not its key advantage over grep.
D.rg is installed by default○
⚠ rg is not installed by default — it must be installed via brew or apt.
💬 Discussion — why B is correct

ripgrep respects .gitignore so it skips .venv/ and .git/. It is significantly faster than grep on large codebases.

3
Multiple Choice — Auto-graded
Which step of the pre-submission ritual must come BEFORE git add?
A.git commit○
⚠ git commit comes AFTER git add, not before. You must stage before committing.
B.git push○
⚠ git push comes last, after committing.
C.pytest✓
D.git status○
⚠ git status checks what is staged/unstaged but does not affect what happens — it is informational.
💬 Discussion — why C is correct

The ritual order is: pwd, tree, activate venv, ruff format, ruff check, pytest, git status, git add, commit, push. Pytest must pass before staging.

4
Multiple Choice — Auto-graded
What does "ruff format ." do?
A.Checks code for errors and reports them○
⚠ ruff check REPORTS issues; ruff format FIXES formatting. These are different operations.
B.Auto-formats all Python files in the current directory in place✓
C.Deletes unused imports○
⚠ ruff does not delete unused imports on its own (though it can flag them). format fixes whitespace/style.
D.Runs all tests○
⚠ ruff is a linter/formatter, not a test runner. pytest runs tests.
💬 Discussion — why B is correct

ruff format automatically reformats Python files to follow consistent style. It modifies files in place.

5
Multiple Choice — Auto-graded
If pytest fails during the ritual, what should you do?
A.Run git add and commit anyway○
⚠ Committing with failing tests is explicitly prohibited in this course. The ritual is not complete until pytest is green.
B.Skip pytest and push○
⚠ Pushing with failing tests is the most severe violation of the ritual.
C.Fix the code until all tests pass before continuing✓
D.Delete the failing test○
⚠ Deleting a failing test does not fix the underlying code problem.
💬 Discussion — why C is correct

Submitting with failing tests is not acceptable. Fix the code, re-run pytest, then continue the ritual.

6
Multiple Choice — Auto-graded
What does "git status" show?
A.The history of all commits○
⚠ git log --oneline shows commit history. git status shows the CURRENT state of working files.
B.Which files have been changed, staged, or are untracked✓
C.The current branch name only○
⚠ git status shows status of files, not just the branch name (though branch is also shown).
D.The remote repository URL○
⚠ git status does not show the remote URL — that is git remote -v.
💬 Discussion — why B is correct

git status shows three categories: staged changes (green), unstaged changes (red), and untracked files. Always run before git add.

7
Multiple Choice — Auto-graded
What is the correct order for a first GitHub push?
A.git push, git remote add, git commit○
⚠ Pushing before remote setup fails with 'no remote configured'.
B.git init, git add ., git commit, git remote add origin, git push✓
C.git commit, git init, git push○
⚠ commit before init is impossible — you cannot commit in a non-git folder.
D.git add, git push, git commit○
⚠ add before push skips the commit step — git would reject this.
💬 Discussion — why B is correct

The correct order: init, add, commit, remote add, branch -M main, push. The remote must be connected before pushing.

8
Multiple Choice — Auto-graded
Why should .venv/ be in .gitignore before the first git add?
A.Because Git cannot read binary files○
⚠ Git can track binary files — the issue is not binary vs text, it is size and portability.
B.Because .venv/ is large, platform-specific, and unnecessary — requirements.txt recreates it✓
C.Because GitHub charges for storage○
⚠ GitHub storage costs exist but the primary reason is repo portability.
D.Because pytest requires it○
⚠ pytest works fine regardless of whether .venv is committed — the issue is collaboration.
💬 Discussion — why B is correct

Committing .venv/ inflates the repo with thousands of platform-specific files. requirements.txt is all a collaborator needs.

Multiple Choice — Auto-graded
What does a good commit message look like?
A.'update'○
⚠ 'update' tells the reader nothing about what changed.
B.'fix'○
⚠ 'fix' is slightly better but still non-descriptive — what was fixed?
C.'add calculate_tax function with 7% rate'✓
D.'asdfgh'○
⚠ 'asdfgh' is the textbook example of a meaningless commit message.
💬 Discussion — why C is correct

Good commit messages are specific and describe what changed. They help anyone reading the history understand each commit.

Code Trace / Short Answer — Manual grading

✓ Model Answer
14
Multiple Choice — Auto-graded
What does "ruff check ." do differently from "ruff format ."?
A.ruff check formats; ruff format checks○
B.ruff check reports style and logic issues without changing files; ruff format auto-fixes formatting✓
C.They are identical○
D.ruff check only checks .py files○
💬 Discussion — why B is correct

ruff format changes files. ruff check only reports issues. Run format first (fix style), then check (verify no remaining issues).
