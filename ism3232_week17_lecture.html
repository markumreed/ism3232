Week 17 — Final Demonstrations and Portfolio Submission
Students present their complete business application prototype, walk through their codebase, and submit their final GitHub portfolio. This is the culmination of 17 weeks of professional workflow, Python, OOP, SQL, Streamlit, and responsible AI.
⚠
16-week terms: collapse Week 17 into Week 16
If your term is 16 weeks, the final demo session replaces the Week 16 GenAI work session. Students submit the completed capstone and present in the same final week. Adjust the grading timeline accordingly.
Section 1 — Required Final Repository Structure
Complete capstone repository — every file must exist and be committed
final-project/              # or your chosen repo name
  README.md                 # full project documentation
  app.py                    # Streamlit application
  models.py                 # OOP classes
  database.py               # all five database functions
  ai_feature.py             # GenAI feature
  requirements.txt          # from pip freeze inside .venv
  .gitignore                # .venv/, __pycache__/, *.db, .env
  Makefile                  # optional: make run, make test, make install
  data/
    requests.db             # SQLite database (add to .gitignore or include)
  output/
    summary.txt             # exported report (if applicable)
  screenshots/
    app_screenshot.png
    report_screenshot.png
  tests/
    test_models.py
    test_database.py
    test_ai_feature.py
Section 2 — Required Presentation Topics
#Presentation topicWhat to show or explain
1Business problem and user workflowWhat problem does your system solve? Walk through a user's typical session.
2Developer workflow usedShow your terminal: venv activated, ruff passing, pytest green, git log.
3OOP designWalk through models.py: class names, key attributes, methods, and how they relate.
4SQL schema and key queriesShow the CREATE TABLE, an INSERT, a SELECT with WHERE, and the GROUP BY report.
5Streamlit interface demoLive demo: submit a record, view it, filter it, update its status, show the report.
6GenAI featureDemo the AI feature: show the disclosure label, the review step, explain the prompt.
7Testing and known limitationsShow pytest passing. Describe one known limitation or thing you would improve.
8AI use statementExplain honestly what AI tools you used during development and how.

Section 4 — Automation Bonus (+5%)
⭐
Automation is optional — core requirements come first
Students who have completed all required capstone features and want to earn bonus credit may implement one automation feature: a CSV export of the database, an Excel report generated with openpyxl, a scheduled daily summary email, or a similar workflow automation. Submit automation.py with documentation explaining what it does and how to run it.
Section 5 — Final Submission Checklist
Before the final demo: run this check

☐ All five database functions in database.py pass their tests.

☐ All five Streamlit features work end-to-end in a browser.

☐ The GenAI feature shows disclosure label and human review button.

☐ The README includes: project description, setup instructions, AI feature section with limitations, AI use statement during development.

☐ requirements.txt is current from pip freeze inside the active venv.

☐ .gitignore includes .venv/, __pycache__/, *.db, and .env.

☐ The full pre-submission ritual passes: ruff format, ruff check, pytest, git status clean, git push.

☐ The GitHub repository is accessible to the instructor (set to public or add instructor as collaborator.)

☐ Screenshots folder contains at least two images of the running application.

☐ You can explain every file and every function in your codebase without reading notes.
