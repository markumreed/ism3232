What does the elif keyword mean?
A.else if — checks another condition only if the previous if was False✓
B.else — the final fallback○
⚠ else is the final fallback with no condition. elif always has a condition.
C.else if — always runs after if○
D.A loop keyword○
⚠ elseif is Python 2 / other language syntax. Python uses elif.
💬 Discussion — why A is correct

elif stands for "else if". It checks a new condition only when all previous if/elif conditions evaluated to False.

Multiple Choice — Auto-graded
What does list.append(item) do?
A.Inserts item at position 0○
⚠ insert(0, item) adds to the beginning. append() is specifically for the end.
B.Adds item to the end of the list✓
C.Removes the last item○
⚠ pop() or pop(-1) removes the last item. append() adds.
D.Returns the length of the list○
⚠ len() returns the length — it does not modify the list.
💬 Discussion — why B is correct

append() adds one item to the END of the list. It modifies the list in place and returns None.

What is a list of dictionaries used for in business applications?
A.Storing one value repeatedly○
⚠ A list of identical values is a different pattern — not a record table.
B.Modelling a table of business records — one dict per row✓
C.Sorting numbers○
⚠ Sorting is a list method, not what the list-of-dicts pattern is for.
D.Encrypting data○
⚠ Encryption is unrelated to data structures.
💬 Discussion — why B is correct

A list of dicts is the standard Python pattern for a collection of records before a database is introduced. Each dict is one record (row); the list is the table.

4
Multiple Choice — Auto-graded
Which of the following correctly updates the status in a dictionary?
A.rec["status"] == "Approved"○
⚠ == is a comparison operator that tests equality — it returns True or False but does not change the value.
B.rec["status"] = "Approved"✓
C.rec.status = "Approved"○
⚠ rec.status = 'Approved' is dot notation, which is used for object attributes (class instances), not dictionary keys.
D.rec.update("Approved")○
⚠ rec.update('Approved') is close — dict.update() is a real method, but it takes another dict as argument: rec.update({'status': 'Approved'}). Passing a string raises TypeError.
💬 Discussion — why B is correct

Use square bracket assignment: rec["status"] = "Approved". Double equals == is comparison, not assignment. Dot notation is for objects (classes), not dicts.

Multiple Choice — Auto-graded
In the accumulator pattern, where must the counter variable be initialised?
A.Inside the for loop○
⚠ Initialising inside the loop resets total to 0 on every iteration — the accumulator never grows.
B.After the for loop○
⚠ After the loop, the total is already computed — too late to initialise it.
C.Before the for loop✓
D.Inside an if statement○
⚠ Inside an if statement is inside the loop conditionally — the accumulator would only initialise under certain conditions, not reliably.
💬 Discussion — why C is correct

The accumulator (e.g. total = 0) must be initialised BEFORE the loop. If inside the loop, it resets to 0 on every iteration.

Multiple Choice — Auto-graded
What does items[0] return for: items = ["pen", "notebook", "stapler"]
A."notebook"○
⚠ 'notebook' is at index 1 (zero-indexed: 0='pen', 1='notebook', 2='stapler').
B."stapler"○
⚠ 'stapler' is at index 2 or -1 (last element), not index 0.
C."pen"✓
D.An IndexError○
⚠ An IndexError would only occur if the index is >= len(items). Index 0 always exists for a non-empty list.
💬 Discussion — why C is correct

Lists are zero-indexed. Index 0 is the first element. items[0] returns "pen".

A."pen"○
⚠ 'pen' is at index 0, not -1.
B."notebook"○
⚠ 'notebook' is at index 1 or -2, not -1.
C."stapler"✓
D.An IndexError○
⚠ An IndexError would only occur if the index is out of range. -1 always returns the last element.
💬 Discussion — why C is correct

Index -1 always returns the last element regardless of list length. items[-1] returns "stapler".
