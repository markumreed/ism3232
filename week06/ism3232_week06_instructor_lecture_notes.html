Section 1 — Conditionals
1

Conditionals 0–15 min

Lecture + live demo · Business rule framing throughout

🗣 Say — framing
“Every business rule you have ever seen is a conditional. An approval threshold. An eligibility check. A routing decision. In Python, these are if/elif/else statements. When you write one, you should be able to point to a real business rule it represents.”
✏ Write on board — the business rule to conditional mapping
Business rule: requests over $1000 require manager review → if amount > 1000: ...
Build a multi-tier approval rule together
if amount <= 500:
    print('Auto-approved')
elif amount <= 2000:
    print('Manager approval required')
elif amount <= 10000:
    print('Director approval required')
else:
    print('Executive approval required')
💡 Common mistake — indentation
Python uses indentation to define code blocks. Tab vs spaces issues from copying code cause confusing errors. If a student sees an IndentationError, check that they are using spaces (or tabs) consistently, not mixing them. Ruff format will fix this automatically.
Section 2 — Lists and for Loops
2

Lists and for Loops 15–35 min

Live demo · Accumulator pattern explicitly named

🗣 Say — what lists are for
“A list holds multiple items of the same kind. In a business system, a list might hold all pending requests, all products in a category, or all transactions from today. A for loop processes every item one at a time.”
🗣 Say — the accumulator pattern
“The accumulator pattern is one of the most important patterns in business programming: start a counter at zero, loop through the data, add to the counter each iteration. This is how you compute totals, filtered counts, and summaries.”
Demo the accumulator with business data
amounts = [450, 1200, 89, 3500, 250]

total   = 0       # accumulator starts at zero
flagged = []

for amount in amounts:
    total += amount
    if amount > 1000:
        flagged.append(amount)

print(f'Total: ${total:,.2f}')
print(f'Flagged: {flagged}')
💬 Pause and ask
“If I add another amount to the list, what happens to total without changing any other code?” Look for: it updates automatically. This is the value of the loop pattern.
Section 3 — Dictionaries as Business Records
3

Dictionaries as Business Records 35–50 min

Live demo · Introduce the list-of-dicts pattern explicitly

🗣 Say — what a dict is
“A dictionary stores data as key–value pairs — like a single row in a spreadsheet. A business request has a requester, a category, an amount, and a status. A dictionary holds all of these together under one name.”
🗣 Say — list of dicts
“A list of dictionaries is a collection of records. It mirrors a database table. One dict = one row. A list of dicts = the whole table. You will use this pattern for the rest of the course until we replace it with actual SQL in Week 13.”
Demo the list-of-dicts pattern
requests = [
    {'id': 101, 'requester': 'Taylor', 'amount': 1200, 'status': 'Pending'},
    {'id': 102, 'requester': 'Jordan', 'amount': 450,  'status': 'Pending'},
    {'id': 103, 'requester': 'Morgan', 'amount': 3500, 'status': 'Approved'},
]

for req in requests:
    if req['status'] == 'Pending':
        print(f"Pending: {req['requester']} - ${req['amount']}")
✏ Write on board — the data pattern chain
dict = one record · list of dicts = a table of records · for loop = process the table · if inside loop = apply a business rule · accumulator = compute a summary
Section 4 — Live Coding — Business Record Processor
4

Live Coding — Business Record Processor 50–70 min

Type together · Students extend for assignment

🗣 Say before starting
“This script combines everything from Weeks 5 and 6. It is the Module 4 capstone. Students will extend it for Assignment 6.”
business_record_processor.py
requests = [
    {'id': 101, 'requester': 'Taylor', 'amount': 1200, 'status': 'Pending'},
    {'id': 102, 'requester': 'Jordan', 'amount': 450,  'status': 'Pending'},
    {'id': 103, 'requester': 'Morgan', 'amount': 3500, 'status': 'Approved'},
    {'id': 104, 'requester': 'Riley',  'amount': 89,   'status': 'Pending'},
]
APPROVAL_LIMIT = 1000
total, flagged = 0, []

for req in requests:
    if req['status'] == 'Pending':
        total += req['amount']
        if req['amount'] > APPROVAL_LIMIT:
            flagged.append(req)

print(f'Total pending: ${total:,.2f}')
print(f'Needing review: {len(flagged)}')

with open('output/summary.txt', 'w') as f:
    f.write(f'Total: ${total:,.2f}\n')
💡 Watch for — the output/ folder
Students will get a FileNotFoundError if output/ does not exist. Walk through creating it with mkdir output before running. This is a teachable moment about relative paths and file system state.
