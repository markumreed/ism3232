Requirement-by-Requirement Walkthrough
1
Five records as list of dicts

Open the script. Verify the data structure is a Python list of dictionaries with at least 5 records and consistent keys.

🔍 What the grader checks
Python list (square brackets) containing dictionaries (curly braces)
At least 5 records
All records have identical keys — no missing keys on any record
A status field present on each record (Pending, Approved, or similar)
At least some records have different status values to enable filtering
⚠ Common mistakes at this step
Records stored as parallel lists (names = [...], amounts = [...]) instead of list of dicts
Keys inconsistent across records — e.g. 'amount' in some, 'amt' in others
All records have the same status — filtering produces all-or-nothing results
2
Two distinct business rules in the loop

Both rules must be inside the for loop as separate if statements. They must test different conditions.

🔍 What the grader checks
for loop iterates over the list of records
Rule 1: a conditional inside the loop (e.g. filter by status)
Rule 2: a different conditional inside the loop (e.g. flag by amount threshold)
Both rules produce observable output — a count, a list, or a printed flag
Threshold constants defined at the top with ALL_CAPS names
✅ Two distinct rules
if rec['status'] == 'Pending': # Rule 1: status filter total += rec['amount'] if rec['amount'] > LIMIT: # Rule 2: threshold flag flagged.append(rec)
❌ One rule disguised as two
if rec['amount'] > 1000: # Rule 1 count1 += 1 if rec['amount'] > 2000: # Same field, same logic, different threshold — marginal
⚠ Common mistakes at this step
Both rules test the same field with different thresholds — partial credit (10/20), note the requirement
Rules defined outside the loop (applied globally) instead of inside it
No threshold constants — hardcoded numbers inside the loop
3
Accumulator pattern

Look for a variable initialized to 0 (or []) before the loop, updated inside it.

🔍 What the grader checks
Accumulator variable initialized before the loop (total = 0, flagged = [])
Updated inside the loop body (total += rec['amount'])
Value printed in the summary output
Accumulates the correct field (amount, not record count)
⚠ Common mistakes at this step
Total computed with sum() outside the loop — works but does not demonstrate the accumulator pattern (award 10/15)
Accumulator initialized inside the loop — resets to 0 every iteration
Accumulates count instead of numeric sum — partial credit (8/15)
4
Formatted summary output

Check the terminal screenshot. Must be labelled, 4+ lines, currency formatted.

🔍 What the grader checks
At least 4 lines in the summary
Each line labelled (not raw numbers)
Currency values formatted with :.2f or :,.2f
Lists of flagged records printed with details (not just count)
5
data/summary.txt file

Check the repository for the data/ folder and summary.txt. Open it and verify 3+ lines.

🔍 What the grader checks
data/ folder exists in the repository
data/summary.txt file exists and is committed
File contains at least 3 lines of meaningful data (not empty, not just a heading)
os.makedirs('data', exist_ok=True) used before open() — check the code
⚠ Common mistakes at this step
data/summary.txt missing — most common failure; student forgot os.makedirs or the path was wrong
File committed but empty — pip freeze ran before writing, or file was opened in 'r' mode
data/ folder not committed because of .gitignore misfire
6
Three pytest tests verifying business rule logic

Tests must verify the actual logic — not just Python built-ins.

🔍 What the grader checks
tests/test_week6.py with 3+ test functions
Tests construct small lists of records and assert filtered/accumulated results
At least one test verifies the status filter logic
At least one test verifies the amount threshold rule
Known Common Mistakes
⚠ Known common mistakes (from the assignment page)
Applying rules outside the loop instead of inside it — both conditionals must be inside the for block
Using the same condition for both rules — rules must be distinct (different fields or thresholds)
data/summary.txt missing because the data/ folder doesn't exist — use os.makedirs('data', exist_ok=True)
Loop accumulates the wrong variable — verify your accumulator adds rec['amount'], not the record itself

Rubric discussion
The two-business-rules requirement (20 pts) is the most nuanced to grade. Two rules on the same field (amount > 1000 and amount > 2000) earn partial credit (10/20) because they demonstrate the same logic, not two distinct business concepts. Full credit requires testing different dimensions of the data (status AND amount, or category AND amount). The accumulator (15 pts) awards 10/15 for sum() outside the loop — it shows understanding of the concept without the pattern.
