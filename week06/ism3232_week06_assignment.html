Overview
Build a business record processor in a domain of your choosing. The processor must store at least five records as a list of dictionaries, loop through them applying two distinct business rules using conditionals, produce a formatted summary, and save the output to a file. Test your logic with pytest.
Requirements
1
At least five business records as a list of dicts

Each record must have at least four keys: an ID, a name or identifier, a numeric amount, and a status field. All records must use consistent key names across the list. At minimum, some records should have a ‘Pending’ status and others a different status to allow filtering.

2
A for loop with at least two distinct conditional business rules

Your loop must apply two separate business rules using if statements. Examples:

• Rule 1: Filter by status (e.g., only process Pending records)

• Rule 2: Flag records that exceed a threshold amount

Both rules must produce observable output (a count, a list, a flag). Define meaningful threshold constants using ALL_CAPS variable names at the top of the script.

3
Accumulator pattern — compute a running total

Use the accumulator pattern to compute a running total of at least one numeric field across the filtered records. The total must be displayed in the formatted summary.

4
Formatted summary output

Print a human-readable summary that includes: total count of records processed, count(s) matching each business rule, the running total, and a labelled list of records that triggered each rule.

5
Save output to data/summary.txt

Write the summary (or a condensed version of it) to data/summary.txt. Use os.makedirs('data', exist_ok=True) before opening the file. The file must contain at least three lines of meaningful data.

6
At least three pytest tests in tests/test_week6.py

Tests must verify the logic of your business rules directly — not just that Python arithmetic works. At minimum, test the filtering/accumulation logic by constructing small lists of records in the test functions.

⚠ Common mistakes
  • Applying rules outside the loop instead of inside it — both conditionals must be inside the for block
  • Using the same condition for both rules — rules must be distinct (different fields or thresholds)
  • data/summary.txt missing because the data/ folder doesn't exist — use os.makedirs('data', exist_ok=True)
  • Loop accumulates the wrong variable — verify your accumulator adds rec['amount'], not the record itself
