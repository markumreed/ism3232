Conditionals — business rules as code

Business rules are conditionals

Every approval threshold, every routing rule, every eligibility check is an if/elif/else statement. The code structure mirrors the business policy.

Multi-tier approval routing
LIMIT_AUTO     =  500.00
LIMIT_MANAGER  = 2000.00
LIMIT_DIRECTOR = 5000.00

def get_approver(amount: float) -> str:
    if amount <= LIMIT_AUTO:
        return 'Auto-approved'
    elif amount <= LIMIT_MANAGER:
        return 'Manager'
    elif amount <= LIMIT_DIRECTOR:
        return 'Director'
    else:
        return 'Board'

# Test every boundary
for amt in [500, 501, 2000, 2001, 5000, 5001]:
    print(f'${amt:>6,.2f}  →  {get_approver(amt)}')
⚠ Test boundary values

Testing with 100 and 1000 is not enough. Test exactly at the threshold (500) and one unit above (501). The boundary is where bugs hide.

Lists and the accumulator pattern

List operations and the accumulator
amounts = [200.00, 1500.00, 400.00, 2200.00, 850.00]

# The accumulator pattern — initialise BEFORE the loop
total   = 0.0    # ← OUTSIDE the loop
flagged = []     # ← OUTSIDE the loop

for amount in amounts:
    total += amount              # add to running total
    if amount > 1000:
        flagged.append(amount)   # collect matching items

print(f'Total:   ${total:,.2f}')       # $5,150.00
print(f'Flagged: {flagged}')            # [1500.0, 2200.0]
⚠ Initialise outside the loop

If you write total = 0 inside the loop body, it resets to zero on every iteration. The accumulator must be created before the loop starts.

Dictionaries — one record per dict, many records per list

Dict operations and the list-of-dicts pattern
# One record = one dict
request = {
    'id':        101,
    'requester': 'Taylor',
    'category':  'Equipment',
    'amount':    1250.00,
    'status':    'Pending',
}

print(request['requester'])  # Taylor
request['status'] = 'Approved'  # update a field

# Many records = list of dicts (one dict = one row)
records = [
    {'id': 101, 'requester': 'Taylor',  'amount': 1250.00, 'status': 'Pending'},
    {'id': 102, 'requester': 'Jordan',  'amount':  350.00, 'status': 'Pending'},
    {'id': 103, 'requester': 'Morgan',  'amount': 2800.00, 'status': 'Approved'},
]

# Filter: all Pending records
pending = [r for r in records if r['status'] == 'Pending']
print(f'Pending: {len(pending)} records')

# Total pending amount
total = sum(r['amount'] for r in pending)
print(f'Total pending: ${total:,.2f}')
The list-of-dicts table

One dict = one row. A list of dicts = the table. The dict keys are the column names. In Week 14 we replace this list with a real SQLite database — the structure and the operations are identical.

Key terms this week

if / elif / elseBranching statement — runs different code depending on which condition is True.
for loopIterates over every item in a sequence, executing the loop body once per item.
accumulator patternInitialise a variable before the loop; update it inside. Produces a running total or collected list.
append()Adds one item to the end of a list.
dict key accessrecord['key'] — retrieves the value for that key.
list-of-dictsOne dict per record; the list holds all records. One dict = one row, list = the table.
