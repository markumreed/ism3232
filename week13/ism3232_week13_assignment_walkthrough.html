Requirement-by-Requirement Walkthrough
1
PROPOSAL.md — all ten fields complete

Open PROPOSAL.md. Read every field. Award points based on completeness and specificity — vague answers lose points.

🔍 What the grader checks
Project Name: clear, descriptive, not 'My Project'
Business Problem: specific problem, specific user, 2-3 sentences
Primary User: named and described (not just 'users')
Records Stored: every type of record named
Business Rules: at least 2, each specific and testable (not 'validate data')
OOP Classes: each class with attribute list and method list
SQL Tables: each table with column names and types
Streamlit Features: all 5 named
GenAI Feature: exactly what AI will do (not 'use AI')
Out of Scope: at least 2 specific exclusions
✅ Full credit business rule
Requests over $1000 require manager approval before status can change to Approved
❌ No credit business rule
Validate data / Use business logic / Check inputs
⚠ Common mistakes at this step
Out of Scope left blank — deduct 3 pts; this field is required
GenAI Feature says 'use AI for something' — must specify exactly what (summarise, classify, draft)
Business rules are not actually rules (no threshold, no condition, no consequence)
OOP Classes listed without attribute or method detail
2
schema.sql with correct SQL

Open schema.sql. Run it mentally or in sqlite3 and verify it is correct SQL.

🔍 What the grader checks
CREATE TABLE IF NOT EXISTS (not CREATE TABLE alone)
id INTEGER PRIMARY KEY AUTOINCREMENT
At least 3-4 other columns with appropriate types (TEXT, INTEGER, REAL — not VARCHAR or NUMBER)
NOT NULL on required fields
status column with DEFAULT 'Pending' or equivalent
✅ Correct schema
CREATE TABLE IF NOT EXISTS requests ( id INTEGER PRIMARY KEY AUTOINCREMENT, requester TEXT NOT NULL, amount REAL, status TEXT DEFAULT 'Pending' );
❌ Wrong types
CREATE TABLE requests ( id INT, -- should be INTEGER PRIMARY KEY AUTOINCREMENT amount VARCHAR(50) -- should be REAL );
⚠ Common mistakes at this step
Using VARCHAR instead of TEXT — not standard SQLite; partial credit (15/25)
Missing PRIMARY KEY AUTOINCREMENT — student will have to manage IDs manually
status column missing — every capstone table needs a status field
NOT NULL missing on obviously required fields (requester name, etc.)
3
sqlite3 shell verification screenshot

Check the screenshot for all four required operations in the sqlite3 shell.

🔍 What the grader checks
.schema tablename — shows the CREATE TABLE statement
INSERT INTO — at least one row inserted with actual values
SELECT * — all records shown with column headers (.mode column .headers on visible)
GROUP BY query — status report showing counts or totals per group
⚠ Common mistakes at this step
Screenshot is from Python sqlite3 module, not the shell (.tables and .schema are shell commands)
GROUP BY query missing — most commonly omitted
.headers not on — data shows without column names (partial credit)
Known Common Mistakes
⚠ Known common mistakes (from the assignment page)
Business rules that are not specific: “validate data” is not a rule. “Requests over $1000 require manager approval” is a rule.
schema.sql not tested before submitting — load it in sqlite3 and confirm it runs without errors
Using TEXT for numeric fields — amounts should be REAL, counts should be INTEGER
Proposal Out of Scope left blank — every capstone has things it does not do; naming them shows clear thinking

Additional grading notes
Assignment 13 requires instructor sign-off before the student may begin Week 14. Leave a Canvas comment explicitly stating 'Approved to proceed to Week 14' or 'Revisions required before proceeding'. Students who did not receive approval before starting Week 14 risk building the wrong system.
💬
Rubric discussion
The proposal (30 pts) is the most important intellectual deliverable of the capstone. A technically complete schema with a vague proposal earns less than a precise proposal with a minor SQL issue. Grade the business rules section strictly — a non-testable rule cannot be implemented. The schema (25 pts) should be checked against SQLite column type standards: TEXT not VARCHAR, REAL not FLOAT, INTEGER not INT.
