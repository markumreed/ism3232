How to use this walkthrough
Week 13 transitions from OOP to the capstone build. This walkthrough covers the project proposal requirements and all required SQL operations. The UPDATE without WHERE question (Q7, Q12) is the highest-stakes SQL safety concept in the course. Students who do not internalise this will accidentally update all records in their capstone database.

Multiple Choice — Auto-graded
Which SQL keyword creates a new table?
A.MAKE TABLE○
⚠ 'MAKE TABLE' is not SQL syntax.
B.INSERT TABLE○
⚠ 'INSERT TABLE' is not SQL syntax — INSERT is for adding rows, not creating tables.
C.CREATE TABLE✓
D.NEW TABLE○
⚠ 'NEW TABLE' is not SQL syntax.
💬 Discussion — why C is correct

CREATE TABLE is the SQL DDL statement for defining a new table structure. CREATE TABLE IF NOT EXISTS is preferred because it does not error if the table already exists.

3
Multiple Choice — Auto-graded
What is the purpose of INTEGER PRIMARY KEY AUTOINCREMENT?
A.It stores text IDs○
⚠ Text IDs would be stored as TEXT, not INTEGER PRIMARY KEY.
B.It creates a unique auto-incrementing integer ID for each row managed automatically by the database✓
C.It requires the user to provide the ID○
⚠ Without AUTOINCREMENT, the user would need to supply the ID on every INSERT.
D.It links two tables○
⚠ Linking tables is done with FOREIGN KEY constraints, not PRIMARY KEY.
💬 Discussion — why B is correct

PRIMARY KEY enforces uniqueness. AUTOINCREMENT means the database assigns the next integer ID automatically. You never provide the ID when inserting.

4
Multiple Choice — Auto-graded
What SQL keyword filters the rows a query returns?
A.GROUP○
⚠ GROUP is part of GROUP BY — alone it is not a clause.
B.ORDER○
⚠ ORDER BY sorts results but does not filter rows.
C.WHERE✓
D.HAVING○
⚠ HAVING filters AFTER grouping — WHERE filters before. HAVING is for aggregate conditions.
💬 Discussion — why C is correct

WHERE filters rows before grouping or sorting. Example: SELECT * FROM requests WHERE amount > 1000 returns only rows exceeding 1000.

5
Multiple Choice — Auto-graded
What is the difference between TEXT, INTEGER, and REAL column types in SQLite?
A.They are all the same○
⚠ They are not the same — SQLite enforces type affinity and incorrect types cause sorting and computation errors.
B.TEXT stores characters, INTEGER stores whole numbers, REAL stores decimals✓
C.Only TEXT can be used in WHERE clauses○
⚠ All column types can appear in WHERE clauses.
D.REAL is for dates○
⚠ REAL is for decimal numbers. Dates are typically stored as TEXT in ISO format in SQLite.
💬 Discussion — why B is correct

Type matters for storage and sorting. Text amounts sort alphabetically not numerically. Use REAL for money, INTEGER for counts and IDs, TEXT for names.

Multiple Choice — Auto-graded
What sqlite3 shell command lists all tables in the current database?
A..list○
⚠ '.list' is not a sqlite3 shell command.
B..tables✓
C..schema○
⚠ '.schema' shows the CREATE TABLE statement for one table. '.tables' lists all tables.
D..show○
⚠ '.show' shows sqlite3 configuration settings, not tables.
💬 Discussion — why B is correct

.tables lists all tables in the open SQLite database. Useful for confirming a table was created correctly.

7
Multiple Choice — Auto-graded
What does the WHERE clause in an UPDATE statement do?
A.Filters which columns to update○
⚠ WHERE filters ROWS, not columns. Filtering columns is done with SELECT specific_column.
B.Targets only the specific rows matching the condition — without it, ALL rows are updated✓
C.Makes the query faster○
⚠ WHERE does not inherently affect query speed — that is the role of indexes.
D.Is optional with no side effects○
⚠ WHERE is NOT optional with no side effects. Omitting it from UPDATE affects every row.
💬 Discussion — why B is correct

Without WHERE, UPDATE changes every row in the table. Always filter to the specific id when updating one record.

Multiple Choice — Auto-graded
A capstone business rule must be — choose the best answer:
A.Vague and general, like "validate data"○
⚠ Canvas requirements exist but the reason to be specific is testability and implementability.
B.Specific and testable, like "Requests over $1000 require manager approval"✓
C.Related to SQL only○
⚠ Business rules do not have to involve SQL — they are implemented as Python conditionals.
D.A Python function name○
⚠ Business rules are expressed as conditionals, not function names.
💬 Discussion — why B is correct

A good business rule is specific enough to implement as a conditional and test with pytest. Vague rules cannot be implemented or tested.
