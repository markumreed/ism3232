Overview
Two deliverables this week: a complete 10-field project proposal that defines every layer of your capstone, and a working SQL schema that you have tested in the sqlite3 shell. Getting both right this week means a clean build in Weeks 14–16.

2
schema.sql — the CREATE TABLE statement for your capstone

Write the SQL schema for your main table. Requirements:

• Use CREATE TABLE IF NOT EXISTS

• Include an id INTEGER PRIMARY KEY AUTOINCREMENT

• Include NOT NULL on required fields

• Include a status TEXT DEFAULT 'Pending' or equivalent

• Column types must be correct: TEXT, INTEGER, REAL

3
Verify schema in the sqlite3 shell

Load and test your schema:

sqlite3 capstone.db
.read schema.sql
.schema
INSERT INTO [yourtable] ([cols]) VALUES ([values]);
SELECT * FROM [yourtable];
SELECT [col], COUNT(*) FROM [yourtable] GROUP BY [col];
.quit

Screenshot required: the shell showing .schema output, a test INSERT, a SELECT, and the GROUP BY query.

⚠ Common mistakes
  • Business rules that are not specific: “validate data” is not a rule. “Requests over $1000 require manager approval” is a rule.
  • schema.sql not tested before submitting — load it in sqlite3 and confirm it runs without errors
  • Using TEXT for numeric fields — amounts should be REAL, counts should be INTEGER
  • Proposal Out of Scope left blank — every capstone has things it does not do; naming them shows clear thinking
