How to use this walkthrough
Week 15 introduces Streamlit — the final layer of the capstone application. This walkthrough explains every question with emphasis on the five required features and the Streamlit execution model. Q2 (entire script reruns) is the most architecturally important concept — students who do not understand it will place create_table() in the wrong location and have confusing state bugs. The FastAPI questions (Q6, Q7, Q10) are concepts-only and require no Streamlit knowledge.
Question-by-question walkthrough
1
Multiple Choice — Auto-graded
Why is create_table() called at the top of app.py, outside any tab or button block?
A.Because it needs to run before any imports○
⚠ Imports run before create_table() regardless of where create_table() is called.
B.Because it is idempotent — CREATE TABLE IF NOT EXISTS does nothing if the table exists, but creates it on first run✓
C.Because Streamlit requires it○
⚠ Streamlit does not require create_table() at the top — idempotency is the reason.
D.Because it is faster there○
⚠ The performance difference is negligible — correctness and idempotency are the reasons.
💬 Discussion — why B is correct

create_table() is idempotent. Calling it at startup ensures the table always exists before any widget runs.

2
Multiple Choice — Auto-graded
What happens to the Streamlit app when the user clicks a button?
A.Only the button code runs○
⚠ Only the button code running is the React/JavaScript model. Streamlit is different.
B.The entire script reruns from top to bottom✓
C.The browser reloads the page○
⚠ The browser does not reload — Streamlit uses WebSockets to update the UI without reloading.
D.Only the changed widget updates○
⚠ Only the changed widget updating is also the React model. Streamlit reruns everything.
💬 Discussion — why B is correct

Streamlit reruns the entire script from top to bottom on every user interaction. This is the Streamlit execution model — everything is recalculated each time.

3
Multiple Choice — Auto-graded
What does st.success("Request submitted!") display?
A.An error message in red○
⚠ An error message in red is st.error().
B.A green success notification banner✓
C.A popup dialog○
⚠ A popup dialog would require st.dialog() (newer Streamlit API) or workarounds.
D.A progress bar○
⚠ A progress bar is st.progress().
💬 Discussion — why B is correct

st.success() displays a green notification message. The four status message functions are: success (green), error (red), warning (yellow), info (blue).

C.st.show_data()○
⚠ st.show_data() is not a real Streamlit function.

Multiple Choice — Auto-graded
In the Update Status tab, why should the record selectbox be built from get_all_records() instead of hardcoded IDs?
A.get_all_records is faster○
⚠ get_all_records() is not inherently faster — the reason is data correctness.
B.Hardcoded IDs break when records are deleted or added — live data ensures the dropdown always shows current records✓
C.It is required by sqlite3○
⚠ sqlite3 has no such requirement about form data.
D.Because forms require live data○
⚠ Live data is not a general requirement for all forms — it is specifically needed here because the records change.
💬 Discussion — why B is correct

Building the selectbox from live database data ensures it reflects the current state. Hardcoded IDs become stale when records change.

Multiple Choice — Auto-graded
What is FastAPI?
A.A faster version of Python○
⚠ FastAPI is not a faster Python interpreter — it is a web framework.
B.A web framework for building REST APIs with Python✓
C.A testing library○
⚠ FastAPI is not a testing library — pytest is used for testing.
D.A Streamlit plugin○
⚠ FastAPI is not a Streamlit plugin — it is a separate independent web framework.
💬 Discussion — why B is correct

FastAPI is a Python web framework for building REST APIs. It handles HTTP requests, validates input, and returns JSON responses.

7
Multiple Choice — Auto-graded
What HTTP method is conventionally used to CREATE a new resource in a REST API?
A.GET○
⚠ GET retrieves data without side effects.
B.PUT○
⚠ PUT updates an existing resource with full replacement.
C.POST✓
D.DELETE○
⚠ DELETE removes an existing resource.
💬 Discussion — why C is correct

POST is the conventional HTTP method for creating new resources. GET retrieves, POST creates, PUT updates (full replacement), DELETE removes.

8
Multiple Choice — Auto-graded
A student's filter selectbox always shows all records regardless of the selected status. What is most likely wrong?
A.The database query is wrong○
⚠ The database query uses parameterised queries — it is correct.
B.The filter condition uses == instead of is○
⚠ 'is' tests object identity, not value equality. == is correct for status comparison.
C.The filter is applied but the result is not used to replace the records variable before displaying✓
D.The selectbox is missing options○
⚠ Missing options would cause a different problem — the selectbox would be empty, not show all records.
💬 Discussion — why C is correct

Common mistake: filter but display the original records list. Fix: records = [r for r in records if r["status"] == status_filter]; st.dataframe(records)
