🗣 🗣 Script

When a user clicks a button in Streamlit, Python runs your entire app.py file from line one to the last line. Not just the button handler. The whole file. This is different from every other framework you may have encountered. It is the Streamlit execution model. Most Streamlit bugs exist because a developer assumed only part of the code would run.

🗣 🗣 Why create_table at top

create_table() uses CREATE TABLE IF NOT EXISTS. It is idempotent — calling it 100 times is identical to calling it once. Put it at the very top, outside any tab or button block. This ensures the table exists before any widget runs on any rerun. If you put it inside a tab, it only runs when that tab is visited.

🗣 🗣 Streamlit vs FastAPI

In Streamlit, Python generates the interface directly. The user clicks a button, Python reruns, Python creates the HTML. Python and UI are coupled. In FastAPI, Python only handles data. It receives an HTTP request, queries the database, returns JSON. A separate client — a browser, a mobile app, another Python script — sends requests and displays the data. FastAPI is the data layer. The client is the interface layer. They are separate.

SAY: In Streamlit, Python generates the interface directly. The user clicks a button, Python reruns, Python creates the HTML. Python and UI are coupled. In FastAPI, Python only handles data. It receives an HTTP request, queries the database, returns JSON.
